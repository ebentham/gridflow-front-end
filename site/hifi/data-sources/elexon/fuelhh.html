Overview

What this dataset is.

fuelhh is the half-hourly generation outturn broken out by fuel type for the entire GB electricity system. Elexon publishes one row per fuel type per settlement period — so each 30-minute window produces ~16 rows covering all technology types from CCGT to nuclear to imports.

It is sourced from the BMRS Insights API at /datasets/FUELHH using the publishDateTimeFrom / publishDateTimeTo pattern, queried in rolling 24-hour windows. The raw JSON lands in bronze, is validated against the Pydantic schema, and written to the silver parquet partition.

Data is available back to 2014-04-01 and is updated every 30 minutes. The Gridflow backfill is complete to that date — see the changelog entry of 2026-04-29.

Schema

Silver layer columns.

Defined in gridflow/schemas/elexon.py · ElexonFuelHH. The silver table is partitioned by settlement_date.

settlement_date PK date — GB settlement date (local calendar date, not UTC). Partition key.
settlement_period PK int — Settlement period 1–48 (SP1 = 00:00–00:30 UTC). 49–50 on DST change days.
fuel_type PK str — Elexon fuel type code. See fuel types section below for full list.

generation_mw float — Average MW output for this fuel type in this period. Can be negative for pumped hydro / interconnector imports.

data_provider str — Always "elexon".

-- GB generation mix by fuel, last 7 days · hourly buckets
SELECT
  date_trunc('hour', timestamp_utc)  AS hour,
  fuel_type,
  round(avg(generation_mw), 1)        AS avg_mw
FROM silver.fuelhh
WHERE settlement_date >= current_date - interval '7 days'
  AND fuel_type NOT IN ('OTHER', 'OIL')
GROUP BY 1, 2
ORDER BY 1, avg_mw DESC;
import pandas as pd

df = pd.read_parquet(
    "data/silver/elexon/fuelhh/",
    filters=[("settlement_date", ">=", "2026-04-01")],
    columns=["settlement_date", "settlement_period", "fuel_type", "generation_mw"],
)
# pivot to wide format: one column per fuel type
mix = df.pivot_table(
    index=["settlement_date", "settlement_period"],
    columns="fuel_type",
    values="generation_mw",
    aggfunc="sum",
)
mix["TOTAL"] = mix.sum(axis=1)
Caveats

Things to know before using this.

These are specific to fuelhh. Also read the Elexon-wide caveats on the vendor page.

01
Embedded generation is excluded

FUELHH only covers BM-registered (metered) units. Sub-1 MW solar, small wind, and embedded CHP are not included — they appear as reduced demand, not as generation. Use NESO embedded estimates to add them back when calculating total GB generation.

02
WIND is onshore + offshore combined

The single WIND fuel type aggregates all metered wind regardless of technology. For an onshore/offshore split, use agpt (PSR type "Wind Onshore" / "Wind Offshore") or agws. Note those are D+1 settled, not live.

03
Interconnectors can be negative

When GB is exporting to France, generation_mw for INTFR will be negative. Sum all fuel types including interconnectors to get net national demand. When summing to total generation, decide whether to include or exclude interconnectors based on your use case.

04
DST clock-change days have SP 49–50

On the autumn clock change, the day has 50 settlement periods (25 hours). On spring forward, it has 46 (23 hours). The silver layer normalises all timestamps to UTC; the settlement_period column can reach 50 on DST days. Check for this when building daily aggregations.

05
publishDateTimeFrom lag is ~5–10 minutes

Elexon typically publishes data ~5 minutes after the end of the settlement period. The connector queries on a 30-minute schedule; median observed lag is 42 s from Gridflow's perspective because the query window overlaps the prior period. Occasionally rows arrive 10–15 minutes late.
