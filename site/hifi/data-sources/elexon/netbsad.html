-- Join netbsad to system_prices to see BSA contribution
SELECT
  sp.settlement_period,
  sp.system_buy_price,
  n.net_buy_price_adjustment,
  round(sp.system_buy_price - n.net_buy_price_adjustment, 2) AS implied_pre_bsa_sbp
FROM silver.system_prices sp
JOIN silver.netbsad n USING (settlement_date, settlement_period)
WHERE sp.settlement_date = '2026-05-01'
ORDER BY sp.settlement_period;
import pandas as pd

sp = pd.read_parquet("data/silver/elexon/system_prices/")

merged = sp.merge(nb, on=['settlement_date','settlement_period'])
merged['pre_bsa_sbp'] = merged['system_buy_price'] - merged['net_buy_price_adjustment']
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
from/to params required

Like disbsad, NETBSAD uses ?from=ISO&to=ISO. Using ?settlementDate= returns 404.

02
One row per period — safe to join directly

Unlike disbsad (multiple rows per period), netbsad has one row per settlement period. You can join directly to system_prices on (settlement_date, settlement_period).

03
Use disbsad for action-level detail

netbsad is the aggregate view. If you need to understand which specific actions drove the adjustment, use disbsad with the same period filter.
