-- Temperature anomaly vs demand correlation
SELECT
  t.timestamp_utc::date AS dt,
  round(t.temperature - t.normal_temperature, 1) AS temp_anomaly_c,
  i.initial_demand_outturn_mw
FROM silver.temp t
JOIN silver.indod i ON t.timestamp_utc::date = i.settlement_date
WHERE t.timestamp_utc >= '2026-01-01'
ORDER BY t.timestamp_utc;
import pandas as pd

temp = pd.read_parquet("data/silver/elexon/temp/")
indod = pd.read_parquet("data/silver/elexon/indod/")
temp['date'] = pd.to_datetime(temp.timestamp_utc).dt.date
indod['date'] = pd.to_datetime(indod.settlement_date)
merged = temp.merge(indod, on='date')
merged['temp_anomaly'] = merged.temperature - merged.normal_temperature
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Single GB reference temperature — not regional

The temp dataset provides one temperature value for all of GB, not regional breakdowns. For more granular spatial temperature data, use Open-Meteo weather data for individual locations.

02
Very low volume

At 30 rows/month, temp can be loaded entirely into memory for any analysis period. No filtering required.
