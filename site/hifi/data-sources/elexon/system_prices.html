-- Daily average SBP/SSP and spread, last 30 days
SELECT
  settlement_date,
  round(avg(system_buy_price), 2)  AS avg_sbp,
  round(avg(system_sell_price), 2) AS avg_ssp,
  round(avg(system_buy_price - system_sell_price), 2) AS avg_spread
FROM silver.system_prices
WHERE settlement_date >= current_date - interval '30 days'
GROUP BY 1 ORDER BY 1;
import pandas as pd

sp = pd.read_parquet(
    "data/silver/elexon/system_prices/",
    filters=[("settlement_date", ">=", "2026-04-01")],
)
sp['spread'] = sp['system_buy_price'] - sp['system_sell_price']
daily = sp.groupby('settlement_date')[['system_buy_price','system_sell_price','spread']].mean()
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Settlement runs: II is always first, DF is final

Elexon publishes multiple settlement runs per period. II (Interim Initial) appears within minutes; the final DF (Dispute Final) may not arrive for months. The silver layer keeps the highest-precedence run_type per period, overwriting earlier runs as they arrive.

02
Path-based API — not a query parameter

Unlike most BMRS endpoints, system_prices requires the date in the URL path: /balancing/settlement/system-prices/YYYY-MM-DD. Sending ?settlementDate= returns a 404. The connector handles this but be aware when testing manually.

03
SBP can be very high or negative in stressed periods

During supply stress SBP can spike to £4,000+/MWh. During oversupply SSP can go negative. The schema validates ge=-500 and le=10000 but real extremes occasionally breach these in stress events — check for outliers before regression analysis.
