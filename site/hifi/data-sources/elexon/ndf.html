-- Forecast vs actual demand error by period
SELECT
  n.settlement_period,
  n.national_demand_mw AS forecast_mw,
  i.initial_demand_outturn_mw AS actual_mw,
  round(i.initial_demand_outturn_mw - n.national_demand_mw, 0) AS error_mw
FROM silver.ndf n
JOIN silver.indo i USING (settlement_date, settlement_period)
WHERE n.settlement_date = '2026-05-01';
import pandas as pd

ndf = pd.read_parquet("data/silver/elexon/ndf/")
indo = pd.read_parquet("data/silver/elexon/indo/")
merged = ndf.merge(indo, on=['settlement_date','settlement_period'])
merged['error_mw'] = merged['initial_demand_outturn_mw'] - merged['national_demand_mw']
mae = merged['error_mw'].abs().mean()
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
One forecast published per day — not updated intraday

Unlike wind forecasts (windfor), NDF is published once per day for the following day and is not updated intraday. For intraday demand signals use INDO as it becomes available.

02
forecast_type column is always "day_ahead"

The forecast_type column distinguishes NDF from NDFD in the shared silver schema. For NDF it will always be "day_ahead".
