-- Total balancing cost per period, split SO vs commercial
SELECT
  settlement_period,
  round(sum(cost) FILTER (WHERE so_flag), 0)  AS so_cost_gbp,
  round(sum(cost) FILTER (WHERE NOT so_flag), 0) AS commercial_cost_gbp
FROM silver.disbsad
WHERE settlement_date = '2026-05-01'
GROUP BY 1 ORDER BY 1;

Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
from/to params required

DISBSAD uses ?from=ISO&to=ISO, not ?settlementDate=. The connector handles this correctly.

02
cost can be negative (clawback)

Negative cost values represent clawbacks from providers — situations where the SO recouped money. Do not filter out negative rows unless you specifically want payments only.

03
Multiple rows per period

Each period can have many adjustment actions. Always GROUP BY when joining to single-row-per-period datasets like system_prices.
