Overview

What this dataset is.

fuelinst is the instantaneous generation outturn by fuel type, published every ~5 minutes. Unlike fuelhh, these values are not settled — they are Elexon's best real-time estimate and can differ significantly from the final half-hourly figure.

It is sourced from /datasets/FUELINST using the publishDateTimeFrom / To pattern. The silver layer deduplicates on (timestamp_utc, fuel_type) and does not apply settlement-period alignment.

Use this dataset for dashboards and live monitoring. For any analytical work — trend analysis, regression inputs, market models — always use the settled fuelhh.

Schema

Silver layer columns.

Defined in gridflow/schemas/elexon.py · FuelInstTransformer. Partitioned by settlement_date.

Column Type Nullable Description
timestamp_utc PK datetime[UTC] — Snapshot timestamp in UTC. Derived from publishDateTime or startTime.
fuel_type PK str — Elexon fuel type code (CCGT, WIND, NUCLEAR, etc.). Same codes as fuelhh.
generation_mw float — Instantaneous generation estimate in MW. Unsettled — values differ from fuelhh.
data_provider str — Always "elexon".
ingested_at datetime[UTC] null When Gridflow wrote this row to bronze.

Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Not settled — do not use for analysis

Values reflect Elexon's real-time estimate before metering data is available. The settled equivalent is fuelhh. For any aggregation, trend, or model work, always prefer fuelhh.

02
Timestamp is publish time, not period start

Unlike fuelhh where timestamp_utc aligns to the settlement period boundary, fuelinst timestamps are when Elexon published the snapshot. They do not align to clean 30-minute boundaries.

03
High volume — filter carefully

At ~8.6M rows/month, fuelinst is 6x the volume of fuelhh. Always filter by date before loading into memory.
