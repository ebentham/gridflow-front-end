-- Total offer volume accepted by fuel type (via bmunits)
SELECT
  b.fuel_type,
  round(sum(abs(a.bid_offer_level_to - a.bid_offer_level_from)) / 2, 0) AS mw_accepted
FROM silver.boal a
JOIN silver.bmunits_reference b USING (bm_unit_id)
WHERE a.settlement_date = '2026-05-01'
  AND a.bid_offer_level_to > 0
  AND NOT a.so_flag
GROUP BY 1 ORDER BY 2 DESC;

Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Old BOAL endpoint decommissioned

The original /datasets/BOAL endpoint returns 404. Gridflow now sources from /datasets/BOALF which uses from/to params. If you have older code referencing the BOAL path, update it.

02
so_flag rows excluded from imbalance cost

Acceptances with so_flag=true are System Operator actions and are excluded from standard imbalance cost calculations. Always filter so_flag=false when computing commercial dispatch costs.

03
High volume — filter by date

At 1.1M rows/month this is one of the larger Elexon datasets. Always apply settlement_date filters before loading into memory.
