SELECT
  u.bm_unit_id, b.fuel_type, b.registered_capacity_mw
FROM silver.uou2t14d u
JOIN silver.bmunits_reference b USING (bm_unit_id)
WHERE u.settlement_date = '2026-05-07'
  AND u.settlement_period = 1
  AND u.output_usable_mw = 0
ORDER BY b.registered_capacity_mw DESC;
import pandas as pd

uou = pd.read_parquet("data/silver/elexon/uou2t14d/",
    filters=[("settlement_date", ">=", "2026-05-07")])

avail = joined.groupby(['settlement_date','fuel_type'])['output_usable_mw'].sum()
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
4-hour maximum API window — connector chunks automatically

Sending a 24-hour range to the UOU2T14D endpoint returns HTTP 400. The connector breaks each day into six 4-hour chunks. Do not query the API directly with >4h windows.

02
Highest volume Elexon dataset

At 5.2M rows/month, UOU2T14D is the largest dataset in the catalog. Always apply aggressive date and unit filters before loading. Consider materialising per-fuel aggregates using FOU2T14D where unit detail is not needed.

03
Zero output_usable_mw means unavailable

A zero value means the unit declared itself unavailable for that period. This is correct data, not a null or missing value. Filter zeros out when computing available capacity totals.
