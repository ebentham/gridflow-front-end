Overview

What this dataset is.

windfor contains NESO's wind generation forecasts for each GB settlement period. Two forecast columns are provided: initial_forecast_mw (published day-ahead, D-1) and latest_forecast_mw (updated hourly intraday as weather models improve).

The difference between initial and latest is a useful proxy for forecast uncertainty and weather model skill. Large gaps signal periods where wind was hard to predict, which often corresponds to price volatility.

Source: /datasets/WINDFOR with publishDateTimeFrom/To params. Deduplication is on (settlement_date, settlement_period).

-- Forecast error: latest vs fuelhh WIND actual
SELECT
  w.settlement_date, w.settlement_period,
  w.latest_forecast_mw,
  f.generation_mw AS actual_mw,
  round(w.latest_forecast_mw - f.generation_mw, 1) AS error_mw
FROM silver.windfor w
JOIN silver.fuelhh f
  ON w.settlement_date = f.settlement_date
  AND w.settlement_period = f.settlement_period
  AND f.fuel_type = 'WIND'
WHERE w.settlement_date >= current_date - interval '7 days';
import pandas as pd

wf = pd.read_parquet("data/silver/elexon/windfor/")
fh = pd.read_parquet("data/silver/elexon/fuelhh/")
fh_wind = fh[fh.fuel_type == 'WIND']

merged = wf.merge(fh_wind, on=['settlement_date', 'settlement_period'])
merged['error_mw'] = merged['latest_forecast_mw'] - merged['generation_mw']
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Combined wind only — no onshore/offshore split

windfor reports total GB wind generation without splitting onshore vs offshore. For that split use agws or agpt (settled D+1).

02
initial_forecast_mw can be null

For periods where NESO did not publish a day-ahead forecast, initial_forecast_mw will be null. latest_forecast_mw is almost always populated.

03
Forecast revisions overwrite on dedup

The silver layer keeps only the latest row per (settlement_date, settlement_period). Earlier forecast revisions are not retained. This means latest_forecast_mw is always the most recent published value.
