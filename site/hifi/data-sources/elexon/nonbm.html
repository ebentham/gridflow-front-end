Overview

What this dataset is.

nonbm records generation from STOR (Short Term Operating Reserve) providers that are not registered as Balancing Mechanism units. These are typically smaller embedded diesel generators, batteries, or demand-side providers contracted to provide reserve capacity.

The volumes are small — typically 50–200 MW across all providers — but nonbm is the only Elexon source capturing this generation. Excluding it from total GB generation calculations introduces a small but systematic undercount.

Source: /datasets/NONBM. Deduplication is on (settlement_date, settlement_period).

-- Add non-BM STOR to total generation
SELECT
  f.settlement_date, f.settlement_period,
  sum(f.generation_mw) AS bm_mw,
  max(n.generation_mw) AS nonbm_mw,
  sum(f.generation_mw) + max(n.generation_mw) AS total_mw
FROM silver.fuelhh f
LEFT JOIN silver.nonbm n
  USING (settlement_date, settlement_period)
WHERE f.settlement_date = '2026-05-01'
GROUP BY 1, 2 ORDER BY 2;
import pandas as pd

nbm = pd.read_parquet("data/silver/elexon/nonbm/")
fhh = pd.read_parquet("data/silver/elexon/fuelhh/")
total_bm = fhh.groupby(['settlement_date','settlement_period'])['generation_mw'].sum().reset_index()

merged = total_bm.merge(nbm, on=['settlement_date','settlement_period'], suffixes=('_bm','_nonbm'))
merged['total_mw'] = merged['generation_mw_bm'] + merged['generation_mw_nonbm']
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Very low volumes — check for zeros

Many settlement periods have zero non-BM STOR generation. This is correct — STOR is only called when the system operator needs reserve. Filter out zero rows before statistical analysis.

02
Not included in fuelhh totals

nonbm generation is separate from the Balancing Mechanism and does not appear in fuelhh. If you are summing fuelhh to get total GB generation, add nonbm separately.

03
D+1 lag

Like other settled datasets, nonbm data is only available the day after the settlement date. Not suitable for real-time generation tracking.
