-- Frequency events: periods below 49.8 Hz
SELECT
  timestamp_utc,
  frequency_hz,
  round(50.0 - frequency_hz, 3) AS deviation_hz
FROM silver.freq
WHERE frequency_hz < 49.8
  AND timestamp_utc >= now() - interval '30 days'
ORDER BY frequency_hz ASC LIMIT 50;
import pandas as pd

Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Very high volume — always filter by date

At 2.6M rows/month, freq is the largest time-series dataset by row count. For most analyses, resample to 1-minute or 30-minute averages. Loading raw 15-second data for more than a few days will be slow.

02
Timestamps are publish time, not exact measurement time

The timestamp_utc reflects when Elexon published the measurement, which lags the actual measurement by ~1 minute. For precise event timing, treat timestamps as approximate.

03
Validation range is 49.0–51.0 Hz

The Pydantic schema validates frequency_hz within [49.0, 51.0]. Values outside this range are rejected by the silver transformer. Real events approaching ±1.0 Hz deviation from nominal are extremely rare.
