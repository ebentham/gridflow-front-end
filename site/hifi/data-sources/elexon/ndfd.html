-- NDFD skill: forecast error by lead days
SELECT
  datediff('day', current_date, n.settlement_date) AS lead_days,
  round(avg(abs(i.initial_demand_outturn_mw - n.national_demand_mw)), 0) AS mae_mw
FROM silver.ndfd n
JOIN silver.indo i USING (settlement_date, settlement_period)
GROUP BY 1 ORDER BY 1;
import pandas as pd

merged = ndfd.merge(indo, on=['settlement_date','settlement_period'])
merged['error_mw'] = merged.initial_demand_outturn_mw - merged.national_demand_mw
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Silver layer keeps only latest forecast per target period

NDFD is published daily, so for a given target date (e.g. D+5) there will be multiple forecasts at different lead times. The silver layer deduplicates on (settlement_date, settlement_period, forecast_type) — only the most recently ingested forecast is retained.

02
D+2 minimum horizon

NDFD does not cover D+1 — use NDF for day-ahead. NDFD settlement_dates will always be at least 2 days after the publish date.
