-- Implied embedded generation: ITSDO - INDO
SELECT
  it.settlement_date, it.settlement_period,
  it.initial_transmission_system_demand_outturn_mw AS itsdo_mw,
  i.initial_demand_outturn_mw AS indo_mw,
  round(it.initial_transmission_system_demand_outturn_mw - i.initial_demand_outturn_mw, 0) AS implied_embedded_mw
FROM silver.itsdo it
JOIN silver.indo i USING (settlement_date, settlement_period)
WHERE it.settlement_date = '2026-05-01';
import pandas as pd

itsdo = pd.read_parquet("data/silver/elexon/itsdo/")
indo  = pd.read_parquet("data/silver/elexon/indo/")
merged = itsdo.merge(indo, on=['settlement_date','settlement_period'])
merged['embedded_est_mw'] = (merged.initial_transmission_system_demand_outturn_mw
                            - merged.initial_demand_outturn_mw)
Caveats

Things to know.

Dataset-specific notes. Also see Elexon-wide caveats.

01
Always higher than INDO

ITSDO > INDO for every period because it counts all load at the transmission boundary without netting embedded generation. If you see ITSDO < INDO, there is a data quality issue.

02
Not the same as ENTSO-E ATL

Although both measure transmission-level demand, ITSDO and ATL can differ due to different metering boundaries and settlement run timing. Do not assume they match exactly.
