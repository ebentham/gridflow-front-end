About this vendor

What Elexon publishes, and why.

Elexon administers the British Balancing and Settlement Code on behalf of NESO. Their public BMRS API is the canonical source for half-hourly settlement data on the GB grid.

Most datasets are settled to a defined timeline — a value published "live" can revise for several days as metered data replaces estimates. Gridflow's silver layer keeps revision history and exposes both the latest and as-of versions.

Caveats & gotchas
  1. 01
    Settlement revisions

    Half-hourly values can revise up to D+14. Gridflow stores all versions and exposes _revision_no.

  2. 02
    Embedded generation

    Sub-1 MW solar and wind isn't metered through BSC. Use NESO embedded estimates for headline figures.

  3. 03
    DST handling

    Settlement period 0 / 49–50 happen on clock-change days. Silver layer normalises to UTC.

  4. 04
    Pagination quirks

    Some endpoints cap responses at 30 days. Backfill loops chunk windows automatically.
