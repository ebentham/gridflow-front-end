Architecture

Bronze · silver · gold.

Raw API responses land in bronze. Cleaned, typed, deduped tables live in silver. Joined, analysis-ready views in gold.

Everything is Hive-partitioned Parquet, queryable from DuckDB or the Python client. Backfills are idempotent. Schema changes are caught at parse time by Pydantic v2 contracts. Bitemporal — every row carries both event-time and ingestion-time so point-in-time queries are trivial.

The warehouse exists to support quantitative work. Probabilistic estimators live in gridflow-models, a sibling repository — one LightGBM model per quantile, walk-forward validated, artifact-pinned to the data fingerprint that trained them.

Day-ahead demand · GB

LightGBM quantile regression on national demand. Calendar + lag-feature pipeline; one model per quantile; six-fold expanding window. Currently the worked example — see the case study.

Query it

SQL or Python — pick one.

All examples use the same canonical silver schema.
-- GB residual demand last 30 days, hourly
SELECT
  date_trunc('hour', settlement_dt) AS hour,
  sum(case when fuel_type = 'WIND'  then mw end) AS wind_mw,
  sum(case when fuel_type = 'NPSHYD' then mw end) AS hydro_mw,
  sum(mw) AS total_mw
FROM silver.fuelhh
WHERE settlement_dt >= now() - interval '30 days'
GROUP BY 1
ORDER BY 1;
from gridflow import GridflowClient

with GridflowClient() as gf:
    df = gf.get_fuel_generation(
        start="2026-04-01",
        end="2026-04-29",
    )
    # pivot to one column per fuel type
    pivot = df.pivot(
        values="generation_mw",
        index="timestamp_utc",
        columns="fuel_type",
    )
import pandas as pd

df = pd.read_parquet(
  "data/silver/elexon/fuelhh/year=2026/",
  columns=["timestamp_utc", "fuel_type", "generation_mw"],
)
df = df[df.timestamp_utc >= pd.Timestamp.now() - pd.Timedelta("30d")]
About

E. Bentham, here.

Based in London · UK

I'm a quant researcher working on UK and European power markets. Gridflow is the data and modelling backbone behind everything I study — built locally, in the open, around the same patterns I'd want at a desk.

I'm interested in fundamentals-driven price formation, the transition away from CCGT marginal pricing, and what probabilistic forecasts mean for short-horizon trading and scheduling decisions. The catalogue captures the data; the modelling stack captures the assumptions. Both are public so the caveats are honest.
