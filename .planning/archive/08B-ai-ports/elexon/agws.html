Overview

What this dataset is.

agws holds Actual or Estimated Wind and Solar Power Generation — wind and solar MW output per settlement period. AGWS (ENTSO-E B1630) uses the same PSR taxonomy as AGPT but is restricted to renewable categories: Wind Onshore, Wind Offshore, and Solar.

It is sourced from the BMRS Insights API at /datasets/AGWS, fetched by publish-time window. The raw JSON lands in bronze and is validated via AGWSTransformer into the silver parquet layer.

As the ENTSO-E-compliant renewable series, AGWS is the primary source for wind/solar attribution within GB generation analysis. The data typically arrives within 1 day of settlement. Verified against vendor docs on 2026-05-08.

Schema

Silver layer columns.

Defined in gridflow/silver/elexon/agws.py · AGWSTransformer. The silver table is partitioned by settlement_date. Dedup key: (settlement_date, settlement_period, psr_type).

Column Type Nullable Description
settlement_date PK date — Settlement date (BST/GMT calendar). Partition key.
settlement_period PK int — 1..50 (DST: 46 spring, 50 autumn).
psr_type PK str — ENTSO-E PSR type (Wind Onshore, Wind Offshore, Solar).
timestamp_utc datetime[UTC] — Derived from (settlement_date, settlement_period) via utils/time.settlement_period_to_utc.
generation_mw float — MW output for this PSR type in this settlement period.
business_type str null ENTSO-E business type (e.g., Wind generation).
document_id str null ENTSO-E document MRID.
document_revision int null Document revision number. Same period+psr_type may be re-issued; latest revision wins.
data_provider str — Default "elexon".
ingested_at datetime[UTC] null Time ingested into bronze. Useful for pipeline debugging.

Wind/solar split

Renewable PSR types in AGWS.

AGWS covers only the renewable subset of the ENTSO-E PSR taxonomy. Three types are present in GB data.

Wind Offshore— largest contributor, GB offshore fleet
Wind Onshore— onshore fleet, smaller than offshore
Solar— zero at night; peaks midday in summer

       SUM(generation_mw) AS total_mw
FROM read_parquet('data/silver/elexon/agws/**/*.parquet')
WHERE settlement_date >= current_date - INTERVAL 7 DAY
GROUP BY settlement_date, psr_type
ORDER BY settlement_date, psr_type;

Caveats

Things to know before using this.

These are specific to agws. Also read the Elexon-wide caveats on the vendor page.

01
Same gotchas as AGPT — limited to wind/solar PSR types

AGWS shares the same B-series (B1630) ENTSO-E lineage as AGPT. Document revision logic applies identically: same (settlement_date, settlement_period, psr_type) can be re-issued; latest revision wins in silver. For historical revision analysis, query bronze directly.

02
No Pydantic schema in schemas/elexon.py

The silver transformer enforces the ENTSO-E PSR-typed output shape directly, without a dedicated Pydantic model in gridflow/schemas/elexon.py. Schema validation is applied at the transformer layer. A future schemas entry may be added; check the transformer for the authoritative column list.
