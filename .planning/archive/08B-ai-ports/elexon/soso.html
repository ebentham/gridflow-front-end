Overview

SOSO records trades between GB National Grid ESO and neighbouring TSOs across all active interconnectors. Each row carries the contract identifier, sending and receiving TSO, trade direction (bid/offer), quantity in MW, price in GBP/MWh, and the delivery window start and end times. All GB interconnectors — IFA, IFA2, BritNed, Moyle, NSL, ElecLink, NEMO, Greenlink, and Viking — appear in the dataset.

The silver schema carries 15 columns. No Pydantic model exists — the SOSOTransformer enforces shape via Polars casting. Filter on senderIdentification or receiverIdentification to isolate a specific interconnector; a gold view is not yet implemented.

SOSO prices are a key input for cross-border arbitrage analysis and for benchmarking implicit allocation prices against day-ahead power prices.

       trade_direction, trade_quantity_mw, trade_price, trader_unit

        .sort("trade_price", descending=True))

Caveats

01
Max 23-hour query window — same vendor cap as REMIT

The Elexon API enforces a hard cap at approximately 24 hours. The connector uses max_chunk_hours=23 — the same fix applied to REMIT (V2-FIX-03, verified live 2026-05-09). Wider queries return HTTP 400.

02
All GB interconnectors in one table — filter by sender/receiver

SOSO covers all GB interconnectors in a single dataset. To isolate a specific interconnector, filter on sender_identification or receiver_identification using the TSO EIC codes. A gold-layer interconnector view has not yet been implemented.
