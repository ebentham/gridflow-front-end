Overview

What this dataset is.

atl holds Actual Total Load Per Bidding Zone — the realised total load for the GB bidding zone per settlement period. ATL (ENTSO-E B0610) is the B-series load series that the EU transparency platform consumes for the GB row of pan-European demand reporting.

It is sourced from the BMRS Insights API at /datasets/ATL, fetched by publish-time window. The raw JSON lands in bronze and is processed via ATLTransformer into the silver parquet layer.

ATL provides a single MW figure per settlement period — total load without PSR breakdown. The data arrives within 1 day of settlement. Verified against vendor docs on 2026-05-08.

7 reconciliation passes.

Like other B-series datasets, ATL is re-published as BSC settlement reconciles. Document revisions reflect updated load estimates across the standard settlement run sequence.

II— Initial Indicative · within hours
SF— Settlement Final · settlement day
R1— Reconciliation 1 · ~10 days
R2— Reconciliation 2 · ~1 month
R3— Reconciliation 3 · ~4 months
RF— Reconciliation Final · ~14 months

-- Daily average and peak load
SELECT settlement_date,
       AVG(total_load_mw) AS avg_mw,
       MAX(total_load_mw) AS peak_mw
FROM read_parquet('data/silver/elexon/atl/**/*.parquet')
WHERE settlement_date >= current_date - INTERVAL 30 DAY
GROUP BY settlement_date
ORDER BY settlement_date;
import polars as pl

df = pl.read_parquet(
    "data/silver/elexon/atl/**/*.parquet",
)
# daily peak and trough load
daily = df.group_by("settlement_date").agg(
    pl.col("total_load_mw").max().alias("peak_mw"),
    pl.col("total_load_mw").min().alias("trough_mw"),
    pl.col("total_load_mw").mean().alias("mean_mw"),
)
print(daily.sort("settlement_date").tail())
Caveats

Things to know before using this.

These are specific to atl. Also read the Elexon-wide caveats on the vendor page.

01
Total load only — no PSR split

total_load_mw is a single aggregated figure per period. There is no technology breakdown in ATL — use agpt for PSR-type generation and build implied load from generation + interconnector flow if a more granular picture is needed.

02
No Pydantic schema in schemas/elexon.py

ATL shares the same B-series ENTSO-E lineage as AGPT (B0610). Schema validation is applied directly in the transformer; no Pydantic model exists in gridflow/schemas/elexon.py.

03
Document revisions update the same key

The same (settlement_date, settlement_period) may be re-issued with a higher document_revision as the System Operator refines load estimates. Silver keeps the latest revision; earlier revisions are overwritten. For historical revision analysis, query bronze directly using ingested_at.
