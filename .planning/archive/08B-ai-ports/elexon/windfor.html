Overview

WINDFOR publishes the GB wind generation forecast per settlement period. Each row carries two forecast values: initial_forecast_mw (the first-issued forecast for that period) and latest_forecast_mw (the most recently revised forecast). The difference between the two quantifies intra-day forecast revision — a key signal for wind variability analysis.

WINDFOR uses the ElexonWindForecast Pydantic schema and WindForecastTransformer. Unlike most Elexon datasets, settlement_date and settlement_period are nullable in the schema — the API response does not always carry them. The point-in-time field is published_at rather than ingested_at.

WINDFOR is the canonical benchmark for actual wind outturn: comparing WINDFOR latest_forecast_mw against FUELHH wind output quantifies wind forecast error per settlement period — a standard metric for renewable integration analysis.

SELECT settlement_date, settlement_period,
       initial_forecast_mw, latest_forecast_mw,
       latest_forecast_mw - initial_forecast_mw AS revision_mw
FROM read_parquet('data/silver/elexon/windfor/**/*.parquet')
WHERE settlement_date = '2026-05-05'
ORDER BY settlement_period;
import polars as pl

df = pl.read_parquet("data/silver/elexon/windfor/**/*.parquet")
print(df.with_columns(
    (pl.col("latest_forecast_mw") - pl.col("initial_forecast_mw"))
    .alias("revision_mw")
).filter(pl.col("settlement_date") == pl.date(2026, 5, 5))
 .sort("settlement_period"))

Caveats

01
Sparse publication — settlement_date and settlement_period are nullable

The API response does not always carry settlementDate or settlementPeriod. The ElexonWindForecast Pydantic schema marks both as nullable. Always check for nulls before joining to settlement-period datasets. Queries within 3-hour windows will typically return zero rows — use at least a 1-day window.

02
initial_forecast_mw vs latest_forecast_mw — use latest for outturn comparison

initial_forecast_mw is the first-published forecast for a period. latest_forecast_mw is the most recently revised value. For outturn comparison with FUELHH, use latest_forecast_mw. The difference quantifies intra-day forecast revision magnitude.
