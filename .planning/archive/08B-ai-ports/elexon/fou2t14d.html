Overview

What this dataset is.

fou2t14d holds 2–14 day-ahead generation availability aggregated by fuel type — a daily forward view of usable MW capacity for each fuel category across the next two weeks. The dataset is published once daily and covers each future delivery date separately.

It is sourced from the BMRS Insights API at /datasets/FOU2T14D and processed by FOU2T14DTransformer. Silver files are append-only (revision-preserving).

FOU2T14D is used for medium-term margin and capacity-factor projections. The settlement_date in silver is the future delivery date, not the publish date. Verified against vendor docs on 2026-05-08.

-- Forward availability by fuel for next 14 days
SELECT settlement_date, fuel_type,
       output_usable_mw
FROM read_parquet('data/silver/elexon/fou2t14d/**/*.parquet')
WHERE settlement_date BETWEEN current_date AND current_date + INTERVAL 14 DAY
ORDER BY settlement_date, fuel_type;
import polars as pl
from datetime import date, timedelta

df = pl.read_parquet(
    "data/silver/elexon/fou2t14d/**/*.parquet",
)
today = date.today()
fwd = df.filter(
    pl.col("settlement_date").is_between(today, today + timedelta(days=14))
).pivot(
    values="output_usable_mw",
    index="settlement_date",
    columns="fuel_type",
    aggregate_function="sum",
)
print(fwd.sort("settlement_date"))
Caveats

Things to know before using this.

These are specific to fou2t14d. Also read the Elexon-wide caveats on the vendor page.

01
settlement_date is the future delivery date, not the publish date

The API uses forecastDate for the future delivery date; silver renames this to settlement_date to match the canonical schema column. Be careful not to confuse the delivery date with the publish date when filtering — use ingested_at to identify which publish produced a row.

02
Silver files are append-only (revision-preserving)

Unlike most Elexon datasets, FOU2T14D silver uses APPEND_ONLY = True. Each daily publish appends a new revision rather than overwriting. To get the latest forecast for a delivery date, filter to the most recent ingested_at for that date.
