Overview

What this dataset is.

agpt holds Actual Aggregated Generation Per Type — every PSR (Production-Storage Resource) type's aggregated MW output per settlement period. AGPT is the GB equivalent of the ENTSO-E B1620 series and feeds the GB row of pan-European generation transparency dashboards.

It is sourced from the BMRS Insights API at /datasets/AGPT, fetched by publish-time window. The raw JSON lands in bronze and is validated via AGPTTransformer into the silver parquet layer.

With one row per (settlement_date, settlement_period, psr_type), AGPT is the primary signal for technology-level dispatch analysis. The data typically arrives within 1 day of settlement. Verified against vendor docs on 2026-05-08.

Schema

Silver layer columns.

Defined in gridflow/silver/elexon/agpt.py · AGPTTransformer. The silver table is partitioned by settlement_date. Dedup key: (settlement_date, settlement_period, psr_type).

Column Type Nullable Description
settlement_date PK date — Settlement date (BST/GMT calendar). Partition key.
settlement_period PK int — 1..50 (DST: 46 spring, 50 autumn).
psr_type PK str — ENTSO-E PSR type code (e.g., Biomass, Wind Onshore, Solar).
timestamp_utc datetime[UTC] — Derived from (settlement_date, settlement_period) via utils/time.settlement_period_to_utc.
generation_mw float — MW output for this PSR type in this settlement period.
business_type str null ENTSO-E business type (e.g., Production).
document_id str null ENTSO-E document MRID (e.g., NGET-EMFIP-AGPT-06476951).
document_revision int null Document revision number. Same period+psr_type may be re-issued; latest revision wins.
data_provider str — Default "elexon".
ingested_at datetime[UTC] null Time ingested into bronze. Useful for pipeline debugging.

SELECT settlement_date, psr_type,
       SUM(generation_mw) AS total_mw
FROM read_parquet('data/silver/elexon/agpt/**/*.parquet')
WHERE settlement_date = '2026-05-06'
GROUP BY settlement_date, psr_type
ORDER BY total_mw DESC;

    values="generation_mw",
    index=["settlement_date", "settlement_period"],
    columns="psr_type",
    aggregate_function="sum",
).sort(["settlement_date", "settlement_period"])
print(pivot.tail())
Caveats

Things to know before using this.

These are specific to agpt. Also read the Elexon-wide caveats on the vendor page.

01
PSR codes follow ENTSO-E codelist (B01–B25)

Silver preserves the raw ENTSO-E PSR type strings verbatim — map to friendly names (e.g., Wind Offshore → "Offshore Wind") in a gold layer. The codelist is managed by ENTSO-E and may be extended; do not hard-code a fixed set of types.

02
Same period/type can be re-issued with a higher document revision

The same (settlement_date, settlement_period, psr_type) may arrive with an incremented document_revision as the System Operator refines its estimate. The transformer keeps the latest revision; earlier revisions are overwritten in silver. For historical revision analysis, query bronze directly.
