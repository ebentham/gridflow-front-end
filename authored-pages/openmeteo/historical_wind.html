Overview

What this dataset is.

historical_wind holds hourly ERA5-reanalysis wind weather — speed and direction at 10 m and 100 m, gusts, temperature, pressure, cloud and dew point — at twelve capacity-weighted GB sites (8 offshore + 4 onshore), reaching back to 1940. It is the archive half of Gridflow's wind-weather pair. Unlike forecast_wind, the ERA5 archive carries only the 10 m and 100 m heights — the deeper hub heights return all-null and are excluded. Each row is one hour at one GB site.

It is sourced from Open-Meteo's ERA5 archive endpoint at archive-api.open-meteo.com/v1/archive, queried once per site over a date range, validated against WindWeather (schemas/weather.py), and written to silver via HistoricalWindWeather. The payload is JSON columnar — parallel hourly arrays the transformer zips into one row per timestamp, applying the km/h→m/s conversion and the F15-B unit-suffix rename.

Multi-decade hub-height wind is the calibration backbone of the wind-generation model — ERA5 supplies the long 100 m history that fixes power-curve and Hellmann-shear features, validated downstream against realised generation from Elexon fuelhh (WIND) and against the NESO windfor public forecast over multi-year backtests. The forward sibling forecast_wind supplies the day-ahead nowcast and the deeper hub heights. Verified against vendor docs: 2026-05-09 · Open-Meteo /archive.

-- Monthly capacity-factor-style 100m wind statistic per site (multi-year backtest)
SELECT date_trunc('month', timestamp_utc) AS month,
       location,
       avg(wind_speed_100m_mps)                       AS mean_100m_mps,
       quantile_cont(wind_speed_100m_mps, 0.50)       AS p50_100m_mps,
       avg(wind_speed_100m_mps / wind_speed_10m_mps)  AS shear_ratio
FROM read_parquet('data/silver/open_meteo/historical_wind/**/*.parquet')
WHERE timestamp_utc >= TIMESTAMP '2020-01-01'
  AND wind_speed_10m_mps > 0.5
GROUP BY 1, 2
ORDER BY 1, 2;
import polars as pl

df = pl.read_parquet("data/silver/open_meteo/historical_wind/**/*.parquet")
# Hellmann shear exponent: ln(v100/v10) / ln(100/10) - hub-height extrapolation feature
shear = (
    df.filter((pl.col("wind_speed_10m_mps") > 0.5)
            & (pl.col("wind_speed_100m_mps") > 0.5))
      .with_columns(
          (pl.col("wind_speed_100m_mps") / pl.col("wind_speed_10m_mps")).log()
          .truediv(pl.lit(10.0).log())
          .alias("alpha_hellmann")
      )
      .group_by("location")
      .agg(pl.col("alpha_hellmann").mean().alias("mean_alpha"))
)
print(shear.sort("mean_alpha"))
Caveats

Things to know before using this.

These are specific to historical_wind. Also read the Open-Meteo-wide caveats on the vendor page.

01
ERA5 archive omits hub heights 80 / 120 / 180 m

The connector excludes those heights from WIND_ARCHIVE_VARS — verified 2026-05-09 against Hornsea and Whitelee: ERA5 returns units: "undefined" and all-null. The WindWeather schema declares them for forecast-symmetry; silver output does not carry them here. Use forecast_wind for the wider hub-height set. vault historical_wind.md §Archive 10m+100m · endpoints.py L109-125

02
DISCREPANCY · vault schema vs silver
Silver column names are unit-suffixed (F15-B)

Silver emits wind_speed_10m_mps, wind_speed_100m_mps, wind_direction_*_deg, wind_gusts_10m_mps with km/h→m/s ÷3.6 — the vault Silver Schema tables list the pre-rename names and are stale. Query the parquet with the suffixed forms. silver/openmeteo/historical.py L101-134 · _output_columns L255-271

01
ERA5 publication lag is ~5 days

Calls for end_date within the last 5 days may return 200 with null trailing values — ERA5 reanalysis cadence is non-real-time by design. Plan backfills accordingly; for very recent past use forecast_wind with past_days (≤92). vault Known Issues "ERA5 reanalysis lag"

04
Approximate site centroids (capacity-weighted, not NRO)

The 12 wind locations are capacity-weighted approximations per ADR-020, not per-turbine NRO coordinates. Sub-cluster resolution requires reading the ADR-020 trade-offs (vendor licensing + ergonomics + modelling granularity). docs/DECISION_LOG/ADR-020-openmeteo-location-approximation.md

05
ERA5 grid-cell snapping

The latitude / longitude echoed in the response can differ slightly from the request — ERA5 snaps to the nearest grid cell (~0.25° resolution). Silver stores the response values, not the request values; don't dedup on lat/lon. vault Known Issues "ERA5 grid-cell snapping"

06
Bronze path uses DOUBLE underscore separator

Bronze partitions are bronze/open_meteo/historical_wind__<site>/<year>/… — a double underscore between dataset key and site. Silver BRONZE_DATASET_PREFIX = "historical_wind" strips it on read; custom bronze-readers must match the __ form or list zero files. connectors/openmeteo/client.py L112 · silver/openmeteo/historical.py L142
