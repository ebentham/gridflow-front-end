Open-Meteo Weather.

An open-source weather API aggregating ECMWF, GFS, and ERA5 reanalysis into a single columnar JSON interface. No authentication required for non-commercial use. Six datasets covering hourly temperature, wind, and solar irradiance across GB population centres and capacity-weighted generation sites — 1–16-day forecasts and ERA5-backed archive to 1940.

Caveats · known quirks

What to watch for.

01

Dual base-URL architecture — not a versioning split

Forecast datasets resolve to api.open-meteo.com; the historical datasets (ERA5 reanalysis) resolve to archive-api.open-meteo.com. The connector routes per dataset.startswith("historical") and both hosts return the same JSON columnar shape, so this is a deliberate API design choice rather than a version difference.

Source · connectors/openmeteo/client.py L90–94 · endpoints.py ARCHIVE_BASE_URL / FORECAST_BASE_URL
02

ERA5 archive runs ~5 days behind, with null hub-height wind

The historical_* datasets draw on ERA5 reanalysis, published roughly five days in arrears — a request whose end_date falls inside that window returns HTTP 200 with null trailing values. ERA5 also omits the 80/120/180 m hub-height wind the forecast model carries, so historical wind must extrapolate from 10 m. For the most recent past, forecast_demand with past_days (≤92) is the bridge.

Source · vault historical_demand.md · historical_wind.md · schemas/weather.py WindWeather
03

Schemas live in weather.py, and the vendor name shifts by layer

The Pydantic classes — DemandWeather, WindWeather, SolarWeather — sit in schemas/weather.py, named for the domain rather than the vendor; there is no schemas/openmeteo.py. The name itself is inconsistent across the stack: vault directory open-meteo, module and slug openmeteo, registry path open_meteo, live host api.open-meteo.com. Transformers split by horizon across silver/openmeteo/forecast.py and historical.py.

Source · schemas/weather.py · silver/openmeteo/forecast.py · silver/openmeteo/historical.py
04

Public access, but data is queried per site — not nationally

Unlike GIE, Open-Meteo needs no key for non-commercial use. Coverage is point-based: 7 demand population centres, 12 capacity-weighted wind sites, and 6 solar sites — 25 named GB locations in total. The wind points are capacity-weighted approximations (ADR-020), not per-turbine coordinates, and solar irradiance is global tilted (GTI) on a fixed UK panel geometry (tilt 35°, azimuth 180°) — plane-of-array, not horizontal.
