Overview

What this dataset is.

forecast_solar holds the hourly solar-irradiance forecast — the full decomposition of global, direct-normal, diffuse and tilted irradiance (GHI / DNI / DHI / GTI) plus cloud cover and snow context — looking 1–16 days ahead at six GW-capacity-weighted GB solar sites. It is the forward-looking half of Gridflow's solar-weather pair, drawn from Open-Meteo's NWP router rather than the ERA5 archive, with GTI computed for the UK fixed-tilt geometry (tilt=35°, azimuth=180°). Each row is one hour at one GB site.

It is sourced from Open-Meteo's forecast endpoint at api.open-meteo.com/v1/forecast, queried once per site, validated against SolarWeather (schemas/weather.py), and written to silver via ForecastSolarWeather. The payload is JSON columnar — parallel hourly arrays (hourly.time[], hourly.global_tilted_irradiance[], …) that the transformer zips into one row per timestamp before the F15-B unit-suffix rename.

GTI is the most direct weather driver of UK fixed-tilt PV output, and this feed is the day-ahead input to the PV-generation model: forecast irradiance here pairs with realised generation from Elexon fuelhh (the SOLAR code) to score nowcast skill, while the ERA5 sibling historical_solar supplies the multi-decade backtest. Schema

Silver layer columns.

Defined in gridflow/schemas/weather.py · SolarWeather (subclass of _BaseWeather) and materialised by gridflow/silver/openmeteo/forecast.py · ForecastSolarWeather. Silver parquet uses F15-B canonical (unit-suffixed) names — vault tables predate the rename. Partitioned by timestamp_utc (year + month). Point-in-time field: available_at (from BaseSilverTransformer); no forecast_run_at. No air_density_kg_m3 — the solar request omits surface_pressure.

-- Daily peak GTI per GB solar site (illustrative PV forecast)
SELECT location,
       date_trunc('day', timestamp_utc) AS day,
       max(global_tilted_irradiance_wm2) AS peak_gti_wm2,
       avg(cloud_cover_pct)              AS avg_cloud_pct
FROM read_parquet('data/silver/open_meteo/forecast_solar/**/*.parquet')
WHERE timestamp_utc >= current_date
  AND timestamp_utc <  current_date + INTERVAL 7 DAY
GROUP BY 1, 2
ORDER BY 2, 1;
import polars as pl

df = pl.read_parquet("data/silver/open_meteo/forecast_solar/**/*.parquet")
# GB-aggregate hourly GTI (equal-weight mean across 6 sites for illustration)
gb = (
    df.group_by("timestamp_utc")
      .agg(
          pl.col("global_tilted_irradiance_wm2").mean().alias("gb_avg_gti_wm2"),
          pl.col("cloud_cover_pct").mean().alias("gb_avg_cloud_pct"),
      )
      .sort("timestamp_utc")
)
print(gb.tail(24))
Caveats

Things to know before using this.

These are specific to forecast_solar. Also read the Open-Meteo-wide caveats on the vendor page.

01
No forecast_run_at — cross-vintage backtests need the archive

Dedup keeps the last silver row per (timestamp_utc, location), overwriting prior forecast vintages on re-ingest. Vintage-vs-vintage skill analysis is not possible here; pair against historical_solar at matching lead times. vault Known Issues · silver/openmeteo/historical.py L217

02
DISCREPANCY · vault schema vs silver
Silver column names are unit-suffixed (F15-B)

Silver emits shortwave_radiation_wm2, direct_normal_irradiance_wm2, global_tilted_irradiance_wm2, temperature_2m_c, cloud_cover_pct, etc. — the vault Silver Schema tables list the pre-rename names and are stale. Query the parquet with the suffixed forms. silver/openmeteo/historical.py L101-134 · _output_columns L255-271

03
GTI requires tilt and azimuth (UK fixed-tilt 35°/180°)

The connector injects tilt=35&azimuth=180 via WeatherDatasetSpec.extra_params for the UK representative geometry (due south). Tracker installations underestimate at this fixed tilt — use DNI plus a tracker-specific transposition model. connectors/openmeteo/endpoints.py L157-160

04
No air_density_kg_m3 on this dataset (unlike forecast_wind)

The solar variable list omits surface_pressure, so BaseOpenMeteoTransformer.DERIVE_AIR_DENSITY cannot run and the column is absent. Property test tests/unit/test_openmeteo_air_density.py locks the contract. silver/openmeteo/historical.py L241-252 · test file

05
No NWP-model pinning — irradiance skill varies by model

Open-Meteo's /forecast router selects an NWP model per request; irradiance forecast skill differs materially between ECMWF / GFS / ICON. Pin via &models=<id> for reproducibility. vault forecast_solar.md Known Issues "Unpinned NWP model"

06
Bronze path uses DOUBLE underscore separator

Bronze partitions are bronze/open_meteo/forecast_solar__<site>/<year>/… — a double underscore between dataset key and site. Silver BRONZE_DATASET_PREFIX = "forecast_solar" strips it on read; custom bronze-readers must match the __ form or list zero files. connectors/openmeteo/client.py L112 · silver/openmeteo/historical.py L142
