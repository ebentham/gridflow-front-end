Overview

What this dataset is.

disbsad holds Disaggregated Balancing Services Adjustment Data — the constituent BSAD components used to derive Net BSAD. Each row captures the cost (GBP) and volume (MWh) of a single non-BM balancing action such as a STOR call-off, ancillary service instruction, or SO action outside of BOALF.

It is sourced from the BMRS Insights API at /datasets/DISBSAD and processed by DISBSADTransformer.

Together with NETBSAD, DISBSAD is what BSC parties consume to reconcile imbalance settlement. Published with a 1-day lag. Verified against vendor docs on 2026-05-08.

-- Daily total BSAD cost and volume
SELECT settlement_date,
       SUM(cost)   AS total_cost_gbp,
       SUM(volume) AS total_volume_mwh
FROM read_parquet('data/silver/elexon/disbsad/**/*.parquet')
WHERE settlement_date >= current_date - INTERVAL 30 DAY
GROUP BY settlement_date
ORDER BY settlement_date;
import polars as pl

df = pl.read_parquet(
    "data/silver/elexon/disbsad/**/*.parquet",
)
# STOR actions only — cost per settlement period
stor = df.filter(pl.col("stor_flag"))
daily = stor.group_by("settlement_date").agg(
    pl.col("cost").sum().alias("stor_cost_gbp"),
    pl.col("volume").sum().alias("stor_volume_mwh"),
)
print(daily.sort("settlement_date").tail())
Caveats

Things to know before using this.

These are specific to disbsad. Also read the Elexon-wide caveats on the vendor page.

01
DISBSAD pairs with NETBSAD — don't double-count

netbsad is the net aggregate of all DISBSAD components for the same period. When joining the two datasets, do not sum DISBSAD volumes on top of NETBSAD volumes — they represent the same underlying actions at different aggregation levels.

02
cost is GBP total, not GBP/MWh

The cost field is the total GBP cost of the balancing action, not a per-MWh price. Do not divide cost by volume without careful handling of zero-volume rows (they appear in off-peak periods where the system is balanced without any non-BM action).
