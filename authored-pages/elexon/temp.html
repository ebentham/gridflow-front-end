Overview

TEMP publishes one record per measurement date carrying the realised GB ambient temperature (Celsius) alongside three seasonal-reference values: normal, low, and high. These reference values are climatological normals — not forecasts — representing the expected seasonal envelope for that date of year.

NESO uses TEMP in its demand-forecast calibration pipeline. Temperature deviations from seasonal normal are the primary driver of short-term demand forecast error. TEMP is the standard meteorological covariate for any regression or time-series model of GB electricity demand.

The silver schema has 7 columns with timestamp_utc as the dedup key. No Pydantic model exists. Note: the API field measurementDate is present in the bronze response but is not currently mapped to a dedicated silver column — the connector derives timestamp_utc from publishTime.

Caveats

01
Daily resolution — empty within sub-day query windows

TEMP publishes once per day. Queries using a 3-hour window will typically return zero rows. Use at least a 1-day window (publishDateTimeTo − publishDateTimeFrom ≥ 24h) to reliably retrieve data.

02
Reference temperatures are climatological normals — not forecasts

normal_temperature, low_temperature, and high_temperature are fixed seasonal climatology values for that date of year. They are not intra-day or short-term weather forecasts and should not be used as such.

03
measurementDate not mapped to a silver column

The API response carries a measurementDate field that is not currently mapped to the silver schema. timestamp_utc is derived from publishTime, which may differ slightly from the measurement date. An implementation delta exists to add an explicit measurement_date column.
