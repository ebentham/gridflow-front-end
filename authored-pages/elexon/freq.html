Overview

What this dataset is.

freq is the second-by-second GB system frequency — the high-resolution telemetry for frequency-response models, RoCoF estimation, and ancillary-services validation. Statutory band is 49.5–50.5 Hz; the schema admits 49.0–51.0 to preserve genuine events.

Gridflow fetches it from /datasets/FREQ using the measurementDateTimeFrom / measurementDateTimeTo pattern (post V2-FIX-01 — earlier bronze captured "latest 5761 samples" from the buggy publish-window). The raw JSON lands in bronze, is validated against ElexonFrequency, and written to silver via FreqTransformer.

Published hourly with ~1 min lag (sample interval ~2 s). Verified against vendor docs on 2026-05-09.

-- Time spent outside the statutory operating range (49.5–50.5 Hz) per day
SELECT date_trunc('day', timestamp_utc) AS day,
       SUM(CASE WHEN frequency_hz < 49.5 OR frequency_hz > 50.5
                THEN 1 ELSE 0 END) AS n_excursion_samples,
       MIN(frequency_hz) AS min_hz,
       MAX(frequency_hz) AS max_hz
FROM read_parquet('data/silver/elexon/freq/**/*.parquet')
WHERE timestamp_utc >= current_date - INTERVAL 30 DAY
GROUP BY 1
ORDER BY 1;
import polars as pl

df = pl.read_parquet("data/silver/elexon/freq/**/*.parquet")
# Rolling 1-minute mean + max deviation from 50 Hz
roll = (
    df.sort("timestamp_utc")
      .with_columns(
          pl.col("frequency_hz").rolling_mean(window_size=30).alias("freq_60s"),
          (pl.col("frequency_hz") - 50.0).abs()
            .rolling_max(window_size=30).alias("dev_60s"),
      )
)
print(roll.tail(20))
