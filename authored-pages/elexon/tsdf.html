Overview

TSDF publishes a transmission-system-only demand forecast per settlement period. Unlike NDF which covers total national demand (including embedded generation), TSDF excludes embedded generation — making it directly analogous to the ITSDO outturn series for forecast purposes.

The silver schema carries 7 columns including a boundary field (N = national, Z = zonal) and forecast_demand_mw. No Pydantic model exists — the TSDFTransformer enforces shape via Polars casting. The relationship NDF − TSDF ≈ embedded generation forecast mirrors the INDO − ITSDO ≈ embedded generation outturn identity.

The optional boundary query parameter allows filtering by national or zonal boundary at the API level.

Caveats

01
Transmission-only — NDF − TSDF ≈ embedded generation forecast

TSDF excludes embedded generation. The difference NDF − TSDF approximates the forecast embedded generation component, mirroring the INDO − ITSDO ≈ embedded generation outturn identity. Do not use TSDF as a total demand figure in system-wide models.

02
No Pydantic schema — dict-based transformer

No Pydantic model in gridflow.schemas.elexon for TSDF. The TSDFTransformer enforces the 7-column silver shape directly via Polars casting.
