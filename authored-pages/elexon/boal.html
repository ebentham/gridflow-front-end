Overview

What this dataset is.

boal is the per-BMU GB balancing-mechanism acceptance feed — every accepted bid or offer instruction, keyed on (settlement_date, settlement_period, bm_unit_id, acceptance_number). It is the canonical audit trail for BSC settlement, BMU dispatch modelling, and STOR analysis. The Elexon API now serves it as BOALF; gridflow keeps the legacy slug.

Gridflow fetches it from /datasets/BOALF using the publishDateTimeFrom / publishDateTimeTo pattern. The raw JSON lands in bronze, is validated against ElexonBOAL, and written to silver via BOALTransformer.

Refreshed hourly with ~10 minute publication lag. Verified against vendor docs on 2026-05-08.

-- Top 20 BMUs by absolute redispatched MW in the last 24h
SELECT bm_unit_id,
       SUM(ABS(bid_offer_level_to - bid_offer_level_from)) AS abs_mw_redispatched,
       COUNT(*) AS n_acceptances
FROM read_parquet('data/silver/elexon/boal/**/*.parquet')
WHERE settlement_date >= current_date - INTERVAL 1 DAY
GROUP BY bm_unit_id
ORDER BY abs_mw_redispatched DESC
LIMIT 20;
import polars as pl

df = pl.read_parquet("data/silver/elexon/boal/**/*.parquet")
# STOR-flagged acceptances by hour-of-day
stor = (
    df.filter(pl.col("stor_flag"))
      .with_columns(pl.col("timestamp_utc").dt.hour().alias("hour"))
      .group_by("hour")
      .agg(pl.col("acceptance_number").count().alias("n_stor"))
      .sort("hour")
)
print(stor)
