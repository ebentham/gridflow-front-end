Overview

What this dataset is.

fuelhh is the half-hourly generation outturn aggregated by fuel type (FUELHH) — the realised MWh in each settlement period split by fuel category (CCGT, coal, nuclear, wind, solar, biomass, etc.). FUELHH is the canonical observation series for GB generation mix and underpins capacity-factor analytics and emissions reporting.

It is sourced from the BMRS Insights API at /datasets/FUELHH using the publishDateTimeFrom / publishDateTimeTo pattern. The raw JSON lands in bronze, is validated against the Pydantic schema, and written to the silver parquet partition via FuelHHTransformer.

Refreshed every 30 minutes with ~5 minute publication lag. Verified against vendor docs on 2026-05-08.

Schema

Silver layer columns.

Defined in gridflow/schemas/elexon.py · ElexonFuelHH. The silver table is partitioned by settlement_date. Point-in-time field: published_at.

Column Type Nullable Description
settlement_date PK date — Settlement date (BST/GMT calendar). Partition key.
settlement_period PK int — 1..50 (DST: 46 spring, 50 autumn).
timestamp_utc datetime[UTC] — Derived from (settlement_date, settlement_period) via utils/time.settlement_period_to_utc.
fuel_type PK str — Fuel category (CCGT, COAL, NUCLEAR, WIND, etc.). See fuel types section below for full list.
generation_mw float — MW. Average output for this fuel type in this period. Can be negative for pumped hydro / interconnector imports.
published_at datetime null Publication time (Canonical: ElexonFuelHH.published_at).
data_provider str — Default "elexon".
ingested_at datetime[UTC] null Time ingested into bronze. Useful for pipeline debugging.

NUCLEAR— Nuclear

NPSHYD— Non-pumped-storage hydro
PS— Pumped storage (negative = charging)

Caveats

Things to know before using this.

These are specific to fuelhh. Also read the Elexon-wide caveats on the vendor page.

01
Settlement period range is 1..50

On the autumn clock change, the day has 50 settlement periods (25 hours). On spring forward, it has 46 (23 hours). DST handling required when bucketing by day or hour. The silver layer normalises all timestamps to UTC; the settlement_period column can reach 50 on DST days.

02
fuelType casing

API returns uppercase fuel codes (CCGT, COAL, NUCLEAR…). The FuelHHTransformer preserves casing into the silver layer — don't normalise to lowercase in joins.

03
Embedded generation is excluded

FUELHH only covers BM-registered (metered) units. Sub-1 MW solar, small wind, and embedded CHP are not included — they appear as reduced demand, not as generation. Use NESO embedded estimates to add them back when calculating total GB generation.

04
WIND is onshore + offshore combined

The single WIND fuel type aggregates all metered wind regardless of technology. For an onshore/offshore split, use agpt (PSR type "Wind Onshore" / "Wind Offshore") or agws. Note those are D+1 settled, not live.

05
Interconnectors can be negative

When GB is exporting to France, generation_mw for INTFR will be negative. Decide whether to include or exclude interconnectors when summing to total generation, based on your use case.
