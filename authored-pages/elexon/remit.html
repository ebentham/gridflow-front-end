Overview

REMIT mandates publication of generation, transmission, and demand-side asset unavailability events affecting the wholesale electricity market. Each Urgent Market Message (UMM) records the affected asset, capacity figures, event window, cause, and current status. Messages are revised throughout the outage lifecycle — a single physical event can accumulate hundreds of revisions as status and end-time are updated.

The silver layer is append-only and revision-preserving: every received revision is stored. The canonical state of an asset is the row with the highest revision_number per mrid. The 24-column silver schema is enforced by REMITTransformer via Polars casting — no Pydantic model exists.

REMIT is the primary fundamental signal for unexpected generation loss — essential for capacity-margin analytics and gas-to-power modelling under outage stress.

-- Latest revision per mrid (canonical state)
SELECT DISTINCT ON (mrid)
  mrid, revision_number, fuel_type, event_status,
  normal_capacity_mw, available_capacity_mw,
  event_start_time, event_end_time
FROM read_parquet('data/silver/elexon/remit/**/*.parquet')
WHERE event_status = 'Active'
ORDER BY mrid, revision_number DESC;
import polars as pl

df = pl.read_parquet("data/silver/elexon/remit/**/*.parquet")
# Latest revision per mrid
latest = (
    df.sort("revision_number", descending=True)
    .unique(subset=["mrid"], keep="first")
    .filter(pl.col("event_status") == "Active")
)
print(latest.select(["mrid", "fuel_type", "normal_capacity_mw", "event_end_time"]))

Caveats

01
Max 23-hour query window — vendor-enforced hard cap

The Elexon API returns HTTP 400 for query windows exceeding ~24 hours. The gridflow connector enforces max_chunk_hours=23 to stay within the cap with a DST safety margin. Verified live 2026-05-09: 23h → HTTP 200; 25h → HTTP 400.

02
Append-only silver — always filter on max revision_number

Every revision of every UMM is stored. A single physical outage event can accumulate hundreds of revisions. The canonical state is the row with the highest revision_number per mrid. Queries that do not filter by max revision will double-count outage capacity.

03
No Pydantic schema — 24-column dict-based transformer

No Pydantic model in gridflow.schemas.elexon for REMIT. The REMITTransformer enforces the 24-column silver schema directly via Polars casting.
