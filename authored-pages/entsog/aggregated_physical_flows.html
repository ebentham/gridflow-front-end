Overview

What this dataset is.

aggregated_physical_flows holds daily, zone-level physical gas flows for every European balancing zone, broken down by adjacent system — Production, Storage, LNG Terminals, Distribution and cross-border counterparties. It is the canonical EU view of who is moving how much gas to whom across the transmission network, reported in kWh/day per the gas convention rather than MWh. Each row is one balancing-zone operator-direction's aggregated flow against a single adjacent system for one gas day.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/aggregatedData, queried over a gas-day range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to the silver parquet partition via GenericEntsogJsonTransformer, with no Pydantic class — the column shape is derived dynamically from the live response rather than validated against a fixed schema.

As the balancing-zone view of supply and offtake, it underpins gas-supply-tightness modelling and pairs naturally with GIE storage levels and ENTSO-E gas-fired generation to trace the gas-to-power chain. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- UK monthly LNG-vs-Production entry mix
SELECT date_trunc('month', period_from) AS month,
       adjacent_systems_key,
       SUM(value) / 1e9 AS twh
FROM read_parquet('data/silver/entsog/aggregated_physical_flows/**/*.parquet')
WHERE bz_short = 'UK'
  AND direction_key = 'entry'
  AND adjacent_systems_key IN ('LNG Terminals', 'Production', 'Storage')
GROUP BY 1, 2
ORDER BY 1, 2;
import polars as pl

df = pl.read_parquet("data/silver/entsog/aggregated_physical_flows/**/*.parquet")
# UK daily net flow (entry - exit) across all adjacent systems
net = (
    df.filter(pl.col("bz_short") == "UK")
      .with_columns(
          pl.when(pl.col("direction_key") == "entry")
            .then(pl.col("value"))
            .otherwise(-pl.col("value"))
            .alias("signed_kwh")
      )
      .group_by("period_from")
      .agg(pl.col("signed_kwh").sum().alias("net_kwh"))
      .sort("period_from")
)
print(net.tail(30))
Caveats

Things to know before using this.

These are specific to aggregated_physical_flows. Also read the ENTSO-G-wide caveats on the vendor page.

01
Unit is kWh/d, NOT MWh

ENTSO-G reports gas flows in kWh/day. Divide by 1,000 for MWh; by 1e9 for TWh. Preserve the unit column rather than assuming. vault Silver schema · unit constant

02
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

03
direction_key casing varies across endpoints

Lowercase here ("entry", "exit"); capitalised in /cmpUnsuccessfulRequests. Lowercase both sides when cross-endpoint joining. vault Known Issues · silver/entsog/generic.py::_normalise_column_names

04
flow_status: "Provisionnal" preserved verbatim

Vendor spelling Provisionnal (double-n) is preserved through bronze and silver. Filter on the exact string to distinguish from "Final". vault Known Issues · vendor JSON

05
Period offset is CEST/CET even with timeZone=UCT

Despite requesting UCT, periodFrom/periodTo carry +02:00 (CEST) / +01:00 (CET). parse_entsog_datetime_expr converts to UTC at silver. silver/entsog/datetime.py · vault Known Issues

06
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115
