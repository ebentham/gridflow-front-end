Overview

What this dataset is.

interconnections is the directed connectivity graph of the EU gas network — each row carries a from* block and a to* block describing one directed interconnection between points, zones, operators and countries. It is the canonical topology reference, returning UK-relevant links by default. Each row is one directed interconnection between a source side and a destination side at a point.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/interconnections with a default fromCountryKey=UK filter and limit=-1 for one-call extraction. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as InterconnectionsTransformer) running with reference_dataset=True, so the output is a single-file snapshot overwritten on each weekly refresh, with no Pydantic class — columns are derived dynamically.

As the point-level topology graph, it resolves which countries, zones and TSOs sit on each side of a cross-border link — the granular companion to aggregate_interconnections and the join key behind physical_flows. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- All UK cross-border interconnections
SELECT point_key, point_label,
       from_bz_label_long, to_bz_label_long,
       from_direction_key, to_direction_key
FROM read_parquet('data/silver/entsog/interconnections.parquet')
WHERE from_country_key = 'UK'
  AND to_country_key != 'UK'
ORDER BY point_label;
import polars as pl

ic = pl.read_parquet("data/silver/entsog/interconnections.parquet")
# How many UK interconnections per adjacent country?
adj = (
    ic.filter(pl.col("from_country_key") == "UK")
      .filter(pl.col("to_country_key").is_not_null())
      .group_by("to_country_label")
      .agg(pl.len().alias("links"))
      .sort("links", descending=True)
)
print(adj)
Caveats

Things to know before using this.

These are specific to interconnections. Also read the ENTSO-G-wide caveats on the vendor page.

01
Single-file reference dataset (no time partitions)

Output is interconnections.parquet overwritten on each refresh. silver/entsog/generic.py L196-202 · endpoints.py L240-247 reference=True

02
Default fromCountryKey=UK filter

The connector passes fromCountryKey=UK by default. For full EU connectivity, override the param. connectors/entsog/endpoints.py L246

03
validto vendor field-name typo preserved

Vendor uses validto (lowercase, no _To) rather than validTo. Preserved verbatim. vault Bronze sample L119

04
to_* block is mostly null for distribution endpoints

Records where to_system_label="Distribution" have null to_country_key, to_operator_key, etc. Filter on non-null to_country_key for cross-border-only views. vault Bronze sample L102-114

05
lastUpdateDateTime carries human-formatted strings

Vendor occasionally returns "May 8 2026 6:43PM"-style strings. parse_entsog_datetime_expr handles common formats but may return null for unparseable strings. vault Bronze sample L120 · silver/entsog/datetime.py
