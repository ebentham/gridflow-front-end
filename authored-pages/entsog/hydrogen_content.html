Overview

What this dataset is.

hydrogen_content held the hydrogen volume fraction of delivered gas at each point — historically the canonical H2-blending indicator. The vendor withdrew the endpoint on 2026-05-19, so the live API now returns HTTP 404 and only archived bronze data remains. It shares the operational-data shape with the other gas-quality indicators; the differentiator was indicator="Hydrogen Content". Each row is one operator-point-direction's H2 fraction for a single (historic) gas day.

When the endpoint was live it was sourced from the ENTSO-G Transparency Platform at /api/v1/operationalData with indicator=Hydrogen Content and timeZone=UCT, written to silver via GenericEntsogJsonTransformer (subclassed as HydrogenContentTransformer), with no Pydantic class — columns are derived dynamically. The connector registration is retained for historic replay; live fetches now bronze the 404 (empty) via the standard short-circuit.

As a historic-only surface, it remains useful for backtests of early hydrogen-blending trends — GB points stayed below ~0.02 mol%, while continental points such as Überackern ran higher. With methane and oxygen content withdrawn the same day, gcv and wobbe_index are the surviving live gas-quality surfaces. Schema verified against the 2026-05-08 vault Bronze sample; ENTSO-G was a public, no-authentication API.

-- Hydrogen content trend at GB points across the historic archive
SELECT date_trunc('month', timestamp_utc) AS month,
       AVG(value) AS avg_h2_fraction
FROM read_parquet('data/silver/entsog/hydrogen_content/**/*.parquet')
WHERE operator_key LIKE 'UK-TSO%'
GROUP BY 1
ORDER BY 1;
import polars as pl

df = pl.read_parquet("data/silver/entsog/hydrogen_content/**/*.parquet")
# Year-over-year hydrogen trend per point
yoy = (
    df.with_columns(pl.col("timestamp_utc").dt.year().alias("year"))
      .group_by(["year", "point_key"])
      .agg(pl.col("value").mean().alias("avg_h2"))
      .sort(["point_key", "year"])
)
print(yoy)
Caveats

Things to know before using this.

These are specific to hydrogen_content. Also read the ENTSO-G-wide caveats on the vendor page.

01
Discrepancy · endpoint withdrawn
Endpoint withdrawn by vendor (2026-05-19)

The live API returns HTTP 404 for ?indicator=Hydrogen%20Content, while the vault frontmatter records removed: 2026-05-19; historic data remains queryable from the bronze archive. Use the archive for backtests; the connector registration is retained for replay. .planning/reconciliation/entsog/01-hydrogen-content-http-404.md closed 2026-05-19 · vault frontmatter removed: 2026-05-19

02
Pre-removal H2 fractions were small

Historic GB values typically <0.02 mol%; the indicator was added in anticipation of blending mandates but never reached production scale in GB. vault Modelling notes; domain knowledge

03
timeZone=UCT (vendor typo, intentional)

When the endpoint was live, the connector sent UCT not UTC. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Historic bronze accessible via connector replay

The connector registration is retained in OPERATIONAL_INDICATORS (endpoints.py L109) for replay/backtest workflows; live fetches now bronze the 404 response (empty) via the standard short-circuit. connectors/entsog/client.py L109-115

05
Sister gas-quality endpoints also withdrawn

methane_content and oxygen_content were withdrawn the same day (2026-05-19). gcv and wobbe_index remain the surviving gas-quality surfaces. .planning/reconciliation/entsog/0[1-4]-*-http-404.md
