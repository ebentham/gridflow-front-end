Overview

What this dataset is.

balancing_zones is the inventory of every ENTSO-G balancing zone, carrying each zone's manager, EIC code, map coordinates and zone-replacement history. It is the canonical reference for resolving the bz_key codes used across the aggregated flow data — keys like "UK---------" resolve here to "British Balancing Zone" and its manager. Each row is one balancing zone, active or deactivated.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/balancingZones with limit=-1 for one-call extraction. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as BalancingZonesTransformer) running with reference_dataset=True, so the output is a single-file snapshot overwritten on each weekly refresh, with no Pydantic class — columns are derived dynamically.

As the zone dictionary, it lets analysts label aggregated_physical_flows zone keys, distinguish GB from the separate Northern Ireland zone, and reconstruct historical-to-current zone mappings after merges (GASPOOL + NCG → THE). Pair it with operators and connection_points. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

Caveats

Things to know before using this.

These are specific to balancing_zones. Also read the ENTSO-G-wide caveats on the vendor page.

01
Single-file reference dataset (no time partitions)

Output is balancing_zones.parquet overwritten on each refresh. silver/entsog/generic.py L196-202 · endpoints.py L223-230 reference=True

02
bz_key carries trailing dashes — preserve verbatim

Zone keys like "UK---------" and "DE-MGAS----" use trailing dashes as padding. Do not strip; the dashes are part of the key. vault Known Issues

03
is_deactivated is "0"/"1" string

Vendor text, not bool. Cast as needed. vault Bronze sample L83

04
Pagination via limit+offset

No from/to accepted. Connector sets limit=-1 for one-call extraction. vault Query-params

05
Zone-replacement history via replaced_since/replaced_by

When zones merge (e.g. GASPOOL + NCG → THE), the old zones get replaced_by set with the new zone key. Use to reconstruct historical-to-current zone mappings. vault Bronze sample L81-82
