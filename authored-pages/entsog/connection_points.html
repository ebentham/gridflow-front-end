Overview

What this dataset is.

connection_points is the full inventory of ENTSO-G connection points, carrying each point's label, map coordinates, infrastructure type, commercial type and a rich set of cross-border / interconnection / import flags. It is the canonical reference for resolving the point_key codes used across operational data — keys like "ITP-00005" resolve here to "Bacton (IUK)" and its point type. Each row is one connection point.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/connectionPoints with limit=-1 for one-call extraction. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as ConnectionPointsTransformer) running with reference_dataset=True, so the output is a single-file snapshot overwritten on each weekly refresh, with no Pydantic class — columns are derived dynamically.

As the point dictionary, it lets analysts filter operational data by point type or cross-border status, distinguish GB's post-Brexit ExtEU designation from intra-EU points, and label physical_flows per-point rows. Pair it with operator_point_directions and interconnections. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

SELECT point_key, point_label, point_type
FROM read_parquet('data/silver/entsog/connection_points.parquet')
WHERE is_cross_border = true
  AND infrastructure_label = 'Transmission'
  AND has_data = true

Caveats

Things to know before using this.

These are specific to connection_points. Also read the ENTSO-G-wide caveats on the vendor page.

01
Single-file reference dataset (no time partitions)

Output is connection_points.parquet overwritten on each refresh. silver/entsog/generic.py L196-202 · endpoints.py L206-213 reference=True

02
is_cam_relevant coalesces uppercase + camelCase

Vendor returns isCAMRelevant here vs isCamRelevant in operationalData. _normalise_column_names collapses both. silver/entsog/generic.py L264-290

03
is_single_operator is "0"/"1" string

Vendor returns text, not bool. Cast as needed. vault Bronze sample L73

04
Pagination via limit+offset

No from/to accepted. Connector sets limit=-1 for one-call extraction. vault Query-params

05
pointType taxonomy is vendor-fixed

Values include "Cross-Border Transmission IP between EU and ExtEU", "Cross-Border Transmission IP within EU", "Distribution Point", "Storage Point", "LNG Terminal". Use as a categorical filter. vault Bronze sample L79
