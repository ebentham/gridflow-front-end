Overview

What this dataset is.

allocations holds the final allocated gas quantities per operator-point-direction — the settlement-grade flow record that sits downstream of nominations and physical flows. It shares its operational column set with nominations and the other ENTSO-G operational indicators; the differentiator is indicator="Allocation". Allocation is the legally settled quantity rather than the metered throughput. Each row is one operator-point-direction's allocated quantity for a single gas day.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/operationalData?indicator=Allocation, queried per operator-point-direction over a gas-day range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as AllocationsTransformer), with no Pydantic class — columns are derived dynamically from the live response rather than validated against a fixed schema.

As the settlement-grade flow record, it is the denominator for nomination-vs-actual accuracy work and the reconciliation point for CMP adjustments, pairing naturally with nominations and physical_flows. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Nomination vs allocation error per point (last 30 days)
SELECT n.period_from::date AS gas_day,
       n.point_key, n.direction_key,
       n.value AS nominated_kwh,
       a.value AS allocated_kwh,
       (a.value - n.value) AS delta_kwh
FROM read_parquet('data/silver/entsog/nominations/**/*.parquet') n
JOIN read_parquet('data/silver/entsog/allocations/**/*.parquet') a
  ON  n.period_from  = a.period_from
  AND n.point_key    = a.point_key
  AND n.operator_key = a.operator_key
  AND n.direction_key= a.direction_key
WHERE n.period_from >= current_date - INTERVAL 30 DAY
ORDER BY 1;
import polars as pl

df = pl.read_parquet("data/silver/entsog/allocations/**/*.parquet")
# Total GB allocations per direction per day
gb = (
    df.filter(pl.col("operator_key").str.starts_with("UK-TSO-"))
      .group_by(["period_from", "direction_key"])
      .agg((pl.col("value").sum() / 1e6).alias("gwh"))
      .sort("period_from")
)
print(gb.tail(14))
Caveats

Things to know before using this.

These are specific to allocations. Also read the ENTSO-G-wide caveats on the vendor page.

01
Indicator string is exact-case

Vendor rejects allocation/allocations; connector sends the literal "Allocation". OPERATIONAL_INDICATORS["allocations"] = "Allocation" · connectors/entsog/endpoints.py L98

02
Allocations are post-flow

A row appears ~1 gas day after the flow date. For real-time analytics, use nominations or physical_flows instead. vault Publication-lag table

03
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

05
Allocation ≠ physical flow

Allocation is the legally settled quantity; physical_flow is the metered throughput. They typically agree but differ when there are CMP adjustments. domain knowledge / ENTSO-G data dictionary
