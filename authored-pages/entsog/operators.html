Overview

What this dataset is.

operators is the inventory of every ENTSO-G transmission system operator, carrying each TSO's short and long names, country, type and logo. It is the canonical reference for resolving operator_key codes — keys like "UK-TSO-0001" resolve here to National Gas Transmission and its country. Each row is one operator, keyed by operator_key.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/operators with limit=-1 and hasData=1 for one-call extraction of operators with operational data. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as OperatorsTransformer) running with reference_dataset=True, so the output is a single-file snapshot overwritten on each weekly refresh, with no Pydantic class — columns are derived dynamically.

As the TSO dictionary, it attaches country and zone metadata to operational rows and supports cross-vendor joins. Note that an operator's registered country can differ from its operating zone — BBL company is NL-registered yet runs the UK-NL pipeline — so resolve operating zone via operator_point_directions. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

Caveats

Things to know before using this.

These are specific to operators. Also read the ENTSO-G-wide caveats on the vendor page.

01
Single-file reference dataset (no time partitions)

Output is operators.parquet overwritten on each refresh, not partitioned. silver/entsog/generic.py L196-202 · endpoints.py L214-222 reference=True

02
hasData=1 recommended

Default hasData=1 restricts to operators with operational data. Without it the inventory includes inactive/legacy TSOs. vault Query-params · endpoints.py L220

03
Pagination via limit+offset

No from/to accepted; full inventory only. Connector sets limit=-1 for one-call extraction. vault Query-params · endpoints.py L214-222

04
Operator country may differ from operating zone

E.g. UK-TSO-0004 (BBL company) has operator_country_key="NL" despite operating a UK-relevant interconnection pipeline. Resolve operating zone via operator_point_directions. vault Known Issues · cross-reference operator_point_directions

05
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115
