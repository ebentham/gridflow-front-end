Overview

What this dataset is.

operator_point_directions is the join table at the heart of ENTSO-G — operator-point-direction triples annotated with validity windows, CAM/CMP flags, contract-product availability, GCV ranges and free-text CMP narrative sentences. It is the canonical reference for resolving the composite keys that operational data is reported against. Each row is one operator-point-direction combination with its full metadata.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/operatorPointDirections with limit=-1 and hasData=1 for one-call extraction of active combos. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as OperatorPointDirectionsTransformer) running with reference_dataset=True, so the output is a single-file snapshot, with no Pydantic class — columns are derived dynamically (the full sample carries 70+ columns).

As the metadata spine, it attaches country, zone, adjacent-counterparty and contract context to physical_flows and the other operational datasets via the operator-point-direction join. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Attach country/zone metadata to physical_flows
SELECT pf.timestamp_utc, pf.point_key, pf.direction_key,
       opd.t_so_country, opd.t_so_balancing_zone, opd.adjacent_country,
       pf.flow_gwh_per_day
FROM read_parquet('data/silver/entsog/physical_flows/**/*.parquet') pf
LEFT JOIN read_parquet('data/silver/entsog/operator_point_directions.parquet') opd
  ON  pf.operator_key  = opd.operator_key
  AND pf.point_key     = opd.point_key
  AND pf.direction_key = opd.direction_key
WHERE pf.timestamp_utc >= current_date - INTERVAL 7 DAY
ORDER BY pf.timestamp_utc;
import polars as pl

opd = pl.read_parquet("data/silver/entsog/operator_point_directions.parquet")
# Which adjacent countries does UK have direct interconnections to?
uk_links = (
    opd.filter(pl.col("t_so_country") == "UK")
       .group_by("adjacent_country")
       .agg(pl.len().alias("combos"))
       .sort("combos", descending=True)
)
print(uk_links)
Caveats

Things to know before using this.

These are specific to operator_point_directions. Also read the ENTSO-G-wide caveats on the vendor page.

01
Single-file reference dataset (no time partitions)

Output is operator_point_directions.parquet overwritten on each refresh. silver/entsog/generic.py L196-202 · endpoints.py L231-239 reference=True

02
is_cam_relevant coalesces uppercase + camelCase variants

Vendor returns isCAMRelevant (uppercase CAM) here vs isCamRelevant in operationalData. _normalise_column_names collapses both into is_cam_relevant via pl.coalesce. silver/entsog/generic.py L264-290 · vault Known Issues

03
tSO* fields produce odd snake_case (t_so_country)

The _camel_to_snake regex splits tSO as t_s_o, then collapses repeated _ to t_so. Match the silver column name, not the camelCase. silver/entsog/generic.py L257-261

04
timeZone not accepted

Reference endpoints take no from/to/timeZone — full-snapshot only with limit=-1. vault Query-params · endpoints.py L231-239

05
hasData=1 excludes recently-dormant combos

Use without the filter for full historical inventory; with the filter for currently-active combos only. vault Known Issues
