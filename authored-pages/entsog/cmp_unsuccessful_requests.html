Overview

What this dataset is.

cmp_unsuccessful_requests holds capacity-auction requests whose requested volume exceeded what was allocated — the canonical CMP signal for excess shipper demand at congested interconnection points. Each record carries requested, allocated and unallocated volume plus an aggregated count of unsuccessful requests. Each row is one operator-point-direction's unsuccessful-request aggregate for a capacity-validity window.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/cmpUnsuccessfulRequests, queried over a date range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as CmpUnsuccessfulRequestsTransformer), with no Pydantic class — columns are derived dynamically. Endpoint-specific fields include requestedVolume, allocatedVolume, unallocatedVolume and occurenceCount (vendor spelling).

As the demand-side scarcity signal, it pairs with cmp_unavailable_firm_capacity (the supply-side trigger) and cmp_auction_premiums (the resulting price) to build a full congestion picture at GB points. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Top 10 GB points by unallocated volume in the current gas year
SELECT point_key, point_label, direction_key,
       SUM(unallocated_volume) / 1e6 AS total_unallocated_gwh,
       SUM(occurence_count) AS total_unsuccessful_requests
FROM read_parquet('data/silver/entsog/cmp_unsuccessful_requests/**/*.parquet')
WHERE operator_key LIKE 'UK-TSO%'
  AND capacity_from >= '2025-10-01'
  AND capacity_from <  '2026-10-01'
GROUP BY 1, 2, 3
ORDER BY total_unallocated_gwh DESC
LIMIT 10;
import polars as pl

df = pl.read_parquet("data/silver/entsog/cmp_unsuccessful_requests/**/*.parquet")

# Top 10 points by unsuccessful request count
print(
    joined.group_by(["point_key", "direction_key"])
          .agg(pl.col("occurence_count").sum())
          .sort("occurence_count", descending=True)
          .head(10)

Caveats

Things to know before using this.

These are specific to cmp_unsuccessful_requests. Also read the ENTSO-G-wide caveats on the vendor page.

01
occurenceCount (vendor typo) — single r

Vendor spells occurenceCount (missing the second r in occurrence). Preserved verbatim in silver as occurence_count. vault Bronze sample L101

02
Discrepancy · cross-endpoint casing
direction_key is capitalised here ("Exit"/"Entry")

Vault Bronze sample shows capitalised "Exit" on this endpoint, while /operationalData returns lowercase entry/exit; _normalise_column_names documents the variance but does not fold the values. Lowercase both sides before joining across endpoint families. vault Bronze sample L93 · vault Known Issues · GenericEntsogJsonTransformer._normalise_column_names

03
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

05
auction_from can be "N/A"

Some CMP records have no specific auction window (long-standing congestion declarations). parse_entsog_datetime_expr returns null for unparseable values. vault Bronze sample L83 · silver/entsog/datetime.py
