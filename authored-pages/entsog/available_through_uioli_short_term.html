Overview

What this dataset is.

available_through_uioli_short_term holds the day-ahead capacity a TSO reclaims when nominated firm-booked capacity goes unused, returned to the market for a single gas day. It shares the canonical ENTSO-G operational-data shape with the other indicators; the differentiator is indicator="Available through UIOLI short-term" (lowercase through, hyphenated short-term). It is the day-ahead sibling of UIOLI long-term. Each row is one operator-point-direction's UIOLI-recovered short-term capacity for a one-day window.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/operationalData with indicator=Available through UIOLI short-term, queried per operator-point-direction over a gas-day range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as AvailableThroughUioliShortTermTransformer), with no Pydantic class — columns are derived dynamically.

Because it reflects what shippers under-nominated the previous gas day, it is the most timely of the CMP recovery signals — pair it with firm_booked to see what triggered the reclamation, and with interruptible_available as an adjacent supply of short-term capacity. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Daily UIOLI short-term recovered (last 30 days)
SELECT period_from::date AS gas_day,
       point_key, direction_key,
       value / 1e6 AS recovered_gwh
FROM read_parquet('data/silver/entsog/available_through_uioli_short_term/**/*.parquet')
WHERE period_from >= current_date - INTERVAL 30 DAY
  AND value > 0
ORDER BY 1 DESC, recovered_gwh DESC;
import polars as pl

df = pl.read_parquet("data/silver/entsog/available_through_uioli_short_term/**/*.parquet")
# Daily total UIOLI-short across GB points
gb_total = (
    df.filter(pl.col("operator_key").str.starts_with("UK-TSO-"))
      .group_by("period_from")
      .agg((pl.col("value").sum() / 1e6).alias("uioli_short_gwh"))
      .sort("period_from")
)
print(gb_total.tail(14))
Caveats

Things to know before using this.

These are specific to available_through_uioli_short_term. Also read the ENTSO-G-wide caveats on the vendor page.

01
Indicator string is exact-case (with hyphen)

Connector sends the literal "Available through UIOLI short-term". OPERATIONAL_INDICATORS["available_through_uioli_short_term"] · connectors/entsog/endpoints.py L114

02
Daily-cadence, day-ahead window

UIOLI short-term is the day-ahead version of UIOLI long-term — recovered capacity is offered for one gas day only. Pair with firm_booked to see what triggered the reclamation. ENTSO-G CMP Network Code

03
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

05
Different cadence from UIOLI long-term

Long-term is annual; short-term is daily — the two report different mechanisms despite the naming similarity. ENTSO-G Network Code on CAM/CMP
