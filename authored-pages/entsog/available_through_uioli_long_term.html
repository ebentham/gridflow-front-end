Overview

What this dataset is.

available_through_uioli_long_term holds capacity the TSO has reclaimed from underused long-term firm bookings under the use-it-or-lose-it (UIOLI) mechanism and returned to auction. It shares the canonical ENTSO-G operational-data shape with the other indicators; the differentiator is indicator="Available through UIOLI long-term" (lowercase through, hyphenated long-term). Unlike surrender, UIOLI is TSO-initiated. Each row is one operator-point-direction's UIOLI-recovered long-term capacity for a validity window.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/operationalData with indicator=Available through UIOLI long-term, queried per operator-point-direction over a gas-day range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as AvailableThroughUioliLongTermTransformer), with no Pydantic class — columns are derived dynamically.

Because the TSO assesses utilisation roughly annually, the timing of UIOLI releases is itself a market-stress signal worth tracking against firm_booked, which it reduces retrospectively. It rounds out the CMP family alongside available_through_oversubscription and available_through_surrender. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Annual UIOLI long-term recovery per point (last 3 gas years)
SELECT EXTRACT(year FROM period_from) AS year_start,
       point_key, direction_key,
       MAX(value) / 1e6 AS max_recovered_gwh
FROM read_parquet('data/silver/entsog/available_through_uioli_long_term/**/*.parquet')
WHERE value > 0
  AND period_from >= current_date - INTERVAL 3 YEAR
GROUP BY 1, 2, 3
ORDER BY 1, max_recovered_gwh DESC;

Caveats

Things to know before using this.

These are specific to available_through_uioli_long_term. Also read the ENTSO-G-wide caveats on the vendor page.

01
Indicator string is exact-case (with hyphen)

Connector sends the literal "Available through UIOLI long-term" — lowercase through, hyphenated long-term. OPERATIONAL_INDICATORS["available_through_uioli_long_term"] · connectors/entsog/endpoints.py L113

02
UIOLI long-term is annual-cadence

The TSO assesses utilisation roughly annually; expect 1–2 release events per gas-year per congested point. ENTSO-G CMP Network Code

03
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

05
Use-It-Or-Lose-It is involuntary

Unlike available_through_surrender (voluntary), UIOLI is TSO-initiated capacity reclamation. Track the timing of UIOLI releases as a market-stress signal. domain knowledge / CMP mechanism
