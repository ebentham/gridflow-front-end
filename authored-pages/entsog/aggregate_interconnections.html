Overview

What this dataset is.

aggregate_interconnections is the zone-pair-aggregated view of ENTSO-G network topology — it rolls the point-level interconnections reference up into zone-to-zone tuples, each annotated with a count of how many adjacent systems are aggregated into it. It is the canonical reference for resolving zone-level flow joins: the adjacent_systems_key values it enumerates are exactly the counterparty categories that turn up in aggregated_physical_flows. Each row is one source-zone operator-direction paired with a single adjacent-system aggregate.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/aggregateInterconnections, queried with a default countryKey=UK filter and limit=-1 for one-call extraction. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer, with no Pydantic class — columns are derived dynamically from the live response. It runs with reference_dataset=True, so the output is a single-file snapshot overwritten on each weekly refresh rather than a time-partitioned table.

As the topology reference behind zone-level flows, it lets analysts enumerate which adjacent-system keys are valid for a zone before joining or aggregating aggregated_physical_flows, and resolves which TSOs sit on each side of a cross-border pair. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- All UK adjacent-system aggregates by direction
SELECT direction_key, adjacent_systems_label,
       SUM(adjacent_systems_count) AS total_sub_systems
FROM read_parquet('data/silver/entsog/aggregate_interconnections.parquet')
WHERE country_key = 'UK'
GROUP BY 1, 2
ORDER BY 1, total_sub_systems DESC;
import polars as pl

agg_ic = pl.read_parquet("data/silver/entsog/aggregate_interconnections.parquet")
# Enumerate adjacent-system keys for UK (the codelist for aggregated_physical_flows)

Caveats

Things to know before using this.

These are specific to aggregate_interconnections. Also read the ENTSO-G-wide caveats on the vendor page.

01
Single-file reference dataset (no time partitions)

Output is aggregate_interconnections.parquet overwritten on each refresh. silver/entsog/generic.py L196-202 · endpoints.py L249-257 reference=True

02
Default countryKey=UK filter

The connector passes countryKey=UK by default. For full EU connectivity, override the param. connectors/entsog/endpoints.py L255

03
direction_key is lowercase here ("entry"/"exit")

Same casing as /operationalData and /aggregatedData (vs capitalised in /cmpUnsuccessfulRequests). Safe to join directly to aggregated_physical_flows. vault Bronze sample L78

04
adjacent_systems_key is a concatenation, not a categorical

Values like "TransmissionUK-NI------" and "LNG Terminals" mix infrastructure types with zone keys. Parse with care for downstream taxonomies. vault Bronze sample L79

05
Pagination via limit+offset

No from/to accepted. Connector sets limit=-1 for one-call extraction. vault Query-params
