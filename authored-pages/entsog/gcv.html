Overview

What this dataset is.

gcv holds the Gross Calorific Value of delivered gas at each point — the energy released per normal cubic metre, in kWh/Nm³. It is the canonical conversion factor for moving between volumetric (Nm³/d) and energy (kWh/d) gas-flow units. It shares the operational-data shape with physical_flows; the differentiators are indicator="GCV" and unit="kWh/Nm³". Each row is one operator-point-direction's GCV for a single gas day.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/operationalData with indicator=GCV, queried per operator-point-direction over a gas-day range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as GcvTransformer), with no Pydantic class — columns are derived dynamically.

As the volumetric↔energy bridge, it lets analysts convert physical_flows energy flows into volumes and is the surviving energy-density proxy now that methane content has been withdrawn. It pairs with wobbe_index for the full interchangeability picture. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- 30-day average GCV per Bacton point
SELECT point_key, point_label,
       AVG(value) AS avg_gcv_kwh_nm3,
       STDDEV(value) AS gcv_volatility
FROM read_parquet('data/silver/entsog/gcv/**/*.parquet')
WHERE timestamp_utc >= current_date - INTERVAL 30 DAY
  AND point_key IN ('ITP-00005', 'ITP-00207')
GROUP BY 1, 2;
import polars as pl

df = pl.read_parquet("data/silver/entsog/gcv/**/*.parquet")
# Convert physical_flows kWh/d to volumetric Nm3/d using GCV
flows = pl.read_parquet("data/silver/entsog/physical_flows/**/*.parquet")
joined = flows.join(
    df.select(["timestamp_utc", "point_key", "value"]).rename({"value": "gcv"}),
    on=["timestamp_utc", "point_key"], how="left",
)
print(joined.with_columns((pl.col("flow_gwh_per_day") * 1e6 / pl.col("gcv")).alias("nm3_per_day")).head())
Caveats

Things to know before using this.

These are specific to gcv. Also read the ENTSO-G-wide caveats on the vendor page.

01
Indicator string is exact-case ("GCV" — uppercase)

Vendor rejects gcv or Gcv; connector sends the literal "GCV". OPERATIONAL_INDICATORS["gcv"] = "GCV" · connectors/entsog/endpoints.py L106

02
Unit kWh/Nm³ — energy per normal cubic metre

Use GCV to convert between volumetric (Nm³/d) and energy (kWh/d) units. Typical North Sea blend = 11.5 kWh/Nm³. vault Bronze sample L110; gas-physics convention

03
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

05
Slow-varying — daily values are smooth

GCV changes slowly (gas-blend variability is small day-to-day). Use for unit-conversion smoothing rather than as a market signal. domain knowledge / vault sample volatility
