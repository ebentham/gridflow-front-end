Overview

What this dataset is.

tariff_simulations holds the vendor's simulated total tariff cost for representative capacity products (yearly, quarterly, monthly, daily) per operator-point-direction. It is the canonical forward-looking transmission-cost benchmark for capacity-booking decisions — a compositional total of all tariff components for a standardised product. Each row is one simulated product cost for an operator-point-direction.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/tariffsSimulations with a default countryKey=UK filter and timeZone=UCT. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as TariffSimulationsTransformer), with no Pydantic class — columns are derived dynamically; both local-currency and EUR cost columns are carried.

As the forward-looking cost benchmark, it builds reservation-cost ladders across product durations and is best read against realised cmp_auction_premiums (simulated vs actual) and the line-item tariffs detail. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Product-duration ladder for National Gas TSO firm
SELECT product_type,
       product_simulation_cost_in_euro AS cost_eur,
       (product_simulation_cost_in_euro / 365.0) AS implied_daily_eur
FROM read_parquet('data/silver/entsog/tariff_simulations/**/*.parquet')
WHERE operator = 'National Gas TSO'
  AND tariff_capacity_type = 'Firm'
  AND period_from = '2025-10-01'
ORDER BY product_simulation_cost_in_euro DESC;
import polars as pl

df = pl.read_parquet("data/silver/entsog/tariff_simulations/**/*.parquet")
# Year-over-year UK firm-yearly tariff cost
yoy = (
    df.filter(pl.col("country_code") == "GB")
      .filter(pl.col("tariff_capacity_type") == "Firm")
      .filter(pl.col("product_type") == "Yearly")
      .with_columns(pl.col("period_from").dt.year().alias("gas_year"))
      .group_by(["gas_year", "operator"])
      .agg(pl.col("product_simulation_cost_in_euro").mean().alias("avg_euro"))
      .sort(["operator", "gas_year"])
)
print(yoy)
Caveats

Things to know before using this.

These are specific to tariff_simulations. Also read the ENTSO-G-wide caveats on the vendor page.

01
Default countryKey=UK filter

Connector passes countryKey=UK. For full EU, override the param. connectors/entsog/endpoints.py L195

02
Simulation is for representative products, not actual auctions

The productSimulationCostIn* figures are the vendor's compositional total of all tariff components for a standardised capacity product. Actual auction-cleared prices may differ — compare against cmp_auction_premiums. vault Modelling notes; tariff regulation framework

03
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

05
EUR and local-currency columns differ under FX

For GB operators both product_simulation_cost_in_local_currency and product_simulation_cost_in_euro are EUR (operatorCurrency = "EUR"). For non-EUR operators (RO, HU) they differ at the published FX rate; check exchangeRateReferenceDate. vault Bronze sample L86; FX-rate convention
