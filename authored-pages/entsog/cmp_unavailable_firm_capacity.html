Overview

What this dataset is.

cmp_unavailable_firm_capacity holds firm capacity that has been classified as unavailable at a point — allocated, but not actually deliverable — alongside the available and technical capacity for reference. It is the canonical CMP signal that, when significant relative to technical_capacity, obliges the TSO to consider UIOLI long-term or oversubscription. Each row is one operator-point-direction's unavailable-capacity declaration for a capacity-validity window.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/cmpUnavailables, queried over a date range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as CmpUnavailableFirmCapacityTransformer), with no Pydantic class — columns are derived dynamically. It shares the CMP shape with cmp_unsuccessful_requests; the differentiator is unavailable_capacity.

As the trigger condition for the capacity-recovery mechanisms, it is the causal lead indicator: persistent unavailability should show up downstream as UIOLI or oversubscription releases. Pair it with the available_through_* datasets to verify the response and with firm_technical for the ceiling. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Unavailability as a percentage of technical capacity per point (current gas year)
SELECT point_key, point_label, direction_key,
       MAX_BY(unavailable_capacity, capacity_from) / 1e6 AS unavailable_gwh,
       MAX_BY(technical_capacity, capacity_from)   / 1e6 AS technical_gwh,
       (MAX_BY(unavailable_capacity, capacity_from) * 100.0
        / NULLIF(MAX_BY(technical_capacity, capacity_from), 0)) AS pct_unavailable
FROM read_parquet('data/silver/entsog/cmp_unavailable_firm_capacity/**/*.parquet')
WHERE capacity_from <= current_date AND capacity_to > current_date
GROUP BY 1, 2, 3
ORDER BY pct_unavailable DESC;
import polars as pl

df = pl.read_parquet("data/silver/entsog/cmp_unavailable_firm_capacity/**/*.parquet")
# How does unavailability evolve year-over-year at Bacton?
yoy = (
    df.filter(pl.col("point_key") == "ITP-00005")
      .with_columns(pl.col("capacity_from").dt.year().alias("year"))
      .group_by("year")
      .agg((pl.col("unavailable_capacity").mean() / 1e6).alias("avg_unavailable_gwh"))
      .sort("year")
)
print(yoy)
Caveats

Things to know before using this.

These are specific to cmp_unavailable_firm_capacity. Also read the ENTSO-G-wide caveats on the vendor page.

01
direction_key is capitalised ("Exit"/"Entry")

CMP-family endpoints capitalise direction values — unlike /operationalData. Lowercase both sides for cross-family joins. vault Known Issues; CMP-family convention

02
Triggers UIOLI / oversubscription

When unavailable_capacity is significant relative to technical_capacity, the TSO is required to consider UIOLI long-term or oversubscription. Pair with the available_through_* datasets to verify the response. ENTSO-G CMP Network Code

03
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

05
available + unavailable typically < technical

The residual is TSO-reserved (system balancing, frequency control). Don't model technical = available + unavailable exactly. ENTSO-G data dictionary
