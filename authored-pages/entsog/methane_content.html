Overview

What this dataset is.

methane_content held the methane volume fraction of delivered gas at each point — historically the canonical CH4-purity indicator. The vendor withdrew the endpoint on 2026-05-19, so the live API now returns HTTP 404 and only archived bronze data remains. It shares the operational-data shape with the other gas-quality indicators; the differentiator was indicator="Methane Content". Each row is one operator-point-direction's CH4 fraction for a single (historic) gas day.

When the endpoint was live it was sourced from the ENTSO-G Transparency Platform at /api/v1/operationalData with indicator=Methane Content and timeZone=UCT, written to silver via GenericEntsogJsonTransformer (subclassed as MethaneContentTransformer), with no Pydantic class — columns are derived dynamically. The connector registration is retained for historic replay; live fetches now bronze the 404 (empty) via the standard short-circuit.

As a historic-only surface, methane share maps directly to energy density, so the surviving gcv is the recommended proxy (CH4 share back-calculates approximately as gcv / 11.05 for North Sea blends). With hydrogen and oxygen content withdrawn the same day, gcv and wobbe_index are the surviving live surfaces. Schema verified against the 2026-05-08 vault Bronze sample; ENTSO-G was a public, no-authentication API.

-- Average methane content per point across 2025
SELECT point_key, point_label,
       AVG(value) AS avg_methane_pct,
       MIN(value) AS min_methane_pct,
       MAX(value) AS max_methane_pct
FROM read_parquet('data/silver/entsog/methane_content/**/*.parquet')
WHERE timestamp_utc BETWEEN '2025-01-01' AND '2026-01-01'
GROUP BY 1, 2
ORDER BY avg_methane_pct DESC;
import polars as pl

df = pl.read_parquet("data/silver/entsog/methane_content/**/*.parquet")
# Daily GB CH4 spread (interchangeability check across GB entry points)
spread = (
    df.filter(pl.col("operator_key").str.starts_with("UK-TSO-"))
      .group_by("timestamp_utc")
      .agg((pl.col("value").max() - pl.col("value").min()).alias("daily_spread_mol_pct"))
      .sort("timestamp_utc")
      .tail(30)
)
print(spread)
Caveats

Things to know before using this.

These are specific to methane_content. Also read the ENTSO-G-wide caveats on the vendor page.

01
Discrepancy · endpoint withdrawn
Endpoint withdrawn by vendor (2026-05-19)

The live API returns HTTP 404 for ?indicator=Methane%20Content, while the vault frontmatter records removed: 2026-05-19; historic data remains queryable from the bronze archive. .planning/reconciliation/entsog/03-methane-content-http-404.md closed 2026-05-19 · vault frontmatter removed: 2026-05-19

02
GCV is the surviving energy-density proxy

Since methane fraction maps directly to energy density, use gcv (still live) as the proxy. Methane share back-calculates approximately as gcv / 11.05 for North Sea blends. gas-physics convention; vault Modelling notes

03
timeZone=UCT (vendor typo, intentional)

When the endpoint was live, the connector sent UCT not UTC. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Historic bronze accessible via connector replay

The connector registration is retained in OPERATIONAL_INDICATORS (endpoints.py L108) for replay/backtest workflows. connectors/entsog/client.py L109-115

05
Sister gas-quality endpoints also withdrawn

hydrogen_content and oxygen_content were withdrawn the same day (2026-05-19). gcv and wobbe_index remain the surviving gas-quality surfaces. .planning/reconciliation/entsog/0[1-4]-*-http-404.md
