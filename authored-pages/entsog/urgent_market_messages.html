Overview

What this dataset is.

urgent_market_messages holds real-time TSO bulletins (UMMs) covering unavailability, capacity reductions and other network events — the live notification stream for unplanned gas-network events. Each message carries the publication time, an event window, status, affected participant and (when known) capacity figures. Each row is one UMM bulletin.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/urgentMarketMessages with limit=-1 (pagination via limit+offset; no from/to accepted). The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as UrgentMarketMessagesTransformer) running with reference_dataset=True, so the output is a single-file snapshot, with no Pydantic class — columns are derived dynamically.

As the live event stream, it is the closest replacement for the withdrawn interruptions endpoint: filter on event_status = "Active" for forward-looking outage signals, join operators on market_participant_key, and cross-check physical_flows for the realised impact. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Active UMMs in the next 30 days (forward-looking outages)
SELECT publication_date_time, market_participant_name, message_type,
       event_start, event_stop,
       unavailable_capacity / 1e6 AS unavailable_gwh
FROM read_parquet('data/silver/entsog/urgent_market_messages.parquet')
WHERE event_status = 'Active'
  AND event_start BETWEEN current_timestamp AND current_timestamp + INTERVAL 30 DAY
ORDER BY event_start;
import polars as pl

df = pl.read_parquet("data/silver/entsog/urgent_market_messages.parquet")
# UMM event count by TSO in the last year
recent = (
    df.filter(pl.col("publication_date_time") >= pl.lit("2025-05-01"))
      .group_by("market_participant_name")
      .agg(pl.len().alias("events"))
      .sort("events", descending=True)
)
print(recent.head(15))
Caveats

Things to know before using this.

These are specific to urgent_market_messages. Also read the ENTSO-G-wide caveats on the vendor page.

01
Single-file reference dataset (no time partitions)

Output is urgent_market_messages.parquet overwritten on each refresh, not the partitioned year=YYYY/month=MM/ layout used by other operational datasets. silver/entsog/generic.py L196-202 · endpoints.py L198-205 reference=True

02
No from/to query params

The endpoint returns a full UMM snapshot; pagination via limit+offset only. Connector sets limit=-1 (all records). vault Query-params table · endpoints.py L198-205

03
uMMType → u_mm_type snake_case

Vendor camelCase uMMType is normalised to u_mm_type (the extra underscore between M and M is a _camel_to_snake artefact, not a typo). silver/entsog/generic.py L257-261, L264-290

04
is_latest_version is a string, not bool

Vendor returns "Yes"/"No" text — the transformer preserves it as str. Cast in DuckDB if you need a boolean predicate. vault Bronze sample L95 · silver/entsog/generic.py does not coerce

05
Long-running events (years-spanning)

A single UMM can span years (e.g. the 2021 OGE notification with eventStop=2030-01-01). Filter on eventStatus = "Active" and the relevant event_start window, not on publicationDateTime alone. vault Bronze sample L78-79
