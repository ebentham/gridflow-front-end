Overview

What this dataset is.

interruptible_booked holds the interruptible-product capacity shippers have already reserved at each point — the demand-side complement to interruptible_available. It shares the canonical ENTSO-G operational-data shape with the other indicators; the differentiator is indicator="Interruptible Booked", and rows carry the booking platform. Each row is one operator-point-direction's booked interruptible capacity for a booking-validity window.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/operationalData with indicator=Interruptible Booked, queried per operator-point-direction over a gas-day range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as InterruptibleBookedTransformer), with no Pydantic class — columns are derived dynamically.

Booked interruptible rows carry curtailment risk and should be modelled with a discount factor rather than as guaranteed flow. Divide by interruptible_available for utilisation and add to firm_booked for the total booked view; cross-check nominations against the sum. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Interruptible utilisation: booked / available per point
WITH a AS (
  SELECT point_key, direction_key, value AS avail
  FROM read_parquet('data/silver/entsog/interruptible_available/**/*.parquet')
  WHERE period_from <= current_date AND period_to > current_date
),
b AS (
  SELECT point_key, direction_key, value AS booked
  FROM read_parquet('data/silver/entsog/interruptible_booked/**/*.parquet')
  WHERE period_from <= current_date AND period_to > current_date
)
SELECT a.point_key, a.direction_key,
       (b.booked * 100.0 / NULLIF(a.avail, 0)) AS utilisation_pct
FROM a JOIN b USING (point_key, direction_key)
ORDER BY utilisation_pct DESC;

Caveats

Things to know before using this.

These are specific to interruptible_booked. Also read the ENTSO-G-wide caveats on the vendor page.

01
Indicator string is exact-case

Connector sends the literal "Interruptible Booked". OPERATIONAL_INDICATORS["interruptible_booked"] · connectors/entsog/endpoints.py L104

02
Interruptible bookings carry curtailment risk

Booked rows are not guaranteed to flow; shippers pay-as-used minus curtailment. Model with a discount factor. domain knowledge / CMP mechanism docs

03
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

04
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

05
Booking platform attribution varies regionally

PRISMA dominates EU-wide; RBP (Hungary/SK) and GSA (FR/IT) appear in regional sub-markets. Don't filter on platform without checking the operator-region map. vault Bronze sample platform-key examples
