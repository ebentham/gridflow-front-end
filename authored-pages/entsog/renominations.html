Overview

What this dataset is.

renominations holds shipper renominations — within-day revisions to the intra-day flow quantities originally nominated at each point. It is the canonical signal for late shifts in expected gas supply and demand. It shares the operational column set with nominations; the differentiators are indicator="Renomination" and the presence of original_period_from (the original gas-day being revised). Each row is one revised nomination at an operator-point-direction.

It is sourced from the ENTSO-G Transparency Platform at /api/v1/operationalData?indicator=Renomination, queried per operator-point-direction over a gas-day range with timeZone=UCT and periodType=day. The raw JSON lands in bronze and is written to silver via GenericEntsogJsonTransformer (subclassed as RenominationsTransformer), with no Pydantic class — columns are derived dynamically.

Comparing renominations against the initial nominations measures within-day drift — a useful nowcasting signal that often clusters around interruption windows — while physical_flows and allocations grade the final outcome. Schema and API verified against the gridflow connector source and the 2026-05-08 vault Bronze sample; ENTSO-G is a public, no-authentication API.

-- Within-day nomination drift: max - first per gas day per point
WITH ordered AS (
  SELECT period_from, point_key, direction_key, value, last_update_date_time,
         ROW_NUMBER() OVER (PARTITION BY period_from, point_key, direction_key
                            ORDER BY last_update_date_time) AS rev_n
  FROM read_parquet('data/silver/entsog/renominations/**/*.parquet')
)
SELECT period_from::date AS gas_day, point_key, direction_key,
       MAX(value) - MIN(value) AS drift_kwh
FROM ordered
WHERE period_from >= current_date - INTERVAL 30 DAY
GROUP BY 1, 2, 3
ORDER BY 1, drift_kwh DESC;
import polars as pl

df = pl.read_parquet("data/silver/entsog/renominations/**/*.parquet")
# Count renominations per point per gas day
counts = (
    df.group_by(["period_from", "point_key", "direction_key"])
      .agg(pl.len().alias("revisions"))
      .filter(pl.col("revisions") > 1)
      .sort("revisions", descending=True)
)
print(counts.head(10))
Caveats

Things to know before using this.

These are specific to renominations. Also read the ENTSO-G-wide caveats on the vendor page.

01
Indicator string is exact-case

Vendor rejects renomination / re-nomination; connector sends the literal "Renomination". vault Known Issues · OPERATIONAL_INDICATORS["renominations"] = "Renomination" · connectors/entsog/endpoints.py L99

02
Multiple renominations per gas day are expected

A single (period_from, point_key, direction_key) may carry multiple silver rows distinguished by last_update_date_time. Dedup on id (vendor concatenation includes the revision sequence). silver/entsog/generic.py L126-130

03
original_period_from identifies the revised gas-day

For revisions made after the gas-day window closes (rare), original_period_from differs from period_from. vault Bronze sample field reference

04
timeZone=UCT (vendor typo, intentional)

Connector sends UCT not UTC; sending UTC returns HTTP 400. connectors/entsog/endpoints.py L17 · ENTSOG_TIMEZONE

05
Empty windows return HTTP 404, not 200

ENTSO-G's empty convention is HTTP 404 + {"message":"No result found"}. Connector short-circuits as empty-bronze. connectors/entsog/client.py L24-28, L109-115

06
Period offset is CEST/CET even with timeZone=UCT

periodFrom carries +02:00/+01:00; parse_entsog_datetime_expr converts to UTC at silver. silver/entsog/datetime.py
