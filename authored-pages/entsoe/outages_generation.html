Overview

What this dataset is.

outages_generation holds unavailability of generation units per bidding zone in MW per unit per interval — mapped from ENTSO-E businessType=A53 (planned) / A54 (unplanned). It is the canonical EU outage feed for capacity-margin modelling: each row carries a unit_mrid and unit_name identifying the specific generator that is offline.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A80&BusinessType=A53&BiddingZone_Domain={EIC}, queried per zone. The raw XML lands in bronze, is validated against the dataset-specific class EntsoeOutagesGeneration (schemas/entsoe.py L126-151), and written to silver via OutagesGenerationTransformer in silver/entsoe/outages_generation.py — the older, A80-specific transformer file, distinct from the H7 outages family (outages_h7.py). Note the connector pins BusinessType=A53 by default — unplanned outages (A54) require manual override.

This dataset is the primary capacity-margin input: join against installed_capacity_units for the unit capacity denominator and compute availability factor; join against actual_generation_units to verify the output drop matches the published outage MW. The newer outages_production (A77) surface carries richer document metadata and adds production_type. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 against the unregistered gridflow default key.

-- Top-10 highest-MW unplanned outages last 30 days
SELECT timestamp_utc, area_code, unit_name, outage_type, unavailable_mw
FROM read_parquet('data/silver/entsoe/outages_generation/**/*.parquet')
WHERE outage_type = 'unplanned'
  AND timestamp_utc >= current_timestamp - INTERVAL 30 DAY
ORDER BY unavailable_mw DESC LIMIT 10;
import polars as pl

out = pl.read_parquet("data/silver/entsoe/outages_generation/**/*.parquet")
cap = pl.read_parquet("data/silver/entsoe/installed_capacity_units/**/*.parquet")
# Availability factor per unit (1 - sum(unavailable) / hours / capacity)
hours_unavail = out.group_by("unit_mrid").agg(
    pl.col("unavailable_mw").sum().alias("total_mwh_unavail")
).join(cap.select(["unit_mrid", "capacity_mw"]), on="unit_mrid")
print(hours_unavail.with_columns(
    (pl.col("total_mwh_unavail") / pl.col("capacity_mw")).alias("hours_equiv_unavail")
).sort("hours_equiv_unavail", descending=True).head())
Caveats

Things to know before using this.

These are specific to outages_generation. Also read the ENTSO-E-wide caveats on the vendor page.

01
outage_type derived from businessType

Transformer maps businessType=A53 → "planned", A54 → "unplanned". Other businessType values pass through unchanged. silver/entsoe/outages_generation.py L72-77

02
Connector pins BusinessType=A53 by default

endpoints.py extra_params pins A53 — only planned outages are fetched by default. Unplanned outages (A54) require manual override via **params. endpoints.py L54-56

03
Dedup on (timestamp_utc, unit_mrid) — no area code

Unique key drops area_code because unit_mrid is globally unique. Caveat: if the same unit_mrid appears in multiple areas (rare), only one row survives. silver/entsoe/outages_generation.py L88-90

04
Distinct from H7 outages family

outages_generation has its own transformer file (older, A80-specific). The H7 family (outages_consumption, _production, _transmission, _offshore_grid) shares outages_h7.py. gridflow source tree

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. .planning/reconciliation/entsoe/24-outages-generation-http-401.md
