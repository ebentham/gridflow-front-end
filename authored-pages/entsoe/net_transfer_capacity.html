Overview

What this dataset is.

net_transfer_capacity holds the day-ahead forecasted net transfer capacity in megawatts per zone-pair direction. NTC is asymmetric — the same border has two values, one per direction — and is the capacity ceiling the day-ahead implicit auction clears against. Each row is one directed border's hourly NTC value tagged with contract_MarketAgreement.Type=A01 (daily auction contract).

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A61&contract_MarketAgreement.Type=A01&In_Domain={EIC}&Out_Domain={EIC} — note the capitalised In_Domain / Out_Domain parameters, distinct from most ENTSO-E surfaces. The raw XML lands in bronze, is validated against the dataset-specific EntsoeNetTransferCapacity (not the shared H6 class — NTC has its own schema), and written to the silver parquet partition via NetTransferCapacityTransformer. The G5-W3 schema update (2026-05) added ingested_at to match the transformer's actual output columns.

NTC is the denominator for utilisation: pair against cross_border_flows for realised utilisation (flow / ntc), against offered_transfer_capacity_implicit for the capacity envelope the day-ahead auction cleared inside, and against transfer_capacity_use for the settlement-side cross-check. When NTC binds, price spreads between zones persist — making this dataset a first-class input for cross-border price arbitrage modelling. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 against the unregistered gridflow default key.

-- Interconnector utilisation per border-direction per hour
SELECT n.timestamp_utc, n.in_area_code, n.out_area_code,
       n.ntc_mw, f.flow_mw, f.flow_mw / n.ntc_mw AS utilisation
FROM read_parquet('data/silver/entsoe/net_transfer_capacity/**/*.parquet') n
JOIN read_parquet('data/silver/entsoe/cross_border_flows/**/*.parquet') f
  ON n.timestamp_utc = f.timestamp_utc
 AND n.in_area_code = f.in_area_code
 AND n.out_area_code = f.out_area_code
WHERE n.ntc_mw > 0
ORDER BY utilisation DESC LIMIT 50;
import polars as pl

ntc = pl.read_parquet("data/silver/entsoe/net_transfer_capacity/**/*.parquet")
# Per-border NTC profile — does NTC derate at predictable hours?
profile = ntc.with_columns(pl.col("timestamp_utc").dt.hour().alias("hour")).group_by(
    ["in_area_code", "out_area_code", "hour"]
).agg(pl.col("ntc_mw").mean().alias("mean_ntc"))
print(profile.sort(["in_area_code", "out_area_code", "hour"]).head(24))
Caveats

Things to know before using this.

These are specific to net_transfer_capacity. Also read the ENTSO-E-wide caveats on the vendor page.

01
Directional — same border has two NTC values

NTC is asymmetric — GB → FR can differ from FR → GB depending on which constraint binds. Pair both directions for the full interconnector picture. vault Known Issues

02
contract_MarketAgreement.Type=A01 pinned by connector

Connector pins A01 (daily auction). Other contract types (intraday A02, monthly A03) return different surfaces. endpoints.py L135-141

03
Domain params capitalised

A61 uses In_Domain / Out_Domain (capitalised) — different from the lowercase in_Domain / out_Domain of most ENTSO-E surfaces. Common footgun. endpoints.py L135-141 · vault Known Issues

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code) re-publication overwrites silently on dedup. silver/entsoe/net_transfer_capacity.py L73-75

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/19-net-transfer-capacity-http-401.md
