Overview

What this dataset is.

installed_capacity holds yearly installed generation capacity in megawatts per EIC bidding zone, broken out by Primary Source Resource (PSR) production type from the B01..B25 EIC codelist (solar, wind onshore, wind offshore, nuclear, gas, hard coal, biomass and the other categories). Each row is one production type's installed nameplate capacity in a single zone, anchored at the year-boundary timestamp.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A68&processType=A33, queried per bidding zone over a year-bounded date range. The raw XML lands in bronze as a GL_MarketDocument, is validated against EntsoeInstalledCapacity, and written to the silver parquet partition via InstalledCapacityTransformer.

This is the structural denominator for capacity-factor models — paired with actual_generation it gives the annual utilisation rate per PSR per zone, and on its own it tracks new-build and closure trends across the European fleet. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 for the unregistered gridflow default key.

-- Year-over-year mix change per zone
SELECT timestamp_utc::date AS year_start, area_code, production_type,
       capacity_mw, lag(capacity_mw) OVER (
         PARTITION BY area_code, production_type ORDER BY timestamp_utc
       ) AS prev_year_mw
FROM read_parquet('data/silver/entsoe/installed_capacity/**/*.parquet')
ORDER BY area_code, production_type, year_start;
import polars as pl

cap = pl.read_parquet("data/silver/entsoe/installed_capacity/**/*.parquet")
gen = pl.read_parquet("data/silver/entsoe/actual_generation/**/*.parquet")
# Capacity factor per PSR per zone (annual mean output / installed capacity)
util = gen.group_by(["area_code", "production_type"]).agg(
    pl.col("generation_mw").mean().alias("mean_mw"),
).join(cap, on=["area_code", "production_type"])
print(util.with_columns(
    (pl.col("mean_mw") / pl.col("capacity_mw")).alias("cf")
).sort("cf", descending=True).head())
Caveats

Things to know before using this.

These are specific to installed_capacity. Also read the ENTSO-E-wide caveats on the vendor page.

01
Use a yearly window — P1Y resolution

Short windows return EMPTY. Build period bounds at year boundaries. vault Known Issues #1

02
GB EMPTY post-Brexit

The GB capacity registry is via Elexon and DUKES instead of ENTSO-E. vault Known Issues #2

03
Distinct from installed_capacity_units

A68 is aggregate by PSR type. A71/A33 (installed_capacity_units) is unit-level. Same processType A33; select on dataset key. vault Known Issues #4

04
Intra-year ingestion is idempotent

Values change at year boundaries; intra-year re-ingestion just rewrites the same data on keep="last" dedup. silver/entsoe/installed_capacity.py L77-79

05
Entitlement-blocked on the default API key

The live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/16-installed-capacity-http-401.md
