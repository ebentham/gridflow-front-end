Overview

What this dataset is.

outages_consumption holds aggregated unavailability of consumption units in MW per bidding zone — large-load outages (industrial consumers, smelters, demand-response participants) published without unit identification for privacy reasons. Each row is one zone's aggregated unavailable consumption MW for a single interval, tagged with businessType=A53 (planned) or A54 (unplanned).

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A76&BusinessType=A53&BiddingZone_Domain={EIC}, queried per zone. The raw XML lands in bronze, is validated against the dataset-specific class EntsoeOutagesConsumption (schemas/entsoe.py L457-477), and written to silver via OutagesConsumptionTransformer in the H7 outages family (outages_h7.py). Unlike the generation/production outages which carry unit_mrid, this surface omits unit identification — the zone-level aggregate is all that's published.

Consumption outages are rare events — most days have zero publications in a given zone — but an unexpected industrial trip can materially affect short-term demand forecast accuracy. For the forecast-side view, load_forecast already incorporates expected (planned) consumption outages; this dataset surfaces the surprises. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 against the unregistered gridflow default key.

Caveats

Things to know before using this.

These are specific to outages_consumption. Also read the ENTSO-E-wide caveats on the vendor page.

01
No unit identification

A76 publishes aggregated zone-level outages — no unit_mrid field. Individual consumer plants are not identified (privacy / commercial-sensitivity). vault Known Issues

02
H7 transformer family

Uses shared _H7OutageTransformer base. Dedup on (timestamp_utc, area_code, business_type, timeseries_mrid) — no unit_mrid in the key. silver/entsoe/outages_h7.py L109-127

03
Rare events — empty partitions are normal

Consumption outages are published infrequently — many days have zero events in a given zone. Empty silver partitions are expected behaviour. vault Known Issues

04
Use load_forecast for the demand-side capacity view

For forecast-side demand modelling, load_forecast already incorporates expected consumption outages. This dataset is for unexpected outages after the forecast is published. domain knowledge

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. .planning/reconciliation/entsoe/23-outages-consumption-http-401.md
