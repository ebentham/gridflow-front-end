Overview

What this dataset is.

congestion_income is the congestion rent collected by TSOs in EUR per directed zone-pair — the income that arises when day-ahead prices diverge across a capacity-constrained border and implicit-allocated capacity earns the spread. It is another surface of the overloaded A25 DocumentType, selected with businessType=B10, and is distinct from auction_revenue (B07, explicit-auction proceeds). Each row carries (timestamp_utc, in_area_code, out_area_code, business_type) as the primary key and amount_eur as the value.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A25&businessType=B10&in_Domain={EIC}&out_Domain={EIC}, queried per directed border-pair. The raw XML lands in bronze and is validated against the shared H6 amount class EntsoeTransmissionMarketAmount (schemas/entsoe.py L577-594) — the same shape as auction_revenue — then written to silver via CongestionIncomeTransformer, which uses value_tag="price.amount" to parse currency into amount_eur.

It is the income half of cross-border congestion economics: read against congestion_management_costs (A92) it gives a net congestion P&L per zone, and tracked against day_ahead_prices it follows the spread × allocated-capacity identity. Together with auction_revenue it sums to total TSO interconnection income. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

-- Annual congestion income by border (last 12 months)
SELECT in_area_code, out_area_code,
       sum(amount_eur) AS annual_income_eur,
       count(*) FILTER (WHERE amount_eur > 0) AS active_hours
FROM read_parquet('data/silver/entsoe/congestion_income/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 365 DAY
GROUP BY 1, 2 ORDER BY annual_income_eur DESC;
import polars as pl

ci = pl.read_parquet("data/silver/entsoe/congestion_income/**/*.parquet")
da = pl.read_parquet("data/silver/entsoe/day_ahead_prices/**/*.parquet")
# Congestion income vs implicit spread
gb = da.filter(pl.col("area_code") == "10YGB----------A").select(["timestamp_utc", "price_eur_mwh"]).rename({"price_eur_mwh": "gb_price"})
fr = da.filter(pl.col("area_code") == "10YFR-RTE------C").select(["timestamp_utc", "price_eur_mwh"]).rename({"price_eur_mwh": "fr_price"})
spread = gb.join(fr, on="timestamp_utc").with_columns((pl.col("gb_price") - pl.col("fr_price")).abs().alias("spread"))
print(ci.join(spread, on="timestamp_utc").tail())
Caveats

Things to know before using this.

These are specific to congestion_income. Also read the ENTSO-E-wide caveats on the vendor page.

01
A25 is heavily overloaded

A25 covers congestion_income (B10), auction_revenue (B07), transfer_capacity_use (B05), net_positions (B09). Always select on (documentType, businessType). endpoints.py L229-291

02
Distinct from auction_revenue

Both are TSO income but from different allocation mechanisms — auction (explicit) vs congestion-rent (implicit). Sum both for total cross-border revenue. domain knowledge

03
H6 amount transformer

Uses H6 amount-transformer variant (value_tag="price.amount"). silver/entsoe/h6_market.py L121-124

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code, business_type) overwrites silently. silver/entsoe/h6_market.py L85-91

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/07-congestion-income-http-401.md
