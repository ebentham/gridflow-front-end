Overview

What this dataset is.

balancing_financial_expenses_income is the financial P&L of the balancing mechanism — the expenses a TSO pays to acquire balancing energy and the income it earns from absorbing surplus, in EUR per control area under DocumentType A87. It is the cash view that complements the physical balancing surfaces: where activated_balancing_qty gives MWh and activated_balancing_prices gives €/MWh, this gives the settled monetary total. Each row carries (timestamp_utc, area_code, business_type) as the primary key and amount_eur as the value, with business_type distinguishing expense from income line items.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A87&controlArea_Domain={EIC}, queried per control area. The raw XML lands in bronze and is validated against EntsoeBalancingFinancial (schemas/entsoe.py L680-696), then written to silver via BalancingFinancialExpensesIncomeTransformer in the H8 balancing family. Uniquely for an H8 dataset, the transformer runs with value_tag="price.amount" rather than "quantity" — so it outputs amount_eur instead of quantity_mw and reads the area from controlArea_Domain.

It closes the balancing-cost loop: joined against activated_balancing_qty it yields an implied €/MWh, and read alongside procured_balancing_capacity it separates reservation cost from activation cost. It also sits beside congestion economics (congestion_income) as one of the TSO's recurring financial flows. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

-- Daily net balancing P&L per zone
SELECT date_trunc('day', timestamp_utc) AS day, area_code,
       sum(CASE WHEN amount_eur > 0 THEN amount_eur ELSE 0 END) AS expense_eur,
       sum(CASE WHEN amount_eur < 0 THEN amount_eur ELSE 0 END) AS income_eur,
       sum(amount_eur) AS net_eur
FROM read_parquet('data/silver/entsoe/balancing_financial_expenses_income/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 30 DAY
GROUP BY 1, 2 ORDER BY 1, net_eur DESC;
import polars as pl

fin = pl.read_parquet("data/silver/entsoe/balancing_financial_expenses_income/**/*.parquet")
qty = pl.read_parquet("data/silver/entsoe/activated_balancing_qty/**/*.parquet")
# €/MWh implied by financial / quantity
joined = fin.group_by(["timestamp_utc", "area_code"]).agg(
    pl.col("amount_eur").sum().alias("total_eur")
).join(
    qty.group_by(["timestamp_utc", "area_code"]).agg(
        pl.col("quantity_mwh").sum().alias("total_mwh")
    ),
    on=["timestamp_utc", "area_code"]
)
print(joined.with_columns((pl.col("total_eur") / pl.col("total_mwh")).alias("eur_per_mwh")).tail())
Caveats

Things to know before using this.

These are specific to balancing_financial_expenses_income. Also read the ENTSO-E-wide caveats on the vendor page.

01
value_tag="price.amount" (not quantity)

A87 carries currency values, so the transformer parses <Point><price.amount> rather than <Point><quantity>. Output column is amount_eur. silver/entsoe/h8_balancing.py L197-199

02
Sign convention varies by businessType

Different businessType codes denote different P&L line items — sign convention varies. Aggregate over businessType for full P&L. domain knowledge

03
controlArea_Domain

Uses controlArea_Domain for the area param (renamed by the H8 transformer to area_code). silver/entsoe/h8_balancing.py L200

04
Revisions overwrite

Same (timestamp_utc, area_code, business_type) re-publication overwrites silently on dedup. silver/entsoe/h8_balancing.py L210

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/06-balancing-financial-expenses-income-http-401.md
