Overview

What this dataset is.

commercial_schedules holds the hourly aggregated final commercial schedules in megawatts per zone-pair direction — the nominated commercial flow after capacity allocation, tagged with businessType=A06. Each border is queried as a directed pair (in_Domain → out_Domain), so the GB ↔ FR cluster is two separate series. Each row is one border-direction's nominated schedule for a single PT60M interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A09&in_Domain={EIC}&out_Domain={EIC}, queried per directed border-pair over a date range. The raw XML lands in bronze, is validated against the shared H6-family class EntsoeTransmissionMarketQuantity, and written to the silver parquet partition via CommercialSchedulesTransformer — a subclass of the H6 family's _H6QuantityTransformer base. Per ADR-019 (2026-05-09) the registry-duplicate commercial_schedules_net_positions slug was removed; this dataset is the sole active A09 surface.

Commercial schedules are the nominated view of inter-zone transfer; pair against cross_border_flows to measure the intraday adjustment + losses gap, against net_transfer_capacity to compute allocation utilisation, against net_positions to reconcile zonal aggregates, and against redispatching_cross_border to understand TSO interventions after gate closure. Unlike most ENTSO-E transmission datasets, A09 is one of the 14 endpoints accessible with the free gridflow default API key.

-- Commercial vs physical gap per hour (GB-FR)
SELECT c.timestamp_utc, c.quantity_mw AS commercial,
       p.flow_mw AS physical,
       p.flow_mw - c.quantity_mw AS gap_mw
FROM read_parquet('data/silver/entsoe/commercial_schedules/**/*.parquet') c
JOIN read_parquet('data/silver/entsoe/cross_border_flows/**/*.parquet') p
  ON c.timestamp_utc = p.timestamp_utc
 AND c.in_area_code = p.in_area_code
 AND c.out_area_code = p.out_area_code
WHERE c.in_area_code = '10YGB----------A'
  AND c.out_area_code = '10YFR-RTE------C'
ORDER BY abs(gap_mw) DESC LIMIT 20;
import polars as pl

sch = pl.read_parquet("data/silver/entsoe/commercial_schedules/**/*.parquet")
# Net commercial position per zone = sum of inbound - sum of outbound
inbound = sch.group_by(["timestamp_utc", "out_area_code"]).agg(
    pl.col("quantity_mw").sum().alias("inbound")
).rename({"out_area_code": "area"})
outbound = sch.group_by(["timestamp_utc", "in_area_code"]).agg(
    pl.col("quantity_mw").sum().alias("outbound")
).rename({"in_area_code": "area"})
net = inbound.join(outbound, on=["timestamp_utc", "area"]).with_columns(
    (pl.col("inbound") - pl.col("outbound")).alias("net_mw")
)
print(net.tail())
Caveats

Things to know before using this.

One of the items below is a data-source discrepancy — a stale reconciliation finding now closed by the H6 refactor. Surfaced so downstream consumers don't get spooked by an old "no schema declared" note that no longer applies. Also read the ENTSO-E-wide caveats on the vendor page.

01
A09 dedup resolved in V2 — commercial_schedules_net_positions removed

Per ADR-019 (2026-05-09), the duplicate commercial_schedules_net_positions key was removed. This dataset is the sole active A09 surface. endpoints.py L155-159 · vault Implementation Delta

02
Discrepancy · stale reconciliation finding
Pydantic class is shared across the H6 family

EntsoeTransmissionMarketQuantity covers 12 zone-pair quantity datasets. Reconciliation finding 43-commercial-schedules-manual-transformer-schema.md ("no importable Pydantic class declared") is stale — the class is importable and used by silver/entsoe/h6_market.py L27; the finding was closed by the H6 refactor. schemas/entsoe.py L557 · .planning/reconciliation/entsoe/43 (closed)

03
Variable-resolution curveType=A03

A09 payloads may use curveType=A03 which compresses unchanged intervals; the previous point's value persists until the next point. Downstream consumers must expand. vault Known Issues

04
Directional — sum both directions for true commercial position

Each border requires two requests. Net commercial position requires both directions paired. vault Known Issues

05
Not entitlement-blocked

A09 is one of the 14 ENTSO-E endpoints accessible with the free gridflow default API key. .planning/reconciliation/entsoe/ — no commercial_schedules-http-401.md
