Overview

What this dataset is.

dc_link_intraday_transfer_limits holds hourly intraday transfer limits for HVDC cross-border links in megawatts per zone-pair direction — the capacity ceiling applied to interconnectors like IFA, IFA2, ElecLink, BritNed, NordLink, and NSL for intraday continuous trading. Each row is one DC-link border-direction's intraday limit for a single PT60M interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A93&in_Domain={EIC}&out_Domain={EIC}, queried per directed border-pair. The raw XML lands in bronze, is validated against the shared H6-family class EntsoeTransmissionMarketQuantity, and written to silver via DcLinkIntradayTransferLimitsTransformer in silver/entsoe/h6_market.py. The "intraday" in the dataset name refers to which timeframe these limits apply to (intraday continuous trading), not how often they're published — publication is hourly.

A93 covers HVDC links only — AC interconnectors use the standard net_transfer_capacity surface. Pair against net_transfer_capacity to see when DC-link intraday limits derate below the day-ahead NTC, against cross_border_flows to confirm realised flow never exceeds this dataset's limit, and against offered_transfer_capacity_continuous to see the corresponding continuous-allocation product. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 against the unregistered gridflow default key.

-- Hours where DC-link limit derates below day-ahead NTC
SELECT d.timestamp_utc, d.in_area_code, d.out_area_code,
       n.ntc_mw, d.quantity_mw AS dc_intraday_limit,
       n.ntc_mw - d.quantity_mw AS derate_mw
FROM read_parquet('data/silver/entsoe/dc_link_intraday_transfer_limits/**/*.parquet') d
JOIN read_parquet('data/silver/entsoe/net_transfer_capacity/**/*.parquet') n
  ON d.timestamp_utc = n.timestamp_utc
 AND d.in_area_code = n.in_area_code
 AND d.out_area_code = n.out_area_code
WHERE n.ntc_mw - d.quantity_mw > 100
ORDER BY derate_mw DESC LIMIT 20;
import polars as pl

dc = pl.read_parquet("data/silver/entsoe/dc_link_intraday_transfer_limits/**/*.parquet")
# Per-link distribution of intraday derates
print(dc.group_by(["in_area_code", "out_area_code"]).agg(
    pl.col("quantity_mw").min().alias("min_mw"),
    pl.col("quantity_mw").max().alias("max_mw"),
).sort("min_mw"))
Caveats

Things to know before using this.

These are specific to dc_link_intraday_transfer_limits. Also read the ENTSO-E-wide caveats on the vendor page.

01
DC-only — not for AC interconnectors

A93 covers HVDC links (IFA, IFA2, ElecLink, BritNed, NordLink, NSL etc.). AC interconnectors use the standard A61 NTC surface. vault Known Issues

02
"Intraday" is the timing — not the cadence

The "intraday" in the name refers to which timeframe these limits apply to (intraday continuous trading), not how often they're published. Publication is hourly. vault Known Issues

03
Pydantic class is shared (H6 family)

EntsoeTransmissionMarketQuantity covers 12 H6 datasets. schemas/entsoe.py L557

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code, business_type) overwrites silently. silver/entsoe/h6_market.py L85-91

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. .planning/reconciliation/entsoe/13-dc-link-intraday-transfer-limits-http-401.md
