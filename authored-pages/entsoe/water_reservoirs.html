Overview

What this dataset is.

water_reservoirs is the weekly hydro reservoir filling level in MWh per bidding zone under DocumentType A72 — the stored potential energy sitting behind the dams of Nordic and Alpine hydro systems. It is a slow-moving, structural series: one reading per zone per week, tracing the seasonal arc of snowmelt fill and winter drawdown. Each row carries (timestamp_utc, area_code) as the primary key, stamped at the week boundary, and reservoir_mwh as the value — energy, not a percentage of capacity.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A72&processType=A16&in_Domain={EIC}, queried per zone over a multi-week window. The raw XML lands in bronze, is validated against EntsoeWaterReservoirs (schemas/entsoe.py L240-255), and written to the silver parquet partition via WaterReservoirsTransformer. Only zones with significant reservoir hydro publish — NO-1, SE-1, AT, CH and similar — so most continental and GB zones return EMPTY.

It is the canonical scarcity feature for Nordic price modelling: high fill points to abundant cheap hydro and soft prices, while a drought-driven low fill is the classic precursor to a price spike. Joined to day_ahead_prices it powers hydro-scarcity models, and read against actual_generation it tracks dispatch decisions against the reservoir trajectory. Its cross-vendor analogue is GIE storage for gas. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

-- Year-over-year reservoir level (Nordic zones)
SELECT timestamp_utc, area_code, reservoir_mwh,
       reservoir_mwh - lag(reservoir_mwh, 52) OVER (
         PARTITION BY area_code ORDER BY timestamp_utc
       ) AS yoy_change_mwh
FROM read_parquet('data/silver/entsoe/water_reservoirs/**/*.parquet')
WHERE area_code IN ('10YNO-1--------2', '10Y1001A1001A44P')
ORDER BY area_code, timestamp_utc;
import polars as pl

res = pl.read_parquet("data/silver/entsoe/water_reservoirs/**/*.parquet")
prices = pl.read_parquet("data/silver/entsoe/day_ahead_prices/**/*.parquet")
# Weekly mean price vs reservoir level — the Nordic hydro-scarcity feature
weekly_p = prices.group_by_dynamic("timestamp_utc", every="1w").agg(
    pl.col("price_eur_mwh").mean()
)
joined = res.join(weekly_p, on=["timestamp_utc", "area_code"])
print(joined.tail())
Caveats

Things to know before using this.

These are specific to water_reservoirs. Also read the ENTSO-E-wide caveats on the vendor page.

01
Only Nordic and Alpine zones publish

Most continental and GB zones return EMPTY — only zones with significant reservoir hydro (NO-1, SE-1, AT, CH) publish A72/A16. vault Known Issues

02
Unit is MWh (energy stored), not %

The value is potential energy in MWh if all reservoir water were released through the turbines, not a percentage of nameplate capacity. Compute percentage by joining against the TSO-declared maximum (rarely published — use rolling max). ENTSO-E spec

03
Weekly cadence (P7D)

Short windows (<1 week) may return EMPTY. Use weekly periods. vault Known Issues

04
GB EMPTY post-Brexit

No GB hydro reservoir data via ENTSO-E. GB pumped-storage facilities (Dinorwig, Cruachan) are not "reservoirs" in the A72 sense. vault Known Issues

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/34-water-reservoirs-http-401.md
