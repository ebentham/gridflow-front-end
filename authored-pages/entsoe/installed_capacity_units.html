Overview

What this dataset is.

installed_capacity_units is the yearly installed MW capacity declared for each individual EIC generation unit — the unit-level disaggregation of the aggregate installed_capacity (A68). Where the aggregate collapses a zone's fleet into per-PSR totals, this names every plant and its nameplate. Each row carries (timestamp_utc, area_code, unit_mrid) as the primary key, the unit's production_type (EIC PSR code, "unknown" if absent), and capacity_mw as the value, stamped at the year boundary.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A71&processType=A33&in_Domain={EIC}, queried per zone for a calendar year. The raw XML lands in bronze as a GL_MarketDocument, is validated against EntsoeInstalledCapacityUnits (schemas/entsoe.py L176-194), and written to silver via InstalledCapacityUnitsTransformer, which drops rows with an empty unit_mrid. Note documentType=A71 is reused for generation_forecast with a different processType.

It is the denominator for plant-level capacity-factor work: joined to actual_generation_units on unit_mrid it yields each unit's utilisation, and to generation_units_master_data it gains human-readable names and types. It surfaces the long tail of small generators (single wind farms, small CHP) that A68 folds into the aggregate. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

SELECT area_code, unit_mrid, unit_name, production_type, capacity_mw
FROM read_parquet('data/silver/entsoe/installed_capacity_units/**/*.parquet')
WHERE timestamp_utc = (
    SELECT max(timestamp_utc)
    FROM read_parquet('data/silver/entsoe/installed_capacity_units/**/*.parquet')

import polars as pl

cap = pl.read_parquet("data/silver/entsoe/installed_capacity_units/**/*.parquet")
gen = pl.read_parquet("data/silver/entsoe/actual_generation_units/**/*.parquet")
# Per-unit capacity factor (annual mean output / installed capacity)
util = gen.group_by("unit_mrid").agg(
    pl.col("generation_mw").mean().alias("mean_mw"),
).join(cap.select(["unit_mrid", "capacity_mw"]), on="unit_mrid")
print(util.with_columns(
    (pl.col("mean_mw") / pl.col("capacity_mw")).alias("cf")
).sort("cf", descending=True).head())
Caveats

Things to know before using this.

These are specific to installed_capacity_units. Also read the ENTSO-E-wide caveats on the vendor page.

01
A71 doc type shared with generation_forecast

A71 is reused for generation_forecast (A71/A01) with a different processType. Select on dataset key, not documentType alone. endpoints.py L100-102

02
Empty unit_mrid rows filtered

Transformer drops rows where unit_mrid == "". Aggregate-only data passes through other A71 surfaces, not here. silver/entsoe/installed_capacity_units.py L68

03
unit_mrid is opaque

Identifiers vary by TSO encoding (IGCC, EIC, internal). Treat as opaque strings. vault Known Issues

04
GB EMPTY post-Brexit

GB unit registry is via Elexon bmunits / BMRS, not ENTSO-E. vault Known Issues

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/17-installed-capacity-units-http-401.md
