Overview

What this dataset is.

actual_load is realised total load in MW for each EIC bidding zone — the canonical European demand observation. Where forecasts try to anticipate consumption, this is the settled actual against which forecast skill is measured. Resolution varies by zone: PT15M for DE/AT/FR, PT30M for GB, PT60M for most continental zones. Each row carries (timestamp_utc, area_code) as the primary key and load_mw as the value.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A65&processType=A16&outBiddingZone_Domain={EIC}, queried per zone over a date range. Note the outBiddingZone_Domain param style (domain_style="out_bidding_zone") — distinct from generation's in_Domain. The raw XML lands in bronze as a GL_MarketDocument, is validated against EntsoeActualLoad (schemas/entsoe.py L29-43), and written to silver via ActualLoadTransformer.

It is the demand half of residual-load modelling: joined against actual_generation, it produces the renewables-net residual demand (load − wind − solar) that drives short-term prices. Unlike A75 (empty for GB post-Brexit), GB continues to publish total load on A65/A16, so both Elexon ndf and ENTSO-E actual_load are usable for GB. A65/A16 is one of the ~14 ENTSO-E endpoints accessible with the free gridflow default API key. Verified against vendor docs on 2026-05-08.

-- Daily peak load per zone (last 30 days)
SELECT date_trunc('day', timestamp_utc) AS day, area_code,
       max(load_mw) AS peak_mw, min(load_mw) AS trough_mw,
       max(load_mw) - min(load_mw) AS swing_mw
FROM read_parquet('data/silver/entsoe/actual_load/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 30 DAY
GROUP BY 1, 2
ORDER BY 1, peak_mw DESC;
import polars as pl

load = pl.read_parquet("data/silver/entsoe/actual_load/**/*.parquet")
gen = pl.read_parquet("data/silver/entsoe/actual_generation/**/*.parquet")
# Residual load (renewables-net demand) — the canonical EU price feature
renewables = gen.filter(
    pl.col("production_type").is_in(["B16", "B18", "B19"])
).group_by(["timestamp_utc", "area_code"]).agg(
    pl.col("generation_mw").sum().alias("renew_mw")
)
res = load.join(renewables, on=["timestamp_utc", "area_code"], how="left").with_columns(
    (pl.col("load_mw") - pl.col("renew_mw").fill_null(0)).alias("residual_mw")
)
print(res.tail())
Caveats

Things to know before using this.

These are specific to actual_load. Also read the ENTSO-E-wide caveats on the vendor page.

01
outBiddingZone_Domain (not in_Domain)

A65/A16 uses outBiddingZone_Domain (domain_style="out_bidding_zone"). Wrong param style returns EMPTY. endpoints.py L31-33

02
Resolution varies by zone

DE/AT/FR are PT15M; GB is PT30M; most continental zones are PT60M. Filter on resolution before joining settlement-period datasets. vault Known Issues

03
GB publishes — unlike actual_generation

Unlike A75 (empty for GB post-Brexit), GB continues to publish total load on A65/A16 via ENTSO-E. Both Elexon ndf and ENTSO-E actual_load (GB) are usable. vault Known Issues

04
Revisions overwrite

Same (timestamp_utc, area_code) re-publication overwrites silently on dedup keep="last". No published_at for point-in-time. silver/entsoe/actual_load.py L62

05
Not entitlement-blocked

A65/A16 is one of the 14 ENTSO-E endpoints accessible with the free gridflow default API key. .planning/reconciliation/entsoe/ — no actual_load-http-401.md
