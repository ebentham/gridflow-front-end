Overview

What this dataset is.

actual_generation_units is realised MW output for individual generation units, identified by EIC unit mRID rather than aggregated to the PSR-type level. It is the unit-level disaggregation of actual_generation (A75) — where that surface gives you a zone's total wind or nuclear output, this names the specific plant behind each megawatt. Each row carries (timestamp_utc, area_code, unit_mrid) as the primary key, the unit's production_type (EIC PSR code), and generation_mw as the value.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A73&processType=A16&in_Domain={EIC}, queried per bidding zone over a date range. The raw XML lands in bronze as a GL_MarketDocument, is validated against EntsoeActualGenerationUnits (schemas/entsoe.py L219-237), and written to the silver parquet partition via ActualGenerationUnitsTransformer. The transformer drops rows with an empty unit_mrid, so silver carries only genuinely unit-attributed output.

It is the European analogue of GB's Elexon pn (physical notifications): the input for plant-by-plant capacity-factor studies when paired with installed_capacity_units for the denominator and generation_units_master_data for human-readable names. In practice A73/A16 is sparse — many TSOs publish per-unit data only through national portals such as SMARD rather than ENTSO-E. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

-- Per-unit daily mean output (last 30 days)
SELECT date_trunc('day', timestamp_utc) AS day, unit_mrid, unit_name,
       avg(generation_mw) AS mean_mw
FROM read_parquet('data/silver/entsoe/actual_generation_units/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 30 DAY
GROUP BY 1, 2, 3
ORDER BY 1, mean_mw DESC;
import polars as pl

df = pl.read_parquet("data/silver/entsoe/actual_generation_units/**/*.parquet")
# Per-unit capacity factor (vs declared capacity from installed_capacity_units)
cap = df.group_by("unit_mrid").agg(
    pl.col("generation_mw").mean().alias("mean_mw"),
    pl.col("generation_mw").max().alias("peak_mw"),
)
print(cap.sort("peak_mw", descending=True).head())
Caveats

Things to know before using this.

These are specific to actual_generation_units. Also read the ENTSO-E-wide caveats on the vendor page.

01
GB and most continental zones EMPTY

Both GB and DE-LU returned Acknowledgement reason 999 in live verification. Many TSOs publish per-unit data only via national portals (e.g. SMARD), not ENTSO-E. Use Elexon pn / boal for GB plant-by-plant. vault Known Issues #1

02
unit_mrid is opaque

Identifiers vary by TSO encoding (IGCC, EIC, internal). Treat as opaque strings; do not normalise or split. vault Known Issues #2

03
psrType filter ergonomics

psrType is documented as an optional query filter but is NOT in connector optional_params (endpoints.py L103-108). Callers can still inject via **params. vault Implementation Delta

04
Revisions overwrite

Same (timestamp_utc, area_code, unit_mrid) re-publication overwrites silently on dedup (keep="last"). silver/entsoe/actual_generation_units.py L62-65

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/02-actual-generation-units-http-401.md
