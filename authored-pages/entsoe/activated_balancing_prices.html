Overview

What this dataset is.

activated_balancing_prices holds the activated balancing-energy clearing price in EUR/MWh per control area, split by reserve type and direction. The transformer maps the raw businessType codes A95 / A96 / A97 / A98 to the human-readable reserve_type values fcr / afrr / mfrr / rr, and flowDirection A01 / A02 to direction up / down. Each row is one reserve type × direction's clearing price for a single activation interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A84&processType=A16, queried per control area over a date range. The connector pins businessType=A96 (aFRR) by default; other reserve types require **params override. The raw XML lands in bronze, is validated against EntsoeActivatedBalancingPrices, and written to the silver parquet partition via ActivatedBalancingPricesTransformer.

Pair with activated_balancing_qty on the full (timestamp_utc, area_code, reserve_type, direction) key for activation revenue (€/MWh × MWh), with contracted_reserves for the pre-arranged pool these activations draw from, and with imbalance_prices for the volume-weighted reserve cost. Down-direction prices are typically negative — the TSO pays to receive surplus generation back. Verified against vendor docs on 2026-05-08; A84/A16 is one of the ENTSO-E endpoints accessible with the free gridflow default API key.

-- Up-direction reserve prices by reserve_type per zone (last 30 days)
SELECT reserve_type, area_code, count(*) AS hours,
       avg(price_eur_mwh) AS mean_up_price,
       max(price_eur_mwh) AS peak_up_price
FROM read_parquet('data/silver/entsoe/activated_balancing_prices/**/*.parquet')
WHERE direction = 'up'
  AND timestamp_utc >= current_timestamp - INTERVAL 30 DAY
GROUP BY 1, 2 ORDER BY peak_up_price DESC;
import polars as pl

ap = pl.read_parquet("data/silver/entsoe/activated_balancing_prices/**/*.parquet")
# Up/down spread per reserve_type — width of the balancing-price ladder
print(ap.pivot(
    index=["timestamp_utc", "area_code", "reserve_type"],
    on="direction", values="price_eur_mwh"
).with_columns((pl.col("up") - pl.col("down")).alias("spread")).tail())
Caveats

Things to know before using this.

These are specific to activated_balancing_prices. Also read the ENTSO-E-wide caveats on the vendor page.

01
reserve_type code mapping

businessType A95/A96/A97/A98 → reserve_type "fcr"/"afrr"/"mfrr"/"rr" via replace_strict. Unknown businessType values fail the transformer. silver/entsoe/activated_balancing_prices.py L68-70

02
Connector pins businessType=A96 (aFRR)

endpoints.py extra_params pins businessType=A96 — only aFRR prices fetched by default. Override via **params to fetch other reserve types. endpoints.py L307

03
Down prices can be negative

Down activation means the TSO pays to receive surplus generation back. Negative prices are normal — don't filter them out. domain knowledge

04
controlArea_Domain (not bidding zone)

Control area and bidding zone EICs coincide for GB and the gridflow default zones, but may differ in some continental TSOs. endpoints.py L302-308

05
Not entitlement-blocked

A84/A16 is one of the 14 ENTSO-E endpoints accessible with the free gridflow default API key. .planning/reconciliation/entsoe/ — no activated_balancing_prices-http-401.md
