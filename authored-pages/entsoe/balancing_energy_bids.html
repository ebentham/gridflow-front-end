Overview

What this dataset is.

balancing_energy_bids holds the raw, per-bidder balancing-energy bid stack in MW per connecting area. Each bid surfaces as a separate row keyed by bid_mrid (renamed from the source <TimeSeries><mRID>), with direction, the original and the standardised market-product codes side-by-side. Each row is one individual bid for one settlement interval — typical evening-peak hours carry 20–50 separate bids per direction.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A37&processType=A47&businessType=B74, queried per connecting area with optional offset paging for large bid stacks. The raw XML lands in bronze, is validated against EntsoeBalancingEnergyBid, and written to the silver parquet partition via BalancingEnergyBidsTransformer in the shared H8 family.

This is the per-bidder ladder that procured_balancing_capacity clears against and activated_balancing_qty dispatches from; the area-aggregated view sits in aggregated_balancing_energy_bids when the use case is "how much was offered total" rather than ladder analysis. Use standard_market_product for cross-zone aggregation and original_market_product when matching to a specific bidder's local product naming. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 for the unregistered gridflow default key.

-- Bid stack depth per direction per hour (count of bids)
SELECT timestamp_utc, area_code, direction,
       count(*) AS bid_count,
       sum(quantity_mw) AS total_offered_mw
FROM read_parquet('data/silver/entsoe/balancing_energy_bids/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 7 DAY
GROUP BY 1, 2, 3 ORDER BY timestamp_utc DESC LIMIT 100;
import polars as pl

bids = pl.read_parquet("data/silver/entsoe/balancing_energy_bids/**/*.parquet")
# Bid concentration — which standard products dominate the stack?
print(bids.group_by(["standard_market_product", "direction"]).agg(
    pl.col("quantity_mw").sum().alias("total_mw"),
    pl.col("bid_mrid").n_unique().alias("n_bids"),
).sort("total_mw", descending=True).head())
Caveats

Things to know before using this.

These are specific to balancing_energy_bids. Also read the ENTSO-E-wide caveats on the vendor page.

01
connecting_Domain (not area_Domain)

A37 uses connecting_Domain for the area param (renamed by the H8 transformer). endpoints.py L326-338

02
Offset paging

Connector exposes offset as optional — large bid stacks may require multiple requests with offset paging. Default offset is 0. endpoints.py L331-336

03
bid_mrid is the per-bid identifier

bid_mrid is renamed from <TimeSeries><mRID> — each bid is a separate TimeSeries. Dedup includes bid_mrid + direction. silver/entsoe/h8_balancing.py L109, L138

04
standard_market_product vs original_market_product

Standard = harmonised across TSOs (e.g. A04 = mFRR); original = bidder's local code. Use standard for cross-zone aggregation. schemas/entsoe.py L625-626

05
Entitlement-blocked on the default API key

The live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/05-balancing-energy-bids-http-401.md
