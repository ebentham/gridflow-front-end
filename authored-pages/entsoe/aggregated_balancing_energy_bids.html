Overview

What this dataset is.

aggregated_balancing_energy_bids holds the area-aggregated balancing-energy bid stack in MW per single area. The Pydantic class is the shared EntsoeBalancingEnergyBid — same fields as the per-bidder balancing_energy_bids surface — and the surface is distinguished by single-area dispatch (area_Domain, not connecting_Domain). The bid_mrid here identifies an aggregated tranche (one row per area × direction × product) rather than an individual bidder.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A24&processType=A51, queried per single area. The raw XML lands in bronze, is validated against the shared EntsoeBalancingEnergyBid class, and written to the silver parquet partition via AggregatedBalancingEnergyBidsTransformer in the shared H8 family.

Use this surface when the question is "how much was offered total" rather than per-bidder ladder analysis — the balancing_energy_bids dataset is the disaggregated source. Sum-check consistency with the per-bidder ladder on (timestamp_utc, area_code, direction), and pair with procured_balancing_capacity for the cleared fraction of the aggregated offer. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 for the unregistered gridflow default key.

-- Aggregated bid stack per zone per direction (last 7 days)
SELECT date_trunc('day', timestamp_utc) AS day, area_code, direction,
       sum(quantity_mw) AS total_aggregated_mw
FROM read_parquet('data/silver/entsoe/aggregated_balancing_energy_bids/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 7 DAY
GROUP BY 1, 2, 3 ORDER BY 1 DESC, total_aggregated_mw DESC;
import polars as pl

agg = pl.read_parquet("data/silver/entsoe/aggregated_balancing_energy_bids/**/*.parquet")
ind = pl.read_parquet("data/silver/entsoe/balancing_energy_bids/**/*.parquet")
# Aggregate-vs-individual consistency (sanity check)
sum_ind = ind.group_by(["timestamp_utc", "area_code", "direction"]).agg(
    pl.col("quantity_mw").sum().alias("ind_total_mw")
)
sum_agg = agg.group_by(["timestamp_utc", "area_code", "direction"]).agg(
    pl.col("quantity_mw").sum().alias("agg_total_mw")
)
print(sum_ind.join(sum_agg, on=["timestamp_utc", "area_code", "direction"]).tail())
Caveats

Things to know before using this.

These are specific to aggregated_balancing_energy_bids. Also read the ENTSO-E-wide caveats on the vendor page.

01
area_Domain (not connecting_Domain)

A24 uses area_Domain (different from A37's connecting_Domain). The H8 transformer renames both to area_code so silver consumers see a uniform column. silver/entsoe/h8_balancing.py L143 · endpoints.py L339-344

02
Reuses balancing_energy_bids Pydantic schema

Same EntsoeBalancingEnergyBid class. The bid_mrid field here represents the aggregate identifier rather than an individual bidder. schemas/entsoe.py L616

03
Aggregation is by area + direction + product

A24/A51 aggregates A37 over individual bidders; rows are by area / direction / product combination. Use this dataset when the individual ladder is not needed. domain knowledge

04
Revisions overwrite

Same (timestamp_utc, area_code, bid_mrid, direction) re-publication overwrites silently on dedup. silver/entsoe/h8_balancing.py L138

05
Entitlement-blocked on the default API key

The live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/03-aggregated-balancing-energy-bids-http-401.md
