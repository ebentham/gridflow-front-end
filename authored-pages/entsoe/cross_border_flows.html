Overview

What this dataset is.

cross_border_flows holds the hourly physical cross-border flows in megawatts per zone-pair direction — one row per source-zone × destination-zone × hourly interval. Each border is queried as a directed pair (in_Domain → out_Domain), so the GB ↔ FR interconnector cluster is reported as two separate directional series. Each row is one border-direction's realised flow for a single PT60M interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A11&in_Domain={EIC}&out_Domain={EIC}, queried per directed border-pair over a date range. The raw XML lands in bronze as a GL_MarketDocument, is validated against EntsoeCrossborderFlow, and written to the silver parquet partition via CrossBorderFlowsTransformer. The G5-W3 schema update (2026-05) added resolution and ingested_at to align with the transformer's actual output columns.

Cross-border flows are the realised-physical view of inter-zone transfer; pair against commercial_schedules to measure intraday adjustment, against net_transfer_capacity for utilisation ratios, and against day_ahead_prices to study how price spreads drive flow direction. Unlike actual_generation, GB borders continue to publish via ENTSO-E post-Brexit (the flows are physical and reported by the European-side TSO). Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 against the unregistered gridflow default key.

Caveats

Things to know before using this.

These are specific to cross_border_flows. Also read the ENTSO-E-wide caveats on the vendor page.

01
Directional — pair both halves for net flow

Each border requires two requests (one per direction). True net flow = flow(GB→FR) − flow(FR→GB). vault Known Issues · endpoints.py L47-49

02
GB borders still publish post-Brexit

Unlike actual_generation, GB cross-border flows continue to publish via ENTSO-E (the flows are physical and reported by the European-side TSO). vault Known Issues

03
Resolution typically PT60M

Most borders publish PT60M. The G5-W3 schema update (2026-05) added resolution to the Pydantic class to match transformer output. schemas/entsoe.py L66-69

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code) re-publication overwrites silently on dedup. No published_at for point-in-time queries. silver/entsoe/cross_border_flows.py L68-70

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/10-cross-border-flows-http-401.md
