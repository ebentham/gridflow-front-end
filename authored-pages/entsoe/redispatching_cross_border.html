Overview

What this dataset is.

redispatching_cross_border is the cross-zone redispatching volume in MW per directed zone-pair — generation a TSO re-allocates between neighbouring zones to relieve a constrained interconnection. It is the direct-control counterpart to countertrading (which clears through the market), and shares DocumentType A63 with redispatching_internal, distinguished by businessType=A46 (cross-border) versus A85 (internal). Each row carries (timestamp_utc, in_area_code, out_area_code, business_type) as the primary key and quantity_mw as the value.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A63&businessType=A46&in_Domain={EIC}&out_Domain={EIC}, queried per directed border-pair. The raw XML lands in bronze and is validated against the shared H6 quantity class EntsoeTransmissionMarketQuantity (schemas/entsoe.py L557-574), then written to silver via RedispatchingCrossBorderTransformer. The connector pins businessType=A46 for this dataset.

It is the cross-border lever in the congestion-relief toolkit: read against redispatching_internal it separates within-zone from between-zone action, against countertrading it shows direct-control versus market mechanism, and its monetary footprint sits in congestion_management_costs (A92). Comparing it to cross_border_flows reveals where realised flow departs from the market-clearing schedule. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

-- Redispatching frequency by border (last 90 days)
SELECT in_area_code, out_area_code,
       count(*) FILTER (WHERE quantity_mw > 0) AS active_hours,
       sum(quantity_mw) AS total_redispatched_mwh
FROM read_parquet('data/silver/entsoe/redispatching_cross_border/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 90 DAY
GROUP BY 1, 2
ORDER BY total_redispatched_mwh DESC;
import polars as pl

rd = pl.read_parquet("data/silver/entsoe/redispatching_cross_border/**/*.parquet")
cost = pl.read_parquet("data/silver/entsoe/congestion_management_costs/**/*.parquet")
# Pair redispatch MW with congestion cost €
joined = rd.join(cost, on=["timestamp_utc", "in_area_code", "out_area_code"], suffix="_c")
print(joined.with_columns(
    (pl.col("amount_eur") / pl.col("quantity_mw")).alias("eur_per_mw")
).tail())
Caveats

Things to know before using this.

These are specific to redispatching_cross_border. Also read the ENTSO-E-wide caveats on the vendor page.

01
A63 shared with redispatching_internal (businessType A85)

A63 has two variants: A46 (cross-border, this dataset) and A85 (internal, redispatching_internal). Select on dataset key. endpoints.py L160-173

02
businessType A46 pinned by connector

Connector pins businessType=A46. endpoints.py L165

03
Pydantic class is shared (H6 family)

EntsoeTransmissionMarketQuantity covers 12 H6 datasets. Finding 67-redispatching-cross-border-no-table.md (wontfix v3-candidate) is stale. schemas/entsoe.py L557

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code, business_type) overwrites silently. silver/entsoe/h6_market.py L85-91

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/29-redispatching-cross-border-http-401.md
