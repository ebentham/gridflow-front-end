Overview

What this dataset is.

contracted_reserves holds the contracted reserve capacity in megawatts per control area, split by reserve type. The transformer maps businessType codes A95 / A96 / A97 / A98 to reserve_type fcr / afrr / mfrr / rr, same pattern as the activated_* surfaces. Note there is no direction field — contracted reserves are bidirectional capacity; direction surfaces only at activation.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A81&processType=A52&businessType=B95, with Type_MarketAgreement.Type=A01 (daily product) pinned by the connector. The raw XML lands in bronze, is validated against EntsoeContractedReserves, and written to the silver parquet partition via ContractedReservesTransformer.

This is the pre-arranged pool that activated_balancing_qty and activated_balancing_prices dispatch from — pair on (timestamp_utc, area_code, reserve_type) for reserve utilisation, with procured_balancing_capacity for the capacity-procurement decisions that feed this pool, and with cross_zonal_balancing_capacity to track imported reserve capacity. Verified against vendor docs on 2026-05-08; A81/A52 is accessible with the free gridflow default API key.

-- Daily mean contracted reserve by reserve_type per zone
SELECT date_trunc('day', timestamp_utc) AS day, area_code, reserve_type,
       avg(quantity_mw) AS mean_contracted_mw
FROM read_parquet('data/silver/entsoe/contracted_reserves/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 30 DAY
GROUP BY 1, 2, 3 ORDER BY 1 DESC, mean_contracted_mw DESC;
import polars as pl

cr = pl.read_parquet("data/silver/entsoe/contracted_reserves/**/*.parquet")
qty = pl.read_parquet("data/silver/entsoe/activated_balancing_qty/**/*.parquet")
# Activation rate: how much of contracted was actually called?
util = cr.join(
    qty.group_by(["timestamp_utc", "area_code", "reserve_type"]).agg(
        pl.col("quantity_mwh").sum().alias("activated_mwh")
    ),
    on=["timestamp_utc", "area_code", "reserve_type"], how="left"
).with_columns(
    (pl.col("activated_mwh") / pl.col("quantity_mw")).alias("util_hours_equiv")
)
print(util.tail())
Caveats

Things to know before using this.

These are specific to contracted_reserves. Also read the ENTSO-E-wide caveats on the vendor page.

01
Type_MarketAgreement.Type=A01 (daily) pinned

Connector pins A01 (daily) — only daily-product contracted reserves are fetched. Weekly / monthly tranches (A02 / A03) require manual override. The comment in endpoints.py notes this is mandatory per the ENTSO-E API spec despite the Postman catalog listing it as optional. endpoints.py L313-316

02
reserve_type code mapping

Same A95/A96/A97/A98 → "fcr"/"afrr"/"mfrr"/"rr" pattern as activated_*. silver/entsoe/contracted_reserves.py L64-66

03
No direction field

Contracted reserves are bidirectional capacity; direction surfaces only at activation. Pair with activated_balancing_qty for up/down split. schemas/entsoe.py L434-447

04
businessType=B95 pinned

endpoints.py pins businessType=B95. endpoints.py L316

05
Not entitlement-blocked

A81/A52 is accessible with the free gridflow default API key. .planning/reconciliation/entsoe/ — no contracted_reserves-http-401.md
