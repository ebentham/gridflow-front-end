Overview

What this dataset is.

outages_production holds unavailability of production units per bidding zone in MW per unit — the A77 successor to outages_generation (A80). It is richer: it carries production_type (EIC PSR code, e.g. B14 for nuclear), plus document_mrid, document_status, and timeseries_mrid for tracking outage revisions and withdrawals. Each row is one unit's unavailable MW for a single interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A77&BusinessType=A53&BiddingZone_Domain={EIC}, queried per zone. The raw XML lands in bronze, is validated against the dataset-specific class EntsoeOutagesProduction (schemas/entsoe.py L531-554), and written to silver via OutagesProductionTransformer in the H7 outages family (outages_h7.py). The H7 transformer preserves both the raw business_type (A53/A54) and the derived outage_type ("planned"/"unplanned"), useful when querying with either the raw code or the readable label.

Production-type tagging makes this the preferred outage surface for fuel-mix capacity modelling: filter on production_type=B14 for nuclear, B19 for wind onshore, etc. Pair against generation_units_master_data to enrich unit_mrid into human-readable names, and against outages_generation for the older A80 view of the same units. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

-- Unplanned outages by production type (last 90 days)
SELECT production_type, count(DISTINCT document_mrid) AS events,
       sum(unavailable_mw) AS total_mwh_unavail
FROM read_parquet('data/silver/entsoe/outages_production/**/*.parquet')
WHERE outage_type = 'unplanned'
  AND timestamp_utc >= current_timestamp - INTERVAL 90 DAY
GROUP BY 1 ORDER BY total_mwh_unavail DESC;
import polars as pl

out = pl.read_parquet("data/silver/entsoe/outages_production/**/*.parquet")
# Document-status timeline: how often does an outage get revised?
print(out.group_by("document_mrid").agg(
    pl.col("document_status").unique().alias("statuses"),
    pl.col("timeseries_mrid").n_unique().alias("timeseries_count")
).head())
Caveats

Things to know before using this.

These are specific to outages_production. Also read the ENTSO-E-wide caveats on the vendor page.

01
H7 transformer family

This dataset uses the shared _H7OutageTransformer base in outages_h7.py, distinct from the standalone outages_generation.py (A80). H7 family adds document-level metadata. silver/entsoe/outages_h7.py L24-106

02
outage_type derived via replace_strict

businessType (A53/A54/other) maps to outage_type ("planned"/"unplanned"/passthrough). Unknown business types pass through as the raw code. silver/entsoe/outages_h7.py L91-95

03
Document and TimeSeries metadata preserved

Unlike A80, A77 preserves document_mrid, document_status, and timeseries_mrid — useful for tracking outage revisions and document-status changes (active → withdrawn). schemas/entsoe.py L541-544

04
Dedup includes timeseries_mrid

(timestamp_utc, area_code, unit_mrid, timeseries_mrid) — different TimeSeries within the same document survive. silver/entsoe/outages_h7.py L202

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. .planning/reconciliation/entsoe/26-outages-production-http-401.md
