Overview

What this dataset is.

total_nominated_capacity holds total nominated capacity in megawatts per zone-pair direction, tagged with businessType=B08 ("Total nominated capacity"). Nomination is the step after allocation — a holder has bought capacity in advance, and nominates it when they actually intend to flow against it. Each row is one border-direction's nominated quantity for a single PT60M interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A26&businessType=B08, queried per directed border-pair. The raw XML lands in bronze, is validated against the shared H6-family class EntsoeTransmissionMarketQuantity, and written to the silver parquet partition via TotalNominatedCapacityTransformer in silver/entsoe/h6_market.py. A26 is shared with total_capacity_allocated (A29 businessType), so callers must always select on the dataset key — not DocumentType alone.

Allocated vs nominated = use-it-or-lose-it tracking — pair against total_capacity_allocated to see how much pre-bought capacity holders chose not to nominate, and against cross_border_flows to see how nomination translates into realised physical flow after intraday adjustment. Closely related to commercial_schedules but at the capacity-management rather than schedule level. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 against the unregistered gridflow default key.

-- Allocated vs nominated gap (use-it-or-lose-it tracking)
SELECT a.timestamp_utc, a.in_area_code, a.out_area_code,
       a.quantity_mw AS allocated_mw,
       n.quantity_mw AS nominated_mw,
       a.quantity_mw - n.quantity_mw AS lost_mw
FROM read_parquet('data/silver/entsoe/total_capacity_allocated/**/*.parquet') a
JOIN read_parquet('data/silver/entsoe/total_nominated_capacity/**/*.parquet') n
  ON a.timestamp_utc = n.timestamp_utc
 AND a.in_area_code = n.in_area_code
 AND a.out_area_code = n.out_area_code
ORDER BY lost_mw DESC LIMIT 20;
import polars as pl

nom = pl.read_parquet("data/silver/entsoe/total_nominated_capacity/**/*.parquet")
phys = pl.read_parquet("data/silver/entsoe/cross_border_flows/**/*.parquet")
# Nominated vs realised — how often does nomination actually flow?
joined = nom.join(phys, on=["timestamp_utc", "in_area_code", "out_area_code"], suffix="_p")
print(joined.with_columns(
    (pl.col("flow_mw") - pl.col("quantity_mw")).alias("intraday_adj_mw")
).select(["timestamp_utc", "intraday_adj_mw"]).tail())
Caveats

Things to know before using this.

These are specific to total_nominated_capacity. Also read the ENTSO-E-wide caveats on the vendor page.

01
A26 shared with total_capacity_allocated

A26 is also used for total_capacity_allocated (A29 businessType). Select on dataset key. endpoints.py L259-270

02
businessType=B08 pinned by connector

Connector pins businessType=B08. endpoints.py L257

03
Pydantic class is shared (H6 family)

EntsoeTransmissionMarketQuantity covers 12 H6 datasets. Reconciliation finding 70-total-nominated-capacity-no-table.md (wontfix v3-candidate) is stale. schemas/entsoe.py L557

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code, business_type) overwrites silently. silver/entsoe/h6_market.py L85-91

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. .planning/reconciliation/entsoe/32-total-nominated-capacity-http-401.md
