Overview

What this dataset is.

wind_solar_forecast holds the day-ahead TSO-published variable-renewable generation forecast in megawatts per EIC bidding zone, covering exactly three PSR types — B16 (Solar), B18 (Wind Offshore), and B19 (Wind Onshore). Each row is one PSR's forecast for a single hourly settlement interval; non-VRE technologies (hydro, biomass, thermal) are not covered here — for those use generation_forecast (A71/A01).

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A69&processType=A01, queried per bidding zone over a date range. The raw XML lands in bronze, is validated against EntsoeWindSolarForecast, and written to the silver parquet partition via WindSolarForecastTransformer.

This is the canonical input for renewable-driven price-direction models — solar / wind forecast is the dominant short-term feature in DE-LU, NL and IE-SEM. Join against actual_generation on (timestamp_utc, area_code, production_type) where production_type ∈ {B16, B18, B19} for per-PSR forecast skill; sum-check against generation_forecast for the VRE share of the aggregate. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 for the unregistered gridflow default key.

-- Daily peak solar + wind forecast per zone
SELECT date_trunc('day', timestamp_utc) AS day, area_code,
       max(CASE WHEN production_type='B16' THEN generation_forecast_mw END) AS peak_solar,
       max(CASE WHEN production_type='B19' THEN generation_forecast_mw END) AS peak_wind_on,
       max(CASE WHEN production_type='B18' THEN generation_forecast_mw END) AS peak_wind_off
FROM read_parquet('data/silver/entsoe/wind_solar_forecast/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 7 DAY
GROUP BY 1, 2 ORDER BY 1, 2;
import polars as pl

fc = pl.read_parquet("data/silver/entsoe/wind_solar_forecast/**/*.parquet")
act = pl.read_parquet("data/silver/entsoe/actual_generation/**/*.parquet")
# Forecast skill: per-PSR forecast error
joined = fc.join(
    act.filter(pl.col("production_type").is_in(["B16", "B18", "B19"])),
    on=["timestamp_utc", "area_code", "production_type"],
)
err = joined.with_columns(
    (pl.col("generation_mw") - pl.col("generation_forecast_mw")).alias("err_mw")
)
print(err.group_by(["area_code", "production_type"]).agg(
    pl.col("err_mw").std().alias("rmse")
))
Caveats

Things to know before using this.

These are specific to wind_solar_forecast. Also read the ENTSO-E-wide caveats on the vendor page.

01
Only 3 PSR types (B16 B18 B19)

A69 covers solar (B16), wind offshore (B18), wind onshore (B19). Not other renewables (hydro, biomass) — for those use generation_forecast (A71/A01). schemas/entsoe.py L108

02
GB EMPTY post-Brexit

GB renewable forecasts are via Elexon windfor (wind only). No GB solar forecast on ENTSO-E. vault Known Issues

03
Forecast revisions overwrite

Same (timestamp_utc, area_code, production_type) re-publication overwrites silently on dedup. silver/entsoe/wind_solar_forecast.py L74-76

04
Resolution may be PT15M for some zones

Most zones publish PT60M, but a few publish PT15M. Filter on resolution before joining settlement-period datasets. vault Known Issues

05
Entitlement-blocked on the default API key

The live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/35-wind-solar-forecast-http-401.md
