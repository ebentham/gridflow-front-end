Overview

What this dataset is.

offered_transfer_capacity_explicit holds offered transfer capacity in megawatts per zone-pair direction for explicit allocations — capacity offered into yearly / monthly explicit auctions and tagged with Auction.Category=A01. Explicit allocation is the "buy capacity in advance" product: annual auctions sell baseload allocations and monthly auctions sell remaining capacity. Both surface through this dataset.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A31&auction.Category=A01&auction.Type=A01&contract_MarketAgreement.Type=A01 — note the lowercase in_Domain / out_Domain params on this variant (the continuous sibling uses capitalised). The raw XML lands in bronze, is validated against the shared H6-family class EntsoeTransmissionMarketQuantity, and written to silver via OfferedTransferCapacityExplicitTransformer.

Explicit OTC is one of three A31 variants — pair against offered_transfer_capacity_continuous and offered_transfer_capacity_implicit to decompose how a border's capacity is split across allocation methods. Pair with auction_revenue (A25/B07) for €/MW revenue calculations on the explicit tranches, and with total_capacity_allocated to compute unsold capacity per allocation type. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 against the unregistered gridflow default key.

-- Monthly explicit-OTC by border-direction (last 12 months)
SELECT date_trunc('month', timestamp_utc) AS month,
       in_area_code, out_area_code,
       avg(quantity_mw) AS mean_offered_mw
FROM read_parquet('data/silver/entsoe/offered_transfer_capacity_explicit/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 12 MONTH
GROUP BY 1, 2, 3
ORDER BY 1 DESC, mean_offered_mw DESC;
import polars as pl

ex = pl.read_parquet("data/silver/entsoe/offered_transfer_capacity_explicit/**/*.parquet")
cont = pl.read_parquet("data/silver/entsoe/offered_transfer_capacity_continuous/**/*.parquet")
# Capacity split: explicit vs continuous
print(ex.group_by(["in_area_code", "out_area_code"]).agg(
    pl.col("quantity_mw").sum().alias("explicit_total_mwh")
).join(
    cont.group_by(["in_area_code", "out_area_code"]).agg(
        pl.col("quantity_mw").sum().alias("continuous_total_mwh")
    ),
    on=["in_area_code", "out_area_code"],
))
Caveats

Things to know before using this.

These are specific to offered_transfer_capacity_explicit. Also read the ENTSO-E-wide caveats on the vendor page.

01
Lowercase domain params (in_Domain / out_Domain)

Unlike _continuous which uses capitalised In_Domain/Out_Domain, this variant uses lowercase. endpoints.py L213-228

02
Explicit auction is yearly + monthly product

Annual auctions sell baseload allocations; monthly auctions sell remaining capacity. Both surface through this dataset. domain knowledge

03
Pydantic class is shared (H6 family)

EntsoeTransmissionMarketQuantity covers 12 H6 datasets. Reconciliation finding 59-offered-transfer-capacity-explicit-no-table.md (wontfix v3-candidate) is stale. schemas/entsoe.py L557

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code, business_type) overwrites silently. silver/entsoe/h6_market.py L85-91

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. .planning/reconciliation/entsoe/21-offered-transfer-capacity-explicit-http-401.md
