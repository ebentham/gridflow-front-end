Overview

What this dataset is.

outages_transmission holds unavailability of transmission infrastructure in MW per zone-pair direction — interconnector and cross-border line outages with asset identification. Each row carries an asset_mrid and asset_name identifying the specific cable or line offline (e.g. "IFA Pole 1", "BritNed Pole 1") — the key A78 feature that allows linking a published outage to the corresponding NTC derate.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A78&BusinessType=A53&In_Domain={EIC}&Out_Domain={EIC} — note the capitalised In_Domain / Out_Domain parameters (same pattern as A61 NTC). The raw XML lands in bronze, is validated against the dataset-specific class EntsoeOutagesTransmission (schemas/entsoe.py L480-503), and written to silver via OutagesTransmissionTransformer in the H7 outages family (outages_h7.py).

Transmission outages are the primary explanation for NTC derates: when a published outage MW is large enough, the TSO reduces NTC by a comparable amount. Pair against net_transfer_capacity to test causality (does NTC drop coincide with the outage timestamp?), against cross_border_flows to confirm realised flow dropped, and against dc_link_intraday_transfer_limits for the intraday capacity view on DC-link outages. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

-- Asset-by-asset outage MW (last 30 days)
SELECT in_area_code, out_area_code, asset_name, outage_type,
       count(*) AS hours_unavail, max(unavailable_mw) AS peak_unavail_mw
FROM read_parquet('data/silver/entsoe/outages_transmission/**/*.parquet')
WHERE timestamp_utc >= current_timestamp - INTERVAL 30 DAY
GROUP BY 1, 2, 3, 4 ORDER BY peak_unavail_mw DESC LIMIT 20;
import polars as pl

out = pl.read_parquet("data/silver/entsoe/outages_transmission/**/*.parquet")
ntc = pl.read_parquet("data/silver/entsoe/net_transfer_capacity/**/*.parquet")
# Outage → NTC derate causality (does NTC drop coincide with outages?)
joined = out.join(ntc, on=["timestamp_utc", "in_area_code", "out_area_code"])
print(joined.select(["timestamp_utc", "asset_name", "unavailable_mw", "ntc_mw"]).tail())
Caveats

Things to know before using this.

These are specific to outages_transmission. Also read the ENTSO-E-wide caveats on the vendor page.

01
Zone-pair scope (In_Domain + Out_Domain capitalised)

A78 uses capitalised In_Domain / Out_Domain (same as A61 NTC). Different from the lowercase used by most other H6/H7 surfaces. endpoints.py L66-74

02
asset_mrid not unit_mrid

A78 uses asset_mrid and asset_name (cable / line) rather than unit_mrid (generation unit). Don't confuse the two — they live in different columns. schemas/entsoe.py L486-487

03
H7 transformer family

Uses shared _H7OutageTransformer base. Adds asset_mrid / asset_name and zone-pair domains. silver/entsoe/outages_h7.py L129-158

04
Dedup includes asset_mrid

(timestamp_utc, in_area_code, out_area_code, asset_mrid, timeseries_mrid) — different assets and TimeSeries survive separately. silver/entsoe/outages_h7.py L151-157

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. .planning/reconciliation/entsoe/27-outages-transmission-http-401.md
