Overview

What this dataset is.

offered_transfer_capacity_continuous holds offered transfer capacity in megawatts per zone-pair direction for the continuous-allocation product — capacity offered into the explicit continuous auction. Continuous allocation is the "buy capacity as you need it" product, distinct from explicit (yearly/monthly auctions) and implicit (clears with day-ahead price). Each row is one border-direction's continuous-product offered MW for a single PT60M interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A31&Auction.Type=A01&Contract_MarketAgreement.Type=A01&In_Domain={EIC}&Out_Domain={EIC} — note the capitalised In_Domain / Out_Domain on this variant; the explicit and implicit siblings use lowercase. The raw XML lands in bronze, is validated against the shared H6-family class EntsoeTransmissionMarketQuantity, and written to silver via OfferedTransferCapacityContinuousTransformer in silver/entsoe/h6_market.py.

Continuous OTC is one of three A31 variants — pair against offered_transfer_capacity_explicit and offered_transfer_capacity_implicit to decompose how a border's capacity is split across the three allocation methods. Total offered (sum of variants) is bounded by net_transfer_capacity, and the remaining-capacity calculation pairs against total_capacity_allocated. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration and returns HTTP 401 against the unregistered gridflow default key.

-- Continuous-OTC offered vs realised flow (GB-FR)
SELECT o.timestamp_utc, o.quantity_mw AS offered,
       f.flow_mw AS realised, o.quantity_mw - abs(f.flow_mw) AS unused_mw
FROM read_parquet('data/silver/entsoe/offered_transfer_capacity_continuous/**/*.parquet') o
JOIN read_parquet('data/silver/entsoe/cross_border_flows/**/*.parquet') f
  ON o.timestamp_utc = f.timestamp_utc
 AND o.in_area_code = f.in_area_code
 AND o.out_area_code = f.out_area_code
ORDER BY o.timestamp_utc DESC LIMIT 24;
import polars as pl

otc = pl.read_parquet("data/silver/entsoe/offered_transfer_capacity_continuous/**/*.parquet")
# Per-border offered capacity profile — does it follow NTC?
print(otc.group_by(["in_area_code", "out_area_code"]).agg(
    pl.col("quantity_mw").mean().alias("mean_offered_mw")
))
Caveats

Things to know before using this.

These are specific to offered_transfer_capacity_continuous. Also read the ENTSO-E-wide caveats on the vendor page.

01
A31 doc type shared across explicit / implicit / continuous

A31 is reused for three datasets — offered_transfer_capacity_continuous, _explicit, _implicit — differentiated by Auction.Type and Contract_MarketAgreement.Type extra params (capitalisation varies between variants). Select on dataset key, not DocumentType alone. endpoints.py L186-228

02
Capitalised domain params (In_Domain / Out_Domain)

A31 continuous uses In_Domain / Out_Domain (capitalised). The implicit and explicit variants use lowercase in_Domain / out_Domain — easy footgun. endpoints.py L195

03
Pydantic class is shared (H6 family)

EntsoeTransmissionMarketQuantity is shared across 12 H6 datasets. Reconciliation finding 58-offered-transfer-capacity-continuous-no-table.md (wontfix v3-candidate) is stale — class exists. schemas/entsoe.py L557

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code, business_type) re-publication overwrites silently on dedup. silver/entsoe/h6_market.py L85-91

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. .planning/reconciliation/entsoe/20-offered-transfer-capacity-continuous-http-401.md
