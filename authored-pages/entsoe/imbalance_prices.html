Overview

What this dataset is.

imbalance_prices holds the EU imbalance settlement clearing prices in EUR/MWh per control area, split into long (system surplus) and short (system deficit) directions. The transformer maps businessType codes A19 / A20 to direction long / short via replace_strict; unknown values raise. Each row is one direction's settlement price for a single 30-minute interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A85, queried per control area over a date range. The raw XML lands in bronze, is validated against EntsoeImbalancePrices, and written to the silver parquet partition via ImbalancePricesTransformer.

This is the EU cash-out signal — pair with imbalance_volume for settlement value (€/MWh × MWh), with day_ahead_prices to derive the imbalance-vs-day-ahead premium, and with current_balancing_state to relate the area control error to the resulting short-system scarcity premium. The GB-native equivalent is Elexon system_prices. Verified against vendor docs on 2026-05-08; A85 is one of the ENTSO-E endpoints accessible with the free gridflow default API key.

-- Short - long spread per zone (long-short scarcity premium)
SELECT s.timestamp_utc, s.area_code,
       s.price_eur_mwh AS short_price,
       l.price_eur_mwh AS long_price,
       s.price_eur_mwh - l.price_eur_mwh AS spread
FROM read_parquet('data/silver/entsoe/imbalance_prices/**/*.parquet') s
JOIN read_parquet('data/silver/entsoe/imbalance_prices/**/*.parquet') l
  ON s.timestamp_utc = l.timestamp_utc AND s.area_code = l.area_code
WHERE s.direction='short' AND l.direction='long'
ORDER BY spread DESC LIMIT 50;
import polars as pl

ip = pl.read_parquet("data/silver/entsoe/imbalance_prices/**/*.parquet")
da = pl.read_parquet("data/silver/entsoe/day_ahead_prices/**/*.parquet")
# Imbalance-vs-day-ahead premium per direction
premium = ip.join(da, on=["timestamp_utc", "area_code"]).with_columns(
    (pl.col("price_eur_mwh") - pl.col("price_eur_mwh_right")).alias("imbalance_premium")
)
print(premium.group_by(["area_code", "direction"]).agg(
    pl.col("imbalance_premium").mean()
))
Caveats

Things to know before using this.

These are specific to imbalance_prices. Also read the ENTSO-E-wide caveats on the vendor page.

01
direction code mapping

businessType=A19 → direction="long", A20 → direction="short" via replace_strict. Unknown businessType values raise. silver/entsoe/imbalance_prices.py L63-66

02
controlArea_Domain (not bidding zone)

Control area and bidding zone EICs coincide for GB and the gridflow default zones. endpoints.py L294

03
Short price ≥ long price under single-price reform

In most single-price-reformed markets, the short and long prices are equal under non-scarcity. They diverge during system stress. GB has been dual-price since 2018 reform completed. domain knowledge

04
Revisions overwrite

Imbalance settlement can be revised — keep="last" on dedup means the most recent revision wins. No published_at for PIT. silver/entsoe/imbalance_prices.py L75

05
Not entitlement-blocked

A85 is one of the 14 ENTSO-E endpoints accessible with the free gridflow default API key. .planning/reconciliation/entsoe/ — no imbalance_prices-http-401.md
