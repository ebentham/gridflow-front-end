Overview

What this dataset is.

auction_revenue is the revenue earned by TSOs from explicit transfer-capacity auctions, in EUR per directed zone-pair. It is the cash counterpart of explicit capacity allocation: where offered_transfer_capacity_explicit says how much capacity was on sale, this says what bidders paid for it. It is one surface of the heavily overloaded A25 DocumentType — selected with businessType=B07. Each row carries (timestamp_utc, in_area_code, out_area_code, business_type) as the primary key and amount_eur as the value.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A25&businessType=B07&in_Domain={EIC}&out_Domain={EIC}, queried per directed border-pair over a date range. The raw XML lands in bronze and is validated against the shared H6 amount class EntsoeTransmissionMarketAmount (schemas/entsoe.py L577-594), then written to silver via AuctionRevenueTransformer. The H6 amount transformer uses value_tag="price.amount" (rather than quantity) — it parses currency values into amount_eur.

It is the yardstick for cross-border capacity value: paired with offered_transfer_capacity_explicit on (timestamp_utc, border), it yields a €/MW implied yield, and read alongside congestion_income (A25/B10) it completes the two TSO income streams from interconnection. Schema verified against gridflow source on 2026-05-20; the live API requires extended ENTSO-E registration.

-- Monthly auction revenue by border-direction
SELECT date_trunc('month', timestamp_utc) AS month,
       in_area_code, out_area_code,
       sum(amount_eur) AS monthly_revenue_eur
FROM read_parquet('data/silver/entsoe/auction_revenue/**/*.parquet')
GROUP BY 1, 2, 3 ORDER BY 1 DESC, monthly_revenue_eur DESC;
import polars as pl

rev = pl.read_parquet("data/silver/entsoe/auction_revenue/**/*.parquet")
cap = pl.read_parquet("data/silver/entsoe/offered_transfer_capacity_explicit/**/*.parquet")
# €/MW: divide revenue by offered explicit capacity
joined = rev.join(
    cap.group_by(["timestamp_utc", "in_area_code", "out_area_code"]).agg(
        pl.col("quantity_mw").sum().alias("explicit_mw")
    ),
    on=["timestamp_utc", "in_area_code", "out_area_code"]
)
print(joined.with_columns(
    (pl.col("amount_eur") / pl.col("explicit_mw")).alias("eur_per_mw")
).tail())
Caveats

Things to know before using this.

These are specific to auction_revenue. Also read the ENTSO-E-wide caveats on the vendor page.

01
A25 is heavily overloaded

A25 covers auction_revenue (B07), transfer_capacity_use (B05), net_positions (B09), congestion_income (B10). Always select on (documentType, businessType). endpoints.py L229-291

02
H6 amount transformer (price.amount parsing)

Auction revenue uses the H6 amount-transformer variant; value_tag="price.amount" parses currency values. Output column is amount_eur. silver/entsoe/h6_market.py L121-124

03
Pydantic class is shared (H6 amount family)

EntsoeTransmissionMarketAmount covers 3 H6 amount datasets. schemas/entsoe.py L577

04
Revisions overwrite

Same (timestamp_utc, in_area_code, out_area_code, business_type) overwrites silently. silver/entsoe/h6_market.py L85-91

05
Entitlement-blocked on default API key

Live API returns HTTP 401 for the unregistered gridflow default key. Schema verified from gridflow source. .planning/reconciliation/entsoe/04-auction-revenue-http-401.md
