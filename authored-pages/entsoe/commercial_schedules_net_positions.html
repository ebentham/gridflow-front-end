Overview

What this dataset is.

Deprecated · ADR-019 · 2026-05-09

This dataset key was removed in V2. Use commercial_schedules for the underlying A09 data. This page exists only as a pointer for old references and to record the removal reasoning.

commercial_schedules_net_positions was a deprecated pointer in the gridflow connector registry that registered an identical EntsoeDocType to commercial_schedules — same A09 DocumentType, same XML at request time. The "net positions" interpretation suggested by the slug was never implemented; the legacy transformer emitted the same per-direction rows as CommercialSchedulesTransformer.

Per ADR-019 (2026-05-09), the registry entry was removed from connectors/entsoe/endpoints.py (see the removal comment at lines 155–159). No live ingestion runs against this slug; the CommercialSchedulesNetPositionsTransformer was deleted in the same V2 change. Historical parquet may remain under data/silver/entsoe/commercial_schedules_net_positions/ on some data directories, but no scheduled gold consumer reads from that partition.

A genuine signed net_position_mw dataset — pairing both zone-pair directions per period and emitting a single signed value — is tracked in the v3 backlog and would be added under a fresh key when a downstream consumer needs it. Meanwhile, all consumers should read silver.commercial_schedules for the directional view (or net_positions for the A25/B09 implicit-auction net-position surface, which is different DocumentType from this one). Verified against gridflow source on 2026-05-20.

Caveats

Things to know before using this.

One of the items below is a data-source discrepancy — the vault page still exists as a deprecation pointer, but the gridflow registry entry was removed in V2. Surfaced so downstream readers know why this slug appears in old references. Also read the ENTSO-E-wide caveats on the vendor page.

01
Discrepancy · vault vs gridflow registry
Removed in V2 (ADR-019)

The vault page (vault/entsoe/commercial_schedules_net_positions.md) still exists as a deprecation pointer marking status deprecated. The gridflow registry entry (endpoints.py L155-159) was removed in V2 because it registered an identical EntsoeDocType to commercial_schedules and returned the identical XML. The "net positions" interpretation was never implemented. endpoints.py L155-159 (removal comment) · vault deprecation note

02
Use commercial_schedules for the underlying data

All consumers should read silver.commercial_schedules. The API call shape is identical. vault L29-32

03
Legacy parquet may exist locally — not used by gold

Historical files under data/silver/entsoe/commercial_schedules_net_positions/… may remain in some data directories. Deleting them is a separate cleanup; no scheduled gold consumer reads them. vault L29-32

04
A real signed net_position_mw is v3 backlog

A genuine "net positions" dataset would require pairing both zone-pair directions per period and emitting a single signed value. It should be added under a new dataset key when a downstream consumer needs it — tracked in v3. vault L34-37

05
Not entitlement-blocked

The underlying A09 endpoint (now commercial_schedules) is accessible with the free gridflow default API key. .planning/reconciliation/entsoe/ — no 401 file for this slug
