Overview

What this dataset is.

forecast_margin holds the year-ahead forecast margin in megawatts per EIC bidding zone — the TSO-published capacity-adequacy signal, computed as generation forecast minus load forecast across a full forward year. The forecast_margin_mw column is signed: positive values mean comfortable headroom, negative values flag a TSO-anticipated supply shortfall. Each row is one bidding zone's forecast margin for a single hourly settlement interval.

It is sourced from the ENTSO-E Transparency Platform at /api?documentType=A70&processType=A33, queried per bidding zone over a year-bounded window using the outBiddingZone_Domain param style. The raw XML lands in bronze as a GL_MarketDocument, is validated against EntsoeForecastMargin, and written to the silver parquet partition via ForecastMarginTransformer.

The dataset is the LOLE (loss-of-load expectation) input for capacity-adequacy modelling — count of hours per year with forecast_margin_mw < 0 per zone. It pairs with generation_forecast and load_forecast_yearly as the conceptual decomposition, with installed_capacity for structural context, and with water_reservoirs for the latent hydro capacity that can swing margin in Nordic and Alpine zones. Verified against vendor docs on 2026-05-08; A70/A33 is one of the ENTSO-E endpoints accessible with the free gridflow default API key.

-- Hours with negative margin (capacity shortage) per zone per year
SELECT area_code, date_trunc('year', timestamp_utc) AS year,
       count(*) FILTER (WHERE forecast_margin_mw < 0) AS shortage_hours,
       min(forecast_margin_mw) AS worst_margin_mw
FROM read_parquet('data/silver/entsoe/forecast_margin/**/*.parquet')
GROUP BY 1, 2
ORDER BY 1, 2;
import polars as pl

mar = pl.read_parquet("data/silver/entsoe/forecast_margin/**/*.parquet")
# LOLE proxy: count of hours with margin < 0 per area per year
lole = mar.filter(pl.col("forecast_margin_mw") < 0).group_by(
    [pl.col("timestamp_utc").dt.year().alias("year"), "area_code"]
).agg(pl.len().alias("shortage_hours"))
print(lole.sort(["area_code", "year"]))
Caveats

Things to know before using this.

These are specific to forecast_margin. Also read the ENTSO-E-wide caveats on the vendor page.

01
outBiddingZone_Domain (not in_Domain)

A70/A33 uses outBiddingZone_Domain for the area param (domain_style="out_bidding_zone"). Wrong param style returns EMPTY. endpoints.py L132-134

02
Use a yearly window

Year-ahead forecast — short windows may return EMPTY or partial. Build period bounds at year boundaries. vault Known Issues

03
Margin is signed — negative = capacity shortage

forecast_margin_mw < 0 indicates TSO-anticipated supply shortfall (a key LOLE input). ENTSO-E spec

04
Revisions overwrite

Same (timestamp_utc, area_code) re-publication overwrites silently on dedup. No published_at for PIT. silver/entsoe/forecast_margin.py L62

05
Not entitlement-blocked

A70/A33 is one of the 14 ENTSO-E endpoints accessible with the free gridflow default API key (per Phase 7 D-06). .planning/reconciliation/entsoe/ — no forecast_margin-http-401.md
