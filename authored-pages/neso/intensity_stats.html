Overview

What this dataset is.

intensity_stats is aggregated GB carbon-intensity statistics (max / average / min) over a windowed range — the canonical regime feature for rolling carbon labels and per-period QA. Each window summarises the observed series as max / average / min gCO2/kWh plus a categorical intensity_index band. Each row represents one window's aggregate statistics.

It is sourced from the NESO Carbon Intensity API at api.carbonintensity.org.uk/intensity/stats/{from}/{to}, queried over a windowed datetime range, validated against CarbonIntensityStats; the silver transformer (IntensityStatsTransformer) is generated dynamically at import by register_neso_transformers(). The API is a public JSON-REST endpoint — no authentication, just an Accept: application/json header — and the transformer lifts the nested intensity.{max, average, min} aggregates to top-level columns before writing the silver parquet partition.

As a stable rolling-regime feature, it supports carbon labelling and per-period QA without the leakage risk of joining raw per-SP observations. Verified against vendor docs: 2026-05-08 · NESO Carbon Intensity · /intensity/stats ↗

Caveats

          
Things to know before using this.

          
These are specific to intensity_stats. Also read the NESO-wide caveats on the vendor page.

          

            

              
01

              

                
Aggregates derived from observed intensities, not raw computations

                
NESO computes max/avg/min from its own published intensity series — these values may not equal a downstream aggregation of carbon_intensity rows if revisions occurred between publish times. vault Modelling notes L114; NESO methodology.

              

            

            

              
02

              

                
Vendor allows 30-day window; connector chunks at 14

                
Vendor max window for stats is 30 days; the connector uses _MAX_DAYS_PER_REQUEST = 14 uniformly (connectors/neso/carbon_intensity.py L21) — conservative, not incomplete. Stats spans larger than 14 days are split into multiple windows. vault Implementation Delta L109; connectors/neso/carbon_intensity.py L21.

              

            

            

              
03

              

                
No leakage when joining to per-SP labels

                
Block stats include the SP they cover. Joining intensity_stats aggregates back to per-SP training data without windowing creates lookahead. Use stats only for stable rolling regime features, not labels. vault Modelling notes L114.

              

            

            

              
04

              

                
Transformer is dynamically generated

                
IntensityStatsTransformer is created at import time via type() in register_neso_transformers(). silver/neso/carbon_intensity.py L110-136.
