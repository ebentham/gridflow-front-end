Overview

What this dataset is.

intensity_current is current half-hour GB grid carbon intensity in gCO2/kWh — the canonical no-arg snapshot for live dashboards, carbon-aware scheduling, and "is it a clean hour" decisions. Each period carries an ahead-of-period forecast_gco2_kwh and a post-period actual_gco2_kwh, with a categorical intensity_index band of very low / low / moderate / high / very high. Values sit on the GB 30-minute settlement grid in UTC. Each row represents one half-hour settlement period.

It is sourced from the NESO Carbon Intensity API at api.carbonintensity.org.uk/intensity, queried for the current settlement period, validated against CarbonIntensity; the silver transformer (IntensityCurrentTransformer) is generated dynamically at import by register_neso_transformers(). The API is a public JSON-REST endpoint — no authentication, just an Accept: application/json header — and the transformer flattens the nested intensity.{forecast, actual, index} object into top-level columns before writing the silver parquet partition.

As the canonical GB carbon signal, it underpins marginal-emission-factor work alongside Elexon fuelhh, carbon-aware load shifting, and weather-driven carbon attribution with Open-Meteo. Verified against vendor docs: 2026-05-08 · NESO Carbon Intensity · /intensity ↗

Caveats

          
Things to know before using this.

          
These are specific to intensity_current. Also read the NESO-wide caveats on the vendor page.

          

            

              
01

              

                
Actuals lag — null is "not yet published"

                
actual_gco2_kwh is null for the current and most-recent periods until NESO publishes the post-period estimate. Treat null as "not yet available", not "zero carbon". vault Known Issues L99-100.

              

            

            

              
02

              

                
No-arg route — /intensity returns one row

                
Unlike carbon_intensity (range) and intensity_today (all-day), this endpoint returns exactly one record for the current half-hour. Use range siblings for time-series queries. connectors/neso/endpoints.py L46-51.

              

            

            

              
03

              

                
Transformer is dynamically generated

                
IntensityCurrentTransformer is not source-defined — it is created at import time by register_neso_transformers() (silver/neso/carbon_intensity.py L110-118) via type(). Static analysers (mypy, IDE) cannot inspect it without running registration. silver/neso/carbon_intensity.py L121-136.

              

            

            

              
04

              

                
Index thresholds drift as the grid decarbonises

                
intensity_index buckets (very low … very high) are vendor-managed and shift over time as the GB grid decarbonises. Use raw gCO2/kWh for back-fitted analytics; use the index only for stable user-facing displays. domain knowledge — NESO methodology.
