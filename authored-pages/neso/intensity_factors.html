Overview

What this dataset is.

intensity_factors is NESO's published gCO2/kWh emissions factor per generation fuel — the static reference table behind the entire NESO intensity series and the canonical MEF lookup. It is a static lookup — one factor_gco2_kwh per generation fuel — not a time series; renewables and nuclear are treated as zero-carbon under NESO methodology. Each row represents one fuel's emissions factor.

It is sourced from the NESO Carbon Intensity API at api.carbonintensity.org.uk/intensity/factors, queried as a static reference with no range parameters, validated against CarbonIntensityFactor; the silver transformer (IntensityFactorsTransformer) is generated dynamically at import by register_neso_transformers(). The API is a public JSON-REST endpoint — no authentication, just an Accept: application/json header — and the transformer expands the fuel → factor object into one row per fuel before writing the silver parquet file.

As the reference table behind the entire NESO intensity series, it is the canonical MEF lookup — weight it by an Elexon fuelhh fuel mix to attribute gCO2/kWh to fuels, the same factors that drive carbon-aware load shifting. Caveats

          
Things to know before using this.

          
These are specific to intensity_factors. Also read the NESO-wide caveats on the vendor page.

          

            

              
01

              

                
No timestamp_utc — reference schema

                
CarbonIntensityFactor inherits _NesoBase, not _TimestampedNesoBase — there is no timestamp_utc column. The dataset is a static lookup. schemas/neso.py L48-52.

              

            

            

              
02

              

                
Written as a single file, not partitioned

                
reference_dataset=True (endpoints.py L81) routes through _write_silver L87-100 — emits one intensity_factors.parquet at the silver root, overwriting on each ingest. No year=YYYY/month=MM/ partition tree. silver/neso/carbon_intensity.py L87-100.

              

            

            

              
03

              

                
Fuel keys are normalised to snake_case

                
Raw API keys like "Gas (Combined Cycle)" become gas_combined_cycle (silver/neso/carbon_intensity.py L362-363). Joins to fuelhh codes (uppercase) require explicit case mapping. silver/neso/carbon_intensity.py L362-363.

              

            

            

              
04

              

                
Renewables and nuclear are zero-carbon by methodology

                
NESO assigns factor_gco2_kwh = 0 to wind, solar, hydro, nuclear. Lifecycle emissions (manufacturing, fuel cycle) are out of scope — see Caveats #04 if you need full LCA factors. NESO methodology; vault Modelling notes L109.

              

            

            

              
05

              

                
Factor revisions silently update the file

                
The single-file write means a factor revision overwrites prior values with no version history in silver. Snapshot bronze for time-travel; do not assume factors are immutable. silver/neso/carbon_intensity.py L87-100; vault Modelling notes L109.
