Overview

What this dataset is.

generation_current is current GB generation mix percentage by fuel for the live half-hour — the canonical "what's powering the grid right now" snapshot for live dashboards and carbon-aware UI. Each period reports generation_percentage per fuel across the GB mix (gas, wind, solar, nuclear, imports and more), summing to 100%. Values sit on the GB 30-minute settlement grid in UTC. Each row represents one fuel's share of the mix in one half-hour.

It is sourced from the NESO Carbon Intensity API at api.carbonintensity.org.uk/generation, queried for the current settlement period, validated against GenerationMix; the silver transformer (GenerationCurrentTransformer) is generated dynamically at import by register_neso_transformers(). The API is a public JSON-REST endpoint — no authentication, just an Accept: application/json header — and the transformer flattens the nested generationmix[] array into one row per fuel before writing the silver parquet partition.

As the explanatory feature behind the GB carbon signal, it underpins marginal-emission-factor work alongside Elexon fuelhh, renewable-penetration tracking, and weather-driven attribution with Open-Meteo. Verified against vendor docs: 2026-05-08 · NESO Carbon Intensity · /generation ↗

Caveats

          
Things to know before using this.

          
These are specific to generation_current. Also read the NESO-wide caveats on the vendor page.

          

            

              
01

              

                
NESO API status: beta

                
Generation Mix - National carries the beta qualifier (endpoints.py L137). NESO may modify or deprecate without the versioning notice applied to main intensity routes. endpoints.py L137.

              

            

            

              
02

              

                
Dual response envelope

                
Live /generation returns {"data": {...}} (single object); the connector helper _data_entries (silver/neso/carbon_intensity.py L153-161) lifts the dict into a single-item list. Downstream readers of raw bronze must handle both shapes. vault Implementation Delta L109; silver/neso/carbon_intensity.py L153-161.

              

            

            

              
03

              

                
One row per fuel — long shape, not wide

                
Each call produces N silver rows (one per fuel in generationmix). Pivot to wide via polars.pivot or DuckDB's PIVOT clause for visualisation. silver/neso/carbon_intensity.py L387.

              

            

            

              
04

              

                
Transformer is dynamically generated

                
GenerationCurrentTransformer is created at import time via type() in register_neso_transformers(). silver/neso/carbon_intensity.py L110-136.
