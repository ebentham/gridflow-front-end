Overview

What this dataset is.

generation_pt24h is GB generation mix percentage by fuel for the 24 hours preceding a chosen datetime — the canonical rolling-window mix view for diurnal-pattern analysis and recent-history backtests. Each period reports generation_percentage per fuel across the GB mix (gas, wind, solar, nuclear, imports and more), summing to 100%. Values sit on the GB 30-minute settlement grid in UTC. Each row represents one fuel's share of the mix in one half-hour.

It is sourced from the NESO Carbon Intensity API at api.carbonintensity.org.uk/generation/{from}/pt24h, queried over the trailing 24-hour window, validated against GenerationMix; the silver transformer (GenerationPt24HTransformer) is generated dynamically at import by register_neso_transformers(). The API is a public JSON-REST endpoint — no authentication, just an Accept: application/json header — and the transformer flattens the nested generationmix[] array into one row per fuel before writing the silver parquet partition.

As the explanatory feature behind the GB carbon signal, it underpins marginal-emission-factor work alongside Elexon fuelhh, renewable-penetration tracking, and weather-driven attribution with Open-Meteo. Verified against vendor docs: 2026-05-08 · NESO Carbon Intensity · /generation/{from}/pt24h ↗

-- Hour-of-day fuel share over the prior 24h
SELECT extract('hour' FROM timestamp_utc) AS hod, fuel,
       avg(generation_percentage) AS mean_share
FROM read_parquet('data/silver/neso/generation_pt24h/**/*.parquet')
WHERE fuel IN ('wind', 'solar', 'gas', 'nuclear')
GROUP BY 1, 2
ORDER BY 1, 2;

            

          

        


        
        

          
Caveats

          
Things to know before using this.

          
These are specific to generation_pt24h. Also read the NESO-wide caveats on the vendor page.

          

            

              
01

              

                
NESO API status: beta

                
Generation Mix - National is beta (endpoints.py L143); subject to vendor changes without notice. endpoints.py L143.

              

            

            

              
02

              

                
Window anchored to SP enclosing {from}

                
The 24h look-back ends at the SP containing {from}. Mid-period datetimes snap to the boundary. NESO docs — settlement-period semantics.

              

            

            

              
03

              

                
One row per (SP × fuel) — ~480 rows per call

                
Each call yields ~48 SPs × ~10 fuels = ~480 rows. Plan storage / pagination accordingly. silver/neso/carbon_intensity.py L387.

              

            

            

              
04

              

                
Path placeholder {from_dt} internally

                
Docs document /generation/{from}/pt24h; connector templates as {from_dt}. vault Implementation Delta L106.

              

            

            

              
05

              

                
Transformer is dynamically generated

                
GenerationPt24HTransformer is created at import time via type() in register_neso_transformers(). silver/neso/carbon_intensity.py L110-136.
