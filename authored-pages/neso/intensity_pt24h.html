Overview

What this dataset is.

intensity_pt24h is GB carbon intensity for the 24 hours preceding a chosen datetime — the canonical look-back series for forecast-skill scoring, retrospective MEF, and rolling-window features. Each period carries an ahead-of-period forecast_gco2_kwh and a post-period actual_gco2_kwh, with a categorical intensity_index band of very low / low / moderate / high / very high. Values sit on the GB 30-minute settlement grid in UTC. Each row represents one half-hour settlement period.

It is sourced from the NESO Carbon Intensity API at api.carbonintensity.org.uk/intensity/{from}/pt24h, queried over the trailing 24-hour window, validated against CarbonIntensity; the silver transformer (IntensityPt24HTransformer) is generated dynamically at import by register_neso_transformers(). The API is a public JSON-REST endpoint — no authentication, just an Accept: application/json header — and the transformer flattens the nested intensity.{forecast, actual, index} object into top-level columns before writing the silver parquet partition.

As the canonical GB carbon signal, it underpins marginal-emission-factor work alongside Elexon fuelhh, carbon-aware load shifting, and weather-driven carbon attribution with Open-Meteo. Verified against vendor docs: 2026-05-08 · NESO Carbon Intensity · /intensity/{from}/pt24h ↗

-- Rolling 24h MAE: combine fw24h forecasts vs pt24h actuals
SELECT a.timestamp_utc::date AS day,
       avg(abs(f.forecast_gco2_kwh - a.actual_gco2_kwh)) AS mae_g
FROM read_parquet('data/silver/neso/intensity_pt24h/**/*.parquet') a
JOIN read_parquet('data/silver/neso/intensity_fw24h/**/*.parquet') f
  USING (timestamp_utc)
WHERE a.actual_gco2_kwh IS NOT NULL
GROUP BY 1
ORDER BY 1 DESC;

            

          

        


        
        

          
Caveats

          
Things to know before using this.

          
These are specific to intensity_pt24h. Also read the NESO-wide caveats on the vendor page.

          

            

              
01

              

                
Window anchored to the SP enclosing {from}

                
The pt24h window ends at the SP containing {from} and looks back 24h. Pass mid-period datetimes and they snap to the boundary. NESO docs — settlement-period semantics.

              

            

            

              
02

              

                
Most actuals populated — but tail still null

                
The most recent 1-2 SPs in the look-back may carry null actual_gco2_kwh if the post-period estimate hasn't published yet. Filter IS NOT NULL for skill calculations. vault Known Issues L99-100.

              

            

            

              
03

              

                
Path placeholder {from_dt} internally

                
Docs document /intensity/{from}/pt24h; connector templates as {from_dt}. URL identical. vault Implementation Delta L106.

              

            

            

              
04

              

                
Transformer is dynamically generated

                
IntensityPt24HTransformer is created at import time via type() in register_neso_transformers(). silver/neso/carbon_intensity.py L110-136.
