Overview

What this dataset is.

intensity_today is all 48 half-hour GB carbon-intensity records for the current day — the canonical "today so far" series for live dashboards, day-ahead carbon dispatch, and morning-of reporting. Each period carries an ahead-of-period forecast_gco2_kwh and a post-period actual_gco2_kwh, with a categorical intensity_index band of very low / low / moderate / high / very high. Values sit on the GB 30-minute settlement grid in UTC. Each row represents one half-hour settlement period.

It is sourced from the NESO Carbon Intensity API at api.carbonintensity.org.uk/intensity/date, queried across the current day, validated against CarbonIntensity; the silver transformer (IntensityTodayTransformer) is generated dynamically at import by register_neso_transformers(). The API is a public JSON-REST endpoint — no authentication, just an Accept: application/json header — and the transformer flattens the nested intensity.{forecast, actual, index} object into top-level columns before writing the silver parquet partition.

As the canonical GB carbon signal, it underpins marginal-emission-factor work alongside Elexon fuelhh, carbon-aware load shifting, and weather-driven carbon attribution with Open-Meteo. Verified against vendor docs: 2026-05-08 · NESO Carbon Intensity · /intensity/date ↗

Caveats

          
Things to know before using this.

          
These are specific to intensity_today. Also read the NESO-wide caveats on the vendor page.

          

            

              
01

              

                
Actuals lag — future SPs within today are null

                
For periods later than the current half-hour, actual_gco2_kwh is null. Use coalesce(actual, forecast) for "now-and-ahead" display logic. vault Known Issues L104.

              

            

            

              
02

              

                
No path params — always returns today (UTC)

                
/intensity/date is a no-arg route. For a specific date, use intensity_date (/intensity/date/{date}). The boundary follows UTC, not GB local time. connectors/neso/endpoints.py L52-57.

              

            

            

              
03

              

                
DST days carry 46 or 50 SPs

                
Spring-forward day returns 46 rows, autumn-back returns 50, not the docs' 48. The connector handles this via timestamp_utc directly. vault Known Issues L106; connectors/neso/carbon_intensity.py L156-168.

              

            

            

              
04

              

                
Transformer is dynamically generated

                
IntensityTodayTransformer is created at import time by register_neso_transformers() via type(). Static analysers cannot inspect it. silver/neso/carbon_intensity.py L110-136.
