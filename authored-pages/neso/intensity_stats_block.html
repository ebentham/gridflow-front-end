Overview

What this dataset is.

intensity_stats_block is GB carbon-intensity statistics split into configurable hour blocks — the canonical sub-day regime view for diurnal-pattern analysis and intra-day flexible-load planning. Each window summarises the observed series as max / average / min gCO2/kWh plus a categorical intensity_index band. Each row represents one window's aggregate statistics.

It is sourced from the NESO Carbon Intensity API at api.carbonintensity.org.uk/intensity/stats/{from}/{to}/{block}, queried over a windowed datetime range, validated against CarbonIntensityStats; the silver transformer (IntensityStatsBlockTransformer) is generated dynamically at import by register_neso_transformers(). The API is a public JSON-REST endpoint — no authentication, just an Accept: application/json header — and the transformer lifts the nested intensity.{max, average, min} aggregates to top-level columns before writing the silver parquet partition.

As a stable rolling-regime feature, it supports carbon labelling and per-period QA without the leakage risk of joining raw per-SP observations. Verified against vendor docs: 2026-05-08 · NESO Carbon Intensity · /intensity/stats/{from}/{to}/{block} ↗

-- Diurnal pattern: average intensity by 6h block-of-day over 30 days
SELECT extract('hour' FROM timestamp_utc) AS block_start_h,
       avg(average_gco2_kwh) AS mean_g,
       count(*) AS n
FROM read_parquet('data/silver/neso/intensity_stats_block/**/*.parquet')
WHERE timestamp_utc >= current_date - INTERVAL 30 DAY
GROUP BY 1
ORDER BY 1;

            

          

        


        
        

          
Caveats

          
Things to know before using this.

          
These are specific to intensity_stats_block. Also read the NESO-wide caveats on the vendor page.

          

            

              
01

              

                
{block} defaults to 24 in the connector

                
endpoints.py L132 sets default_values={"block": 24} — calling with no override yields one row per day. Pass block=6 for the diurnal split. endpoints.py L132; endpoints.py L16 DEFAULT_STATS_BLOCK_HOURS=24.

              

            

            

              
02

              

                
Block boundaries are clock-aligned, not solar-aligned

                
Blocks start at UTC midnight; sunrise/sunset bias falls inside blocks. For solar-aligned analysis, either request hour-of-day stats over intensity_period rows or compute aggregates client-side. NESO docs — block windowing semantics.

              

            

            

              
03

              

                
14-day connector chunking applies

                
_MAX_DAYS_PER_REQUEST=14 (connectors/neso/carbon_intensity.py L21) applies to stats_block as well as raw stats. Multi-week pulls split into multiple windowed requests. connectors/neso/carbon_intensity.py L21.

              

            

            

              
04

              

                
Transformer is dynamically generated

                
IntensityStatsBlockTransformer is created at import time via type() in register_neso_transformers(). silver/neso/carbon_intensity.py L110-136.
