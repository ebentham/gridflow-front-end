Overview

What this dataset is.

news is the AGSI service-announcement listing — the free-text feed where AGSI staff flag operator reporting issues, planned data outages, and corrections. It is the data-quality companion to the numeric storage datasets: an announcement names the affected operators (via a JSON-encoded entities list of {name, country, eic} dicts) and the time window it applies to. Each row is one announcement, keyed by its numeric url ID.

It is sourced from GIE's AGSI+ platform at agsi.gie.eu/api, queried with an x-key header, with no dedicated Pydantic class — columns are derived dynamically by AgsiJsonTransformer. NewsTransformer extends it with no override; columns emerge from the live JSON via _normalise_row + _camel_to_snake plus datetime/numeric heuristics. Naïve YYYY-MM-DD HH:MM:SS timestamps are parsed as UTC.

It matters as a masking signal: join news.entities[].eic to storage_reports.entity_code over the start_at / end_at window to mask affected periods before fitting storage models. The endpoint is registered implementation_phase = "deferred" — research-grade, not yet v0.7 production-promoted. Verified against vendor docs on 2026-05-08.

-- Announcements affecting a specific facility EIC, last 1 year
SELECT url,
       start_at,
       end_at,
       title
FROM read_parquet('data/silver/gie_agsi/news/**/*.parquet')
WHERE entities LIKE '%"eic":"21X-WIEHE--K"%'
  AND start_at >= current_date - INTERVAL 1 YEAR
ORDER BY start_at DESC;
import polars as pl
import json

df = pl.read_parquet("data/silver/gie_agsi/news/**/*.parquet")
# Unwrap entities JSON and explode into one row per affected EIC
exploded = (
    df.with_columns(pl.col("entities").map_elements(json.loads, return_dtype=pl.List(pl.Object)))
      .explode("entities")
      .with_columns([
          pl.col("entities").struct.field("eic").alias("eic"),
          pl.col("entities").struct.field("country").alias("country"),
      ])
      .drop("entities")
)
print(exploded.head(20))
Caveats

Things to know before using this.

These are specific to news. Also read the GIE-wide caveats on the vendor page.

01
DISCREPANCY · docs vs live response
Live response carries no last_page / total

The catalog declares the endpoint paginated, but the live response is a single ~2.6 MB envelope with 299 records and no pagination keys. Connector _last_page falls back to 1 — single-page behaviour is correct, the docs claim is unverifiable. vault Implementation Delta

02
url is a numeric ID, not a URL

Confusing naming: url: 1713470 is the announcement primary key, not an HTTP link. Use it as turl=<url> to fetch detail via news_item (though see the news_item caveat — the turl filter is silently ignored). vault Known Issues

03
Base64 logos inflate parquet size if untrimmed

Each entities element may carry a base64 PNG logo field; silver JSON-encodes the full array. Strip logos pre-ingest if parquet size matters. vault Known Issues

04
end_at frequently null for ongoing events

Reporting outages with no scheduled resumption have end_at: null. Treat null end_at as "still active" for masking logic. vault Known Issues

05
x-key header MUST be lowercase

Capitalised X-Key returns HTTP 401. vault Known Issues #1

06
implementation_phase = "deferred" — not in v0.7 production cycle

The endpoint is documented + live-served and the silver transformer is registered (silver/gie/agsi.py L628), but production scheduling is deferred. endpoints.py L159 · vault Known Issues
