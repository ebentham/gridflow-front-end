Overview

What this dataset is.

storage holds country-scoped daily gas-storage time-series from GIE's Aggregated Gas Storage Inventory (AGSI+). Its headline column, storage_pct_full, is the canonical winter-tightness indicator — gas in storage as a percentage of working-gas volume — alongside injection / withdrawal flows and capacities, all in GWh. The grain is the gas day, not the calendar day. Each row is one AGSI country's storage snapshot for a single gas day.

It is sourced from GIE's AGSI+ platform at agsi.gie.eu/api, queried per country with an x-key header, validated against GasStorage, and written to the silver parquet partition via GasStorageTransformer. Each gas_day starts at 06:00 UTC, not midnight — the gasDayStart field — so calendar joins need a +06:00 offset.

It matters because storage percent-full is the structural gas-tightness signal for the Oct–Mar withdrawal season: pair it with ENTSO-G pipeline nominations and Elexon CCGT generation to build the full European gas-balance picture, and with LNG send-out for the dual supply-buffer view. Schema verified against gridflow source on 2026-05-20; the live API requires a free GIE registration key.

-- European storage trajectory: percent-full per country, last 90 days
SELECT entity_code,
       gas_day,
       storage_pct_full,
       trend
FROM read_parquet('data/silver/gie_agsi/storage/**/*.parquet')
WHERE status IN ('E', 'C')                      -- exclude GB no-value rows
  AND gas_day >= current_date - INTERVAL 90 DAY
ORDER BY entity_code, gas_day;
import polars as pl

df = pl.read_parquet("data/silver/gie_agsi/storage/**/*.parquet")
# EU-wide rolling stock — sum country gas_in_storage and convert to TWh
eu = (
    df.filter(pl.col("status").is_in(["E", "C"]))
      .group_by("gas_day")
      .agg((pl.col("gas_in_storage_gwh").sum() / 1000).alias("eu_total_twh"))
      .sort("gas_day")
)
print(eu.tail(30))
Caveats

Things to know before using this.

These are specific to storage. Also read the GIE-wide caveats on the vendor page.

01
Unit is GWh — NOT MWh

GIE reports stocks in gas_in_storage_gwh and capacities in *_gwh_per_day. Divide by 1000 for TWh; multiply by 1000 to align with MWh power-market datasets. vault Known Issues · schemas/gie.py L24

02
GB is in the footprint but unusable post-Brexit

country=GB returns "United Kingdom (Pre-Brexit)" with "-" placeholders for all numeric fields and status: "N". _safe_float converts "-" → null. Filter WHERE status IN ('E', 'C') for GB-inclusive aggregates. vault Known Issues

03
Gas day starts at 06:00 UTC, NOT midnight

gas_day is a date. The gas day for 2026-05-06 runs 2026-05-06 06:00 UTC → 2026-05-07 06:00 UTC. Apply the +06:00 offset when joining intra-day calendar-day datasets. vault Known Issues · 20-domain/concepts/gas-day.md

04
x-key header MUST be lowercase

The vendor enforces lowercase. Capitalised X-Key (the default in many HTTP libraries) returns HTTP 401. vault Known Issues #1

05
last_page is the pagination source of truth

last_page is the global page count; total is the per-page row count. Iterating on total mis-paginates. vault Known Issues #2

06
gas_day_start_validation is enforced by the connector

A date=YYYY-MM-DD query that returns a different gas day fails loudly — a guard against silent vendor caching errors. vault Known Issues
