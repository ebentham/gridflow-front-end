Overview

What this dataset is.

about_summary is the AGSI reference payload that organises storage operators in a nested tree — region → country → company → facility. It is the same operator universe as about_listing (~71 companies, ~140 entity rows) but a richer, hierarchical snapshot rather than a time series, carrying company-side metadata such as logos, publication links, and tariff references. Each silver row is one flattened entity — a company or a facility — keyed by (entity_level, entity_code).

It is sourced from GIE's AGSI+ platform at agsi.gie.eu/api, queried with an x-key header, with no dedicated Pydantic class — columns are derived dynamically by AgsiJsonTransformer. AboutSummaryTransformer extends it and overrides _records_from_payload to call _about_summary_records, which recurses through the {"SSO": {"Europe": {<Country>: […]}}} tree and emits flat company + facility rows.

It matters as the labelling and enrichment layer for the storage stack: where about_listing is optimised for query planning, about_summary supplies the company logos, publication links, and human-readable names used to annotate storage_reports facts and UI surfaces. Schema verified against gridflow source on 2026-05-20; the live API requires a free GIE registration key.

Caveats

Things to know before using this.

These are specific to about_summary. Also read the GIE-wide caveats on the vendor page.

01
DISCREPANCY · live shape vs fixture
Live shape is {"SSO": {"Europe": {…}}}, fixture is {"data": {…}}

The live response uses a nested SSO → Europe → Country tree; the fixture uses a flat {"data": {"platform": "AGSI", "totalCompanies": "3", …}} envelope. _about_summary_companies (silver/gie/agsi.py L563-580) recurses through both. The fixture is stale but functionally compatible. vault Implementation Delta

02
No Pydantic schema — reference data via dynamic columns

Unlike storage / storage_reports, there is no typed silver contract. Columns derive from the recursive walker plus the AgsiJsonTransformer dynamic pass. vault Silver layer note · silver/gie/agsi.py L402-411

03
Base64 image logos are summarised at silver

Bronze keeps the full base64 PNG bytes; silver replaces them with has_image: bool to control parquet size. vault Known Issues

04
DISCREPANCY · country value type
country value type varies

The live response carries country: "AT" (a string); some _nested_text paths expect a dict with .code / .name. The fallback at silver/gie/agsi.py L582-587 returns str(value) for the string case. vault Implementation Delta

05
x-key header MUST be lowercase

Capitalised X-Key returns HTTP 401. vault Known Issues #1

06
No pagination — single ~558 KB response

/api/about returns a monolithic JSON object with no last_page / total. The vendor expectation is one fetch per refresh. vault Known Issues
