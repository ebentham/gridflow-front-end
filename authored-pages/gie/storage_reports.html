Overview

What this dataset is.

storage_reports is the catalog-aligned full-scope AGSI storage feed — the same GasStorage shape as storage, but exercising all four query scopes rather than just country. The entity_level discriminator splits aggregate_type (EU rollup), country, company, and facility rows, all in GWh on the gas-day grain. Each row is one entity at one scope for a single gas day.

It is sourced from GIE's AGSI+ platform at agsi.gie.eu/api, queried per scope with an x-key header, validated against GasStorage, and written to silver via StorageReportsTransformer (a GasStorageTransformer subclass). The grain is the gas day — each gas_day starts at 06:00 UTC, not midnight — and company / facility EICs come from about_listing.

It matters when country roll-up hides facility-specific dynamics: facility-level percent-full is the structural gas-tightness signal at asset granularity for the Oct–Mar withdrawal season, supporting operator attribution and stress-test modelling. Pair with ENTSO-G point-level flows to validate network attribution. Schema verified against gridflow source on 2026-05-20; the live API requires a free GIE registration key.

-- Facility-level stock for German EWE Gasspeicher, last 30 days
SELECT gas_day,
       entity_name AS facility,
       gas_in_storage_gwh,
       working_gas_volume_gwh,
       storage_pct_full
FROM read_parquet('data/silver/gie_agsi/storage_reports/**/*.parquet')
WHERE entity_level = 'facility'
  AND country_code = 'DE'
  AND status IN ('E', 'C')
  AND gas_day >= current_date - INTERVAL 30 DAY
ORDER BY entity_name, gas_day;
import polars as pl

df = pl.read_parquet("data/silver/gie_agsi/storage_reports/**/*.parquet")
# Country-vs-facility consistency check: country total should ~match facility sum
country = df.filter(pl.col("entity_level") == "country").select(
    "gas_day", "country_code", pl.col("gas_in_storage_gwh").alias("country_gwh")
)
facility = (
    df.filter(pl.col("entity_level") == "facility")
      .group_by(["gas_day", "country_code"])
      .agg(pl.col("gas_in_storage_gwh").sum().alias("facility_sum_gwh"))
)
check = country.join(facility, on=["gas_day", "country_code"], how="inner")
print(check.tail())
Caveats

Things to know before using this.

These are specific to storage_reports. Also read the GIE-wide caveats on the vendor page.

01
entity_level discriminator must be filtered

Aggregate, country, company, and facility rows co-exist in the same parquet. Summing without filtering double-counts (aggregate row + country rows for the same gas day). Always filter WHERE entity_level = 'country' (or your chosen scope). vault Known Issues · silver/gie/agsi.py L160-165, L496-512

02
Unit is GWh — NOT MWh

Stocks in gas_in_storage_gwh; capacities in *_gwh_per_day. Vendor docs historically called this gas_in_storage_mwh, but the live key is gasInStorage and the units are GWh. vault Known Issues · schemas/gie.py L24

03
GB post-Brexit returns "-" placeholders

country=GB returns "United Kingdom (Pre-Brexit)" with status: "N"; only pre-Oct-2019 historical rows carry numeric data. Filter WHERE status IN ('E', 'C'). vault Known Issues

04
Gas day starts at 06:00 UTC

gas_day is a date. The gas day for 2026-05-06 runs 06:00 UTC → 06:00 UTC the next day. vault Known Issues · 20-domain/concepts/gas-day.md

05
x-key header MUST be lowercase

The vendor enforces lowercase header naming. X-Key (the HTTP-library default) returns HTTP 401. vault Known Issues #1

06
trend sign convention undocumented but consistent

Observed trend values match the daily change of storage_pct_full (negative for net withdrawal). The sign convention is not formally documented by the vendor. vault Implementation Delta
