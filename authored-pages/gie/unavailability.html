Overview

What this dataset is.

unavailability is the per-event AGSI storage outage and maintenance feed — the structured capacity-reduction record that explains sudden drops in storage capacity. Each event carries a facility, a start / end window, the GWh volume and GWh/day injection / withdrawal capacity removed, and a type × end_flag matrix (Planned/Unplanned × Confirmed/Estimate). Each row is one outage event, with windows gas-day-aligned at 06:00 UTC.

It is sourced from GIE's AGSI+ platform at agsi.gie.eu/api, queried per country with an x-key header, with no dedicated Pydantic class — columns are derived dynamically by AgsiJsonTransformer. UnavailabilityTransformer extends it; the nested country / company / facility dicts are JSON-encoded at silver, and naïve YYYY-MM-DD HH:MM:SS timestamps are parsed as UTC.

It matters as the capacity-reduction signal behind short-term withdrawal forecasts and gas-spread trades: join unavailability.facility.eic to storage_reports.entity_code to derive effective capacity net of outages, and cross-reference news for the free-text context behind the same events. Verified against vendor docs on 2026-05-08; the live API requires a free GIE registration key.

-- Total withdrawal capacity offline per country, by month
SELECT json_extract_string(country, '$.code') AS country,
       date_trunc('month', start) AS month,
       SUM(withdrawal) AS gwh_per_day_offline
FROM read_parquet('data/silver/gie_agsi/unavailability/**/*.parquet')
WHERE type = 'Planned'
  AND start >= current_date - INTERVAL 1 YEAR
GROUP BY 1, 2
ORDER BY 1, 2;
import polars as pl

df = pl.read_parquet("data/silver/gie_agsi/unavailability/**/*.parquet")
# Currently-active outages: window contains today
today = pl.lit(pl.datetime(2026, 5, 8, time_unit="us"))
active = (
    df.filter((pl.col("start") <= today) & (pl.col("end") >= today))
      .select("country", "facility", "start", "end", "withdrawal", "type", "end_flag")
      .sort("withdrawal", descending=True)
)
print(active.head(20))
Caveats

Things to know before using this.

These are specific to unavailability. Also read the GIE-wide caveats on the vendor page.

01
Nested-dict columns are JSON-encoded at silver

country, company, facility arrive as nested dicts ({"name": …, "code": …} or {"name": …, "eic": …}). AgsiJsonTransformer JSON-encodes them at silver. Use json_extract_string / pl.col(...).str.json_path_match(...) to unwrap. vault Known Issues · live 2026-05-08

02
GB returns zero rows by design

No UK storage outages are reported on AGSI. country=GB returns {"last_page": 1, "total": 0, "data": []}. vault Known Issues · live 2026-05-08

03
DISCREPANCY · docs ambiguity
Documentation ambiguity (v007 PDF)

The GIE v007 PDF leaves it unclear whether /api/unavailability is part of the AGSI API or a legacy/portal feature. The endpoint is live-served and well-formed (DE returned 30 records / 7 pages, 2026-05-08); treated as active. vault Implementation Delta · docs/gie_agsi_endpoint_catalog.yaml

04
DISCREPANCY · date-param semantics
Date-param alternatives produce different filters

The registry uses start / end; the live API also accepts from / to and produces a different last_page (different filter semantics). Gridflow standardises on start/end. vault Implementation Delta

05
x-key header MUST be lowercase

Capitalised X-Key returns HTTP 401. vault Known Issues #1

06
start / end arrive as naïve local strings

The vendor returns YYYY-MM-DD HH:MM:SS without timezone; _safe_datetime parses with a UTC fallback. vault Known Issues · silver/gie/agsi.py L46-61
