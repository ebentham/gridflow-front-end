Overview

What this dataset is.

news_item is the per-announcement detail endpoint — the intended per-turl lookup for a single announcement's full HTML body, keyed by the same numeric identifier the news listing exposes as url. In practice it is effectively zero-row: the live API ignores the turl filter and returns the full 299-record listing, which the transformer deliberately discards. Each row, when produced, would be one announcement's detail keyed by turl.

It is sourced from GIE's AGSI+ platform at agsi.gie.eu/api, queried with an x-key header, with no dedicated Pydantic class — columns are derived dynamically by AgsiJsonTransformer. NewsItemTransformer extends it and overrides _records_from_payload to discard list-shaped responses (which is exactly what the live API returns), so the silver output is empty against the current API build.

It matters as the planned enrichment layer: once the upstream filter is fixed it would supply full HTML bodies for facility-level masking, joining news_item.entities[].eic to storage_reports.entity_code. Until then, derive the same enrichment from news directly (url ↔ turl). The endpoint is registered implementation_phase = "deferred"; verified against vendor docs on 2026-05-08.

-- All news_item rows for a specific announcement turl
-- (Currently zero results — live `turl` filter is ignored upstream)
SELECT turl,
       start_at,
       end_at,
       title,
       details
FROM read_parquet('data/silver/gie_agsi/news_item/**/*.parquet')
WHERE turl = '1713470'
ORDER BY start_at DESC;
import polars as pl

# Fallback: until the upstream `turl` filter is fixed, derive news_item-style enrichment
# from the news listing directly. Same primary key (url ↔ turl), same fields available.
news = pl.read_parquet("data/silver/gie_agsi/news/**/*.parquet")
detail = news.filter(pl.col("url") == "1713470").select(
    pl.col("url").alias("turl"),
    "title", "summary", "details", "start_at", "end_at", "entities",
)
print(detail)
Caveats

Things to know before using this.

These are specific to news_item. Also read the GIE-wide caveats on the vendor page.

01
DISCREPANCY · documented filter vs live
Live turl filter is silently ignored

/api/news?turl=<id> returns the full 299-record listing byte-for-byte identical to /api/news (verified 2026-05-08, 2.6 MB). The connector's _is_news_item_detail discards the list shape, so silver yields zero rows. vault Known Issues · live 2026-05-08

02
Implementation phase = deferred

Registered as implementation_phase = "deferred" (endpoints.py L167) — not part of the gridflow v0.7 storage delivery. The endpoint is documented + live-served, but production scheduling is deferred. vault Known Issues

03
No Pydantic schema — AgsiJsonTransformer dynamic columns

Like other AGSI reference / news datasets, there is no typed silver contract; columns are dynamic. vault Silver layer note · silver/gie/agsi.py L451-462

04
x-key header MUST be lowercase

Capitalised X-Key returns HTTP 401. vault Known Issues #1

05
Workaround: read from news directly

Until the upstream turl filter behaves, derive news_item-style per-announcement enrichment from the news dataset on url (url and turl are the same identifier). domain knowledge · Python tab above
