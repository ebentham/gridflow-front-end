Overview

What this dataset is.

lng holds country-scoped daily LNG terminal inventory and send-out — the EU LNG-inflow side of the gas-supply balance. It is the ALSI counterpart to AGSI storage: where storage tracks underground stock, lng tracks regasification terminals, with dtrs_pct_full (Declared Technical Regasification Send-out, clamped 0–100) as the headline utilisation indicator. Each row is one ALSI country's terminal aggregate for a single gas day.

It is sourced from GIE's ALSI platform at alsi.gie.eu/api, queried per country with an x-key header, validated against LNGTerminal, and written to the silver parquet partition via LNGTerminalTransformer. As with AGSI, the grain is the gas day — each gas_day starts at 06:00 UTC, not midnight — and stocks and flows are reported in GWh, not MWh.

It matters because LNG send-out is the substitute supply buffer that offsets storage drawdown: pairing lng send-out with AGSI net-withdrawal, ENTSO-G pipeline nominations, and downstream Elexon CCGT generation gives the full dual-buffer picture for security-of-supply analysis. Values are projected from the 2026-05 ALSI footprint and respect the schema validator; verified against vendor docs on 2026-05-11.

# Note: ALSI uses `till=` for range end (NOT `to=` — different from AGSI).
-- EU-wide LNG send-out trajectory, last 90 days
SELECT gas_day,
       SUM(send_out_gwh) AS eu_send_out_gwh,
       SUM(injection_gwh) AS eu_injection_gwh,
       AVG(dtrs_pct_full) AS avg_pct_full
FROM read_parquet('data/silver/gie_alsi/lng/**/*.parquet')
WHERE send_out_gwh IS NOT NULL
  AND gas_day >= current_date - INTERVAL 90 DAY
GROUP BY gas_day
ORDER BY gas_day;
import polars as pl

df = pl.read_parquet("data/silver/gie_alsi/lng/**/*.parquet")
# Spain LNG net contribution: injection - send_out daily, last 30 days
es = (
    df.filter(pl.col("country_code") == "ES")
      .with_columns((pl.col("injection_gwh") - pl.col("send_out_gwh")).alias("net_storage_change_gwh"))
      .select("gas_day", "lng_in_storage_gwh", "send_out_gwh", "injection_gwh",
              "net_storage_change_gwh", "dtrs_pct_full")
      .sort("gas_day")
      .tail(30)
)
print(es)
Caveats

Things to know before using this.

These are specific to lng. Also read the GIE-wide caveats on the vendor page.

01
DISCREPANCY · ALSI vs AGSI range param
ALSI uses till= for the range end, AGSI uses to=

The ALSI range query is from=YYYY-MM-DD&till=YYYY-MM-DD; AGSI uses from/to. The connector's _fetch_country (used by gie_alsi) preserves the legacy till= form. vault lng.md query params · storage.md Implementation Delta

02
Unit is GWh — NOT MWh

Stocks in lng_in_storage_gwh; flows in send_out_gwh / injection_gwh (GWh per gas day). Same convention as AGSI. vault lng.md Silver schema

03
Gas day starts at 06:00 UTC, NOT midnight

gas_day is a date. The gas day for 2026-05-06 runs 06:00 UTC → 06:00 UTC the next day. vault lng.md Modelling notes · 20-domain/concepts/gas-day.md

04
x-key header MUST be lowercase

X-Key returns HTTP 401 — same enforcement as AGSI. shared GIE vendor convention

05
GB null values are publication absence, not zero

Unlike AGSI GB (Pre-Brexit "-" placeholders), ALSI GB still publishes — but lng_in_storage_gwh may carry null during reporting outages. Treat null as publication absence, not a zero LNG inventory. vault lng.md Modelling notes

06
dtrs_pct_full is clamped 0–100 by validator

LNGTerminal.dtrs_pct_full has a field_validator clamping to [0.0, 100.0] — identical pattern to GasStorage.storage_pct_full. schemas/gie.py L63-68
