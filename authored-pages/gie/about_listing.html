Overview

What this dataset is.

about_listing is a flat listing of the AGSI storage universe — every storage company and every storage facility, each with its 16-character EIC and, for facilities, the parent company's EIC. It is a snapshot reference table refreshed on change rather than a time-series: the two-level entity_level discriminator splits company (SSO operators) from facility (DSR assets). Each row represents one entity — a single company or a single storage facility — keyed by (entity_level, entity_code).

It is sourced from GIE's AGSI+ platform at agsi.gie.eu/api, queried with an x-key header, with no dedicated Pydantic class — columns are derived dynamically by AgsiJsonTransformer. AboutListingTransformer extends it and overrides _records_from_payload to call parse_listing_inventory, emitting flat rows for both companies and facilities; the dynamic-column pass adds any other vendor keys present.

It matters because it is the entity master the rest of the storage stack plans against: the company and facility EICs that drive the storage_reports company and facility query scopes come from here, and the country field defines the country footprint. Schema verified against gridflow source on 2026-05-20; the live API requires a free GIE registration key.

Schema

Silver layer columns.

No dedicated Pydantic class — columns are derived dynamically by AgsiJsonTransformer. AboutListingTransformer overrides _records_from_payload to call parse_listing_inventory (connectors/gie/endpoints.py L253-291), which emits flat rows for both companies and facilities. The shape below is the deterministic output of that override; the dynamic pass adds any other vendor keys present. This is a snapshot reference table — partitioned by ingestion date, not a gas-day time series.

-- Facility count per company, current snapshot only
SELECT company_code,
       company_name,
       country_code,
       COUNT(*) AS facility_count
FROM read_parquet('data/silver/gie_agsi/about_listing/**/*.parquet')
WHERE entity_level = 'facility'
  AND operational_end_date IS NULL  -- exclude decommissioned
GROUP BY company_code, company_name, country_code
ORDER BY facility_count DESC;
import polars as pl

df = pl.read_parquet("data/silver/gie_agsi/about_listing/**/*.parquet")
# Build the company → facility EIC map needed for storage_reports facility queries
inventory = (
    df.filter((pl.col("entity_level") == "facility")
              & pl.col("operational_end_date").is_null())
      .select("country_code", "company_code", "entity_code", "entity_name")
      .sort("country_code", "company_code")
)
print(inventory.head(20))
Caveats

Things to know before using this.

These are specific to about_listing. Also read the GIE-wide caveats on the vendor page.

01
No last_page / total — single-response listing

The live response is a top-level list of 71 companies (~53 KB, 2026-05-08) with no pagination envelope. _listing_rows accepts either the bare list or a {"data": [...]} wrapper. vault Known Issues · endpoints.py L454-463

02
No Pydantic schema — reference data via AgsiJsonTransformer

Unlike storage / storage_reports, this dataset has no typed silver contract. Columns are emitted by parse_listing_inventory plus the AgsiJsonTransformer dynamic-column pass. vault Silver layer note · silver/gie/agsi.py L414-442

03
country is a string, not a nested dict

The live shape carries country: "AT" (an ISO-2 string). Some other AGSI endpoints (e.g. unavailability) return {"name": …, "code": …} — listing differs. vault Known Issues

04
operational_end_date flags decommissioned facilities

A non-null operational_end_date means the facility no longer reports to AGSI. Filter WHERE operational_end_date IS NULL for a forward-looking inventory. vault Known Issues

05
x-key header MUST be lowercase

The vendor enforces lowercase. Capitalised X-Key (the default in many HTTP libraries) returns HTTP 401. vault Known Issues #1

06
DISCREPANCY · live shape vs fixture
Live shape is a top-level list; fixture is {"data": [...]}

The live response (2026-05-08) is a bare top-level list of company objects; the test fixture wraps it in a {"data": [...]} envelope. _listing_rows (endpoints.py L454-463) accepts either shape — the fixture is stale but functionally compatible. vault Implementation Delta
